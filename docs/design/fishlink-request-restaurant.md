## 共通仕様（重要）

**① 信頼ブロックは “率” を出さない（§5）**
- メタ行は **取引数＋登録時期** のみ。支払完了率・未払い率などの**率指標は出さない**。
- 理由：立ち上げ期は母数が小さく率が誤誘導になる（3件中1遅延＝67%でも「危ない客」に見える）。取引数＝“続いている”シグナルの方が頑健。危険な客は**運営がアカウント制限で対処**する設計（精算フロー）。
- **リアルタイムの「今支払い遅延中／問題報告中」警告も出さない**（同上）。
- ※未払い等のデータは内部で保持はする（将来必要になれば“率”でなく二値フラグで再検討）。今回はページに出さないだけ。

**② 地図を出す理由**
- 「誰向けの画面か」で決まる：**生産者ページ＝買い手向け→農家の場所は隠す**（私的＋中抜き）／**レストランページ＝農家向け→公開店舗＋配達先なので地図を見せる**。
- 配送管理（chat-spec）で「農家側だけレストランの配達先地図を出す」のと同じ原則。**新規レストランでも地図は表示**（場所は登録済み）。

**③ コメントのマスク＋モデレーション**（公開テキストのため）
- **連絡先マスク（自動）**：本文の電話番号・SNS・URL → 伏字（Q&A spec §6・チャット・レビューと共通ロジック）。
- **モデレーション**：管理画面「トラブル報告」（共通）にマスク発動コメントが「要確認」で上がる → 運営が🗑非表示にする。新画面は作らない。

**④ 自己プレビュー**
- レストラン本人が**プロフィールから「農家に見えているあなたのページ」をread-only**で見られる（自分の評判の把握→振る舞いの自己修正）。
- 上部に案内バナー「**👁 これは農家から見えるあなたのページです（編集はプロフィールから）**」。
- **評者（農家）名は伏せず非匿名**（農家が見るのと同一）。編集は本ページでなくプロフィール側。

**⑤ アクセス経路**
- MVPは**承認待ち詳細からの1経路**（＋本人はプロフィールから自己プレビュー）。
