# FISHLINK カート / 注文確認画面 実装仕様

実装の正本は **`fishlink-cart-mockup.html`**（構造・レイアウト・コンポーネント配置はそこを参照）。
本書はHTMLから読み取れない情報＝**データモデル・金額計算ロジック・状態分岐・バリデーション・遷移・ドメイン制約**を補う。

> ※既存実装は概ね良好（複数農家集約・配送料固定・価格の単一正・クーポン併用不可・空カート・削除確認・数量直接入力・時間帯スロットは実装済）。本書＋モックとの差分は §11 にまとめた。

### 実装方針の優先順位（他spec共通）
- **構成・データ・計算・状態・遷移** → 本書とモックHTMLの通りに実装する。
- **見た目（色・タイポ・余白・アイコン絵柄）** → 既存DSに準拠。モックの色値・アイコンは役割の参考値。

## 1. 概要

レストランが選んだ魚を**農家ごとにまとめて**確認し、数量・内臓処理・納品日時を決めて発注する画面。FISHLINKの核＝**供給不安定ゆえ複数農家から仕入れる**を支える集約画面。

- 流れ：**カート →（確認画面へ）→ 注文確認 →（注文を確定する）→ 農家ごとに独立した注文を作成**。
- 価格は**内臓処理込みの単一値**を全画面で共有（商品詳細 spec §7 と同じ）。
- ドメイン制約は Home spec §1 と共通（KHR・head/kg・GAqP・養殖のみ）。

- カートの割引行・合計（青）等の見た目は据え置き。支払い画面の金額順＝合計（クーポン適用後）→ウォレット充当→お支払い額（KHQR）。

## 2. 画面構成

### カート画面
ヘッダー（戻る＋ロゴ＋言語）→ タイトル「カート」→ **農家ブロック（複数）** → 固定フッター（合計＋「確認画面へ」青）。
- **農家ブロック**＝農家名／配送距離／商品行（複数）／納品日・時間帯／金額明細（魚代・キャンペーン割引・配送料・小計）。
- **商品行**＝写真・魚名・サイズ・単価／ステッパー＋数量／内臓処理トグル／削除／行小計。
  - **内臓処理「しない」出品（`gutFeeKhrPerKg=null`・出品フォームの する/しないトグル）＝内臓処理トグルを出さない**。単価に「（丸魚のみ）」＋注記「丸魚のみ（この出品は内臓処理を選べません）」。買い手はあり/なしを選べない（丸魚のみ）。mock＝`fishlink-cart-mockup.html` 農家B・パンガシウス。posting-create-spec の内臓処理トグルと対（波及＝商品詳細/投稿一覧/カート）。
- 空のときは §5 の空カート表示。

### 注文確認画面
ヘッダー → タイトル「カート」→ 注文確認の注記ボックス → **農家ブロック（read-only サマリ）** → **クーポン（ウォレットから1タップ・使えるクーポンがある時だけ表示）** → 固定フッター（クーポン割引行＋合計＋「注文を確定する」青）。
- read-only：商品は「魚名 ◯kg（内臓処理あり/なし）＋金額」。**内臓処理は全項目に明示**（§8）。
- 修正は左上「←」でカートへ戻る。

## 3. データモデル

```js
cart = {
  farmerGroups: [ farmerGroup ],
  coupon: { code, discountKhr } | null,   // 適用中クーポン（注文レベル）
}

farmerGroup = {
  farmerId, farmerName,
  distanceKm,                 // 配送距離（農家↔レストラン）
  delivery: { deliveryFeePer10km, freeDeliveryKm, leadTimeHours },  // 農家単位の配送設定（既定 leadTimeHours は運営値8）
  items: [ cartItem ],
  deliveryDate,               // 納品日（農家ごと）
  deliverySlot,               // 時間帯（農家ごと。1時間刻み 6:00–18:00）
}

cartItem = {
  listingId, fishName, sizeHeadPerKg, photo,
  unitPriceInclKhr,    // 内臓処理込みの単価（=単一の正・100リエル単位。例 7,800＝切り捨て(販売価格×(1+手数料),100)）
  gutRemoveKhr,        // 内臓処理を外したときの減額（=500。買い手表記・100リエル単位。500×手数料率→100切り捨て）
  gutIncluded,         // true=あり（標準）/ false=なし（外す）
  qtyKg,               // 整数kg
  minOrderKg,          // 最低注文量（null/0 なら制限なし）
  stockKg,             // 在庫（閲覧時点。ソフト上限）
}
```

## 4. 金額計算ロジック（最重要）

**単価の扱い（商品詳細 spec §7 と統一）**
- `unitPriceInclKhr` は**手数料込み・内臓処理込み**の単一の正。
- 行の実効単価 = `gutIncluded ? unitPriceInclKhr : (unitPriceInclKhr − gutRemoveKhr)`。
- 行小計 = 実効単価 × `qtyKg`。
- **金額は100リエル単位（payment-spec §4.2・2026-07-05確定＝A案）**：`unitPriceInclKhr` は**既に100リエル切り捨て済み**（=切り捨て(販売価格×(1+手数料),100)）、`gutRemoveKhr` も100リエル（500）。よって**行小計＝実効単価×整数kg が自動で100単位**になり、**単価×数量＝行小計が一致**する。**「合計だけ丸める」方式は不可**（単価×数量とズレて 15,750≠15,700 の不整合になる＝2026-07-05のカート実機で発覚）。キャンペーン割引も100リエル切り捨て（下記2）。

**農家ブロックの小計**
1. **魚代** = その農家の全行小計の合計（明細ラベルは「魚代」のみ。"サービス料込み" は出さない＝手数料は内包し項目で見せない）。
2. **キャンペーン割引**（`campaignEnabled && クーポン未適用` のとき）= **魚代から `campaignDiscountPct`%**（運営値・例2.5%）。
   - 割引は**魚代ベース**（配送料は対象外）。文言は「**魚代から{%}%割引**」（「お会計から」は使わない）。
   - **計算基準＝買い手の「魚代」（手数料込み・内臓処理込みの表示価格の合計）× {%}（確定 2026-06-19・A案）**。例：6,300/kg × 5kg ＝ 魚代31,500 → 2.5% ＝ 787.5 → **100リエル切り捨て 700**（小計を100単位に保つため割引も100切り捨て・§4.2。旧「≈788」＝1リエル丸めは廃止）。**農家価格（手数料前）ベースにしない**（30,000×2.5%＝750は不可）。理由＝買い手画面は手数料を一切見せない設計なので、割引も**買い手が見える魚代から検算できる値**にする（文言「2.5%」と数字を一致させ信頼を守る）。農家価格ベースの750は買い手の魚代の2.38%になり文言とズレる。**手数料への二重課金ぶん（≒38）はプラットフォームの販促費として許容**（誰の画面にも不整合は出ない＝買い手は魚代×2.5%だけ見る）。※農家側・運営側の内部明細は「実手数料の削減」ロジックでよい（買い手の魚代×2.5%とは別物・買い手画面とは同時に表示されない）。
   - 表示色はキャンペーン＝amber系（`#B36A00`/`#FFF7E6`）。**danger赤は使わない**。
3. **配送料** = §4-配送 を参照。**農家ごとに1回**だけ。
4. **この農家への小計** = 魚代 − キャンペーン割引 + 配送料。

**配送料（A案・農家ごと1回の固定額）**
- `distanceKm ≤ freeDeliveryKm` → **0円＝「送料無料」（緑太字）**。
- それ以外 → `deliveryFeePer10km × (distanceKm / 10)`（0kmから比例）。
- **数量・品数に依存しない固定額**（同農家から何品・何kg買っても1回分）。明細に注記「**配達1回分の固定額です。注文量を増やしても変わりません**」。
- `distanceKm > deliverableRadiusKm` の出品はそもそもカートに入らない（list-spec §7 で非表示）。

**クーポン（注文レベル・キャンペーンと併用不可）**
- **UI＝自動適用（確定 2026-06-22・手動選択なし／旧「1タップ適用」を更新）**：コードの手入力は**撤廃**。**使えるクーポンを持っている時だけ**「使えるクーポン」欄を出し、**期限が近い（古い）順に自動で1枚適用**（適用中は「✓ 適用中」）／「外す」で「今回は使わない（次回に取っておく）」も可。**1注文1枚（併用なし）**。**手動で1枚選ぶUIは作らない**（紹介クーポンは全部同額＝選ぶ意味がない＋期限切れでムダにしない）。使えるクーポンが無ければ欄ごと非表示。data model の `coupon.code` は適用中クーポンの内部識別子として保持。詳細は `fishlink-referral-spec.md` §9 を正とする。
- クーポン適用中は**キャンペーン割引を無効化**（クーポン優先）。注記「※ クーポン使用中のため、キャンペーン割引は無効になります」を出す。解除で戻せる。
- クーポン割引は**注文の合計**に対して適用（フッターに「紹介クーポン割引（コード）−◯」行）。
- 二重割引（キャンペーン＋クーポン）は**当面しない**（採算・複雑性回避）。

**合計** = 全農家ブロックの小計の総和 −（適用中クーポン割引）。

## 5. 状態分岐・バリデーション

| 対象 | ルール |
|------|--------|
| **数量** | **整数kgのみ**。ステッパー（−/＋）＋**直接入力可**（大口対応）。直接入力にも下記min/maxを適用。**入力ボックスは3〜4桁が収まる幅にする**（B2Bは100kg・200kg等が普通。狭いと「100」が「10」に切れて誤認する＝実装バグ。最低「9999」が見える幅） |
| **最低注文量（C案）** | ステッパー**下限＝`minOrderKg`に固定**（未満にできない）＋ステッパー下に「最低注文量 ◯kg」明示。カート投入の初期数量＝`minOrderKg`（未設定は1kg）。**「確認画面へ」時にも最終検証**（他経路の下限割れを止める保険） |
| **在庫上限** | 「＋」「直接入力」で `stockKg` を超えられない。超過時は在庫数に丸め「在庫◯kgまで」表示。※閲覧時点の**ソフト目安**（リアルタイムロックではない） |
| **在庫変動・無効化（軽量ガード）** | カートで無効化するのは**「削除済み」「売切れ(在庫0)」の2つだけ**＝「現在購入できません」＋チェックアウトから除外。**価格は最新表示**。微妙な在庫不足は**農家承認（辞退）に委ねる**（§8）。配達範囲外はそもそも非表示なので対象外 |
| **削除** | **自前の中央モーダルで確認（確定 2026-06-19）**：`confirm()` のブラウザ標準ダイアログ（ドメイン名＋OK紫）は廃止 → 自前モーダル「この商品をカートから削除しますか？」＋〔キャンセル（ニュートラル）〕〔削除する（赤/danger #E5453F）〕。中央配置（確認系＝中央モーダル・投稿一覧の削除と統一）。農家ブロックが空になったらブロックごと消す。全部空→空カート |
| **空カート** | カートアイコン＋「カートに商品がありません」＋「魚を探す」（→魚一覧）。**フッターのCTAは非表示** |
| **送料無料** | `distanceKm ≤ freeDeliveryKm` のとき配送料欄を「送料無料」（緑太字）に |

## 6. 内臓処理コントロール（#11・色は中立）

- 価格は**内臓処理込みが標準**。コントロールは「**内臓処理 あり ✓**」を既定（あり）にし、**「外すと −525 KHR/kg」**をヒント表示。外すと「内臓処理 なし」＋行小計が `−525×qty` 下がる。
  - ラベルに「（標準）」は付けない（あり=デフォルト選択＋「外すと −525」ヒントで"既定であり"は十分伝わる。チップを簡潔に保つ）。
- **行レイアウト（あり/なしを一目で・誤読防止）**:
  - **単価行に内臓処理の状態を併記**＝「7,875 KHR/kg（内臓処理あり）」「5,775 KHR/kg（内臓処理なし）」（あり側だけ無表記にしない。両方明示で price からあり/なしが読める）。
  - **ヒントはトグルの直下にグループ化**（「外すと −525 KHR/kg」／なし時は「戻すと +525 KHR/kg」）。ヒントが削除ボタン等の別操作の近くに浮かないようにする。
  - **削除は商品行の右上**（魚名の横）に置き、内臓処理トグル・ヒントから離す（操作の混同を防ぐ）。
- **"+525（加算風）"の表記は使わない**（込み価格に+525を出すと上乗せに誤読されるため）。
- トグルの色は**中立グレー**（緑＝GAqP/成功 専用ルールと混線させない／青＝CTA/価格に温存）。

## 7. 配達リードタイム（時間帯スロット）

- 時間帯スロット＝**1時間刻み 6:00–18:00**。
- **最短リードタイム＝8h（運営パラメータ）**。スロット開始が `now + 8h` 以降のみ選択可（未満はグレーアウト）。
- **最長リードタイム＝14日（運営パラメータ・2026-07-05確定）**。14日より先は選択不可。理由＝生鮮＋供給不安定で先ほど確度が落ちる／CAA連動価格が古くなる／前払いが長く拘束される。**それ以上先（宴会等の大口）は運営相談ベース**（in-appの前払い自己予約は14日まで）。
- **承認期限（正体は farmer-order-spec §で定義・そちらを更新）＝注文＋1.5h。ただしその期限が夜間 21:00–5:00 に入る場合は翌朝5:00起点にリセット（＝6:30）**。夜間注文が農家の睡眠中に自動辞退で切れるのを防ぐ（買い手には「農家の確認は朝になります」で納得）。
- **スロットは「承認期限 ＋ 農家準備4h」以降のみ選択可（2026-07-05確定・最重要）**。これ1本で「**前日など早めの注文は早朝配達OK／夜間注文は農家の朝準備が間に合わず早朝不可（最速9〜11時）**」が自動で出る（固定の早朝フロアや夜専用ガードは不要）。準備4h＝池揚げ〜処理〜配達（〜30km）の運営パラメータ。将来は**距離連動**（近い農家は短く）。
- **承認されなければ自動辞退＋ウォレット返金**（backstop・§8/payment-spec §4.5）。＝無理なスロットを選んでもお金は必ず戻る。
- 結果：**「早く頼めば早く届く／夜に頼むと朝イチは無理／早すぎ・遠すぎは選べない」**が単一の仕組みから自然に出る。
- **納品日の表示は曜日付きで分かりやすく**：「2026-06-04」（ISO）ではなく **「6/4（木）」**形式（曜日を入れる・年は省略可）。レストランは曜日で動くため。カート・注文確認・Homeの進行中の注文（ステッパーのETA）で**表記統一**。

## 8. 注文確認 → 注文確定（農家ごと独立注文）

- 注文確認は read-only サマリ。各品は **魚名＋サイズ（head/kg）＋数量（kg）＋内臓処理あり/なし＋金額** を表示。
  - **サイズ（head/kg）を必ず出す**（サイズ違いは別商品＝価格も違うため、最終確認で必須）。
  - **内臓処理は全項目に「内臓処理あり/なし」を明示**（無表記＝なし、にしない。発注前の最後の砦で取り違えを防ぐ）。
  - **納品日・時間帯は強調表示**（農家ごとに、アイコン付き・日時を太字の目立つ行/ボックスで）。確認画面では配達日時が重要な確認項目のため、小さなグレー注記にしない。
    - **「納品」ラベルを付ける（確定 2026-06-19）**：日付だけ（「📅 6/21（日）6:00–7:00」）だと何の日付か不明 → **「📅 納品 6/21（日） ・ 🕐 6:00–7:00」**（「納品」ラベル＋時間に🕐）。カートの「納品日」フィールドラベルと用語統一。モックは元からこの形＝実装が「納品」と🕐を落としている。
- **明細のキャンペーン割引行は、カートと注文確認で同じ見た目に統一**（不揃い禁止）。**軽い強調＝🏷アイコン＋オレンジ文字（背景はうっすらamberまで）**に両画面で揃える（カートの重い全幅ボックスは軽くし、注文確認の素のオレンジ文字には🏷を足す）。
- 「注文を確定する」で**農家ごとに独立した注文**を作成（注文番号 FL-xxx・ステータス個別）。
- **承認/辞退は農家ごと**：farmer5 は承認・Ryosuke4 は辞退、のように**片方だけ成立**できる（Home spec の辞退フロー「他の農家を探す」と整合）。
- 承認待ち→農家確認中→出荷→配達中→完了 / 辞退（在庫切れ等）。在庫の最終真実は**農家承認**（カートは軽量ガードのみ＝§5）。
