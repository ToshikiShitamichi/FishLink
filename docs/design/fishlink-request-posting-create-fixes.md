## 修正①：配送サマリーの色＝緑 → 青系に

配送サマリー（「30kmまで配達・5km以内は送料無料 / 10kmあたり5,000 KHR」＋変更）の**ボックス全体が緑**になっています。**我々のモックは青系（薄い青 `#EAF2FB`）**です。

| 箇所 | 現状 | 修正後 |
|---|---|---|
| 配送サマリーのボックス | 緑（全体）| **青系/ニュートラル**（薄い青 `#EAF2FB`）。緑はGAqP・送料無料専用なので、配送ボックス全体を緑にしない |

- 「5km以内は**送料無料**」の"送料無料"の文字だけ緑にするのはOK（緑＝送料無料の用法）。ボックス全体の緑をやめる。

## 修正②：魚単価のプレースホルダ「例：5,000」→ 汎用に

魚単価の入力欄のプレースホルダが「**例：5,000**」と**特定の数字**になっています。すぐ上に **CAA参考販売価格（7,000–8,000等）** が出るので、5,000を例示すると逆に**低くアンカーしてしまう**（＝農家の値付けを引っ張る）。

| 箇所 | 現状 | 修正後 |
|---|---|---|
| 魚単価のプレースホルダ | 「例：5,000」 | 「**例：金額を入力**」等の**汎用**（特定の数字を勧めない）。プリフィル値ではなくプレースホルダのまま |

- **「10kmあたりの送料」の「例：5,000」はそのままでOK**（送料は5,000が妥当な目安・ヘルプ文も5,000で説明）。今回直すのは**魚単価だけ**。
- モックも「例：金額を入力」に更新済み（魚単価のみ）。

## 修正③：必須フィールドに「必須」バッジを表示

現状、任意フィールド（最低注文量・商品の説明）には「任意」バッジが出ていますが、**必須フィールドに「必須」バッジが出ていません**（魚種／サイズ／在庫数量／写真／魚単価／配送可能距離／10kmあたり送料）。低リテラシーの農家が「どれを必ず入れるか」迷わないよう、**必須フィールドにも「必須」バッジを表示**してください（モックは必須・任意の両方マーク済み）。

## 修正④：最低注文量は新規投稿で空（プリフィルしない）

最低注文量（任意）に「**5**」がプリフィルされています。最低注文量は**任意＝空＝最低なし**なので、**新規投稿では空（プレースホルダ「指定なし」）**にしてください。プリフィルすると、農家が気づかず最低5kg制限を課してしまう（誤送信）。※編集画面で既存値が5なら正常。

## 修正⑤：数値入力の上下スピナー（＋1刻み）を消す

魚単価・内臓処理などの数値欄に**ブラウザ標準の上下スピナー（±矢印）**が出ており、**＋1刻み**です。KHRの価格は数千〜なので＋1刻みは無価値＋スクロールで誤って値が変わる事故も起きます。**スピナーを非表示にし、数値キーボードで入力**（`inputmode="numeric"` 等）にしてください。**対象＝全ての数値欄**（魚単価／内臓処理／在庫／配送可能距離／送料単価／送料無料距離／最低注文量）。モックはスピナー無し。

## 修正⑥：送料無料距離のヘルプ文＝緑 → グレーに

配送設定の「送料無料距離」のヘルプ文（「この距離内は送料無料。空欄なら…」）が**文全体 緑**になっています。①の配送サマリー緑と同じ"緑の使いすぎ"で、他のヘルプ文（グレー）とも不揃い。**ヘルプ文はニュートラルグレー**に（"送料無料"の語だけ緑にするのはOK）。

## 修正⑦：初回の配送3欄をプリフィルしない（空＋必須）

初回（配送未設定）の配送3欄に値（配送可能距離 **30**／10kmあたり送料 **5,000**／送料無料距離 **0**）が**プリフィル**されています。デフォルト値の決定どおり、**配送可能距離・10kmあたり送料＝空＋必須（プレースホルダ「例：30」「例：5,000」）／送料無料距離＝空（任意・プレースホルダ「例：5」）**にしてください。理由：配送可能距離の30放置は「届けられない遠方に表示→キャンセル・低評価」の最有害／送料5,000はアンカリング誤誘導。モックは空（プレースホルダ）。

## 修正⑧：送料無料距離のヘルプ文言を「おすすめ」版に

⑥の色（緑→グレー）に合わせ、文言も更新してください：
- 現状：「この距離内は送料無料。空欄なら全配達で送料がかかります。**無料圏を作りたいときだけ入力してください。**」
- 修正後：「この距離内は送料無料になります。**近場は送料無料にするのがおすすめです（例：5km）。**空欄なら全配達で送料がかかります。」（任意のまま・近場無料を軽く促す。プレースホルダ「例：5」）

## 修正⑨：セクションを白いカードで区切る

基本情報／販売価格／配送／取引条件／商品の説明 を、それぞれ**白いカード（角丸・余白・薄い影）**で囲み、視覚的にブロック分けしてください。長いフォームを「1かたまりずつ」処理できるようにし、低リテラシーの農家の認知負荷を下げる狙いです。モック反映済み。

## 修正⑩：投稿成功時に「完了画面（中央）」を表示

投稿が完了したあと、**投稿一覧（ボトムナビ「投稿一覧」）に着地**しますが、**「投稿できた」というフィードバックがありません**。低リテラシーの農家は「ちゃんと投稿できたのか」が不安になり、二重投稿の原因にもなります。投稿成功時に、**画面中央に「完了画面」（大きな✓＋「投稿しました」＋「投稿一覧を見る」ボタン）** を表示してください。

※ 当初「隅で一瞬出るトースト」を検討しましたが、**消えるのが速く見逃しやすい＝完了感が弱い**ため、**能動的にタップして進む「完了画面」**に変更しました（見逃しゼロ・達成感が出る）。

| 項目 | 仕様 |
|---|---|
| 形 | 画面中央の**カード**（白・角丸）＋**背景は暗転**（リストを薄暗く）。一拍止めて「完了」を確実に認識させる |
| 中身 | ① 大きな**緑の✓**（円形バッジ・緑 `#19A05A`）／② タイトル「**投稿しました**」／③ サブ「投稿一覧に表示されます」（グレー）／④ **「投稿一覧を見る」ボタン（青 `#0B5FB0`・主CTA）** |
| 進み方 | **「投稿一覧を見る」をタップ → 投稿一覧へ**（自動では閉じない＝能動確認）。一覧の先頭に今投稿した出品が並ぶ |
| タイミング | 投稿成功（サーバー保存完了）時に表示。**自動フェードアウトはしない**（タップで進む） |

- モックは投稿一覧側（`fishlink-posting-list-mockup.html`）に `.success-overlay` / `.success-card` として反映済み。
- 編集の保存時は完了画面まで出さず、軽い確認（保存しました）で構いません。完了画面は**新規投稿**の達成感のため。

### ⑩-2（併せて）：二重投稿の防止＝投稿ボタンの送信中無効化
完了画面は「できた」の事後フィードバックです。**二重投稿そのものは、投稿ボタンを押した瞬間にボタンを無効化＋ローディング表示**して防いでください（送信完了＝完了画面表示まで再タップ不可）。完了画面が暗転で前面に出ている間も元のフォームには戻れないため、二重に防げます。
