# FISHLINK 配送管理チャット 実装仕様

実装の正本は **`fishlink-chat-mockup.html`**（左＝レストラン側／右＝農家側。構造・レイアウト・配置はそこを参照）。
本書はHTMLから読み取れない情報＝**データモデル・メッセージ種別・ステータス連携・通知・連絡先マスク・通話・ライフサイクル・両者の出し分け**を補う。

チャットは**両アプリ横断**の機能。**レストラン側は新規実装**、**農家側は既存 `fishlink-farmer-order-mockup.html` ⑤ の見直し**（差分は §11）。農家側の注文確認フロー全体は `fishlink-farmer-order-spec.md`、公開Q&Aは `fishlink-qa-spec.md` を参照（連絡先マスクは共通）。

> 配送管理チャットの主要論点はレビュー済（A レストラン側の形／B システム行／C 定型文見送り／D 日付区切り・押下制御／E 音声UI／通知／連絡先マスク／📞通話／ライフサイクル／配送完了ダイアログ／既読見送り）。差分は §11。

### 実装方針の優先順位（他spec共通）
- 構成・データ・メッセージ種別・状態連携・遷移 → 本書とモックの通り。
- 見た目（色・タイポ・アイコン絵柄）→ 既存DSに準拠。**CTAはアプリ全体で青 `#0B5FB0`**（オレンジ廃止）。**ステータスのシステム行アイコンは絵文字でなくモノクロ線画**（§4）。

---

## 1. 概要

承認後（準備中〜）に開く、農家↔レストランの**リアルタイムチャット**。配達の自己解決（遅れ・調整・場所案内）をチャットで完結させ、解決しない問題だけ「問題報告」へエスカレーションする（問題報告はチャットに置かない＝§10）。

- メッセージは **テキスト / 音声 / 画像**。**音声メッセージ**が低リテラシー層の主手段（→定型文/クイック返信は入れない＝§13 経緯）。
- **配送ステータス送信ボタンは農家側のみ**（レストラン側は受け取るだけ）。
- 送られたステータスは**会話とは別の「システム行」**として両者のチャットに出る＋**買い手の Home 5段ステッパーに反映**（§6）。

## 2. 画面構成・両者の出し分け（A）

| 要素 | レストラン側（買い手） | 農家側（売り手） |
|------|------------------------|------------------|
| ヘッダー：相手名 | 農家名 | レストラン名 |
| ヘッダー：ステータス表記 | **買い手5段の言葉**（承認済み→出荷準備→配達中→完了） | 農家の言葉でも可（注文ステータスは単一の真実・表示文言だけ役割別） |
| ヘッダー：注文サマリ | 魚種・数量・納品日（曜日付き）・FL番号 | 同左 |
| ヘッダー：位置 | **📍距離のみ（地図リンク無し）** | **📍地区名＋「地図を見る」**（配達先を開く） |
| ヘッダー：📞通話 | あり（§9） | あり（§9） |
| 本文：会話バブル／音声／画像／システム行／日付区切り | あり（§4・共通） | あり（§4・共通） |
| フッター：配送ステータス送信ボタン | **無し** | **あり**（§5） |
| フッター：メッセージ入力（📷🎤＋テキスト＋送信） | あり | あり |
| フッター：配送完了ボタン | **無し** | **あり**（§10） |

- **地図リンクが農家側のみ**な理由：配達は農家→レストランの向き＝農家がレストランの配達先を開く。レストランが農家の位置を開く必要は薄い（ライブ追跡もMVP無し）。距離📍だけ参考表示。
- 入口：レストラン＝注文状況カードの「**配送状況を確認**」／農家＝注文確認カード・⑤の「**配送管理へ**」。

## 3. データモデル

```
chatThread {
  id,
  orderId,            // 農家ごと独立注文（farmer-order spec §3）
  farmerId, restaurantId,
  orderStatus,        // 注文ステータスを参照（単一の真実）
  openedAt,           // 承認時に生成・オープン
  closedForUserAt,    // 配送完了時にユーザーUIを閉じた時刻（履歴はサーバー保持）
}

message {
  id, threadId,
  sender,             // "farmer" | "restaurant" | "system"
  type,               // "text" | "voice" | "image" | "status"
  body,               // text本文（マスク後は伏字。原文は別保持）
  rawBody,            // マスク前原文（運営閲覧用・非公開）
  masked,             // bool（連絡先検知で伏字化したか）
  voiceDurationSec,   // type=voice
  imageUrl,           // type=image
  statusKind,         // type=status: "approved"|"prepare_start"|"ship_start"|"delay"|"arrived"|"completed"
  createdAt,
}
```

- `statusKind` のうち `approved`（承認）/`completed`（配送完了）はシステム発行、`prepare_start`/`ship_start`/`delay`/`arrived` は農家のステータス送信ボタン由来。

## 4. メッセージ種別と表示

**会話バブル（text / voice / image）**
- 左右で発言者を区別：**自分＝右（青 `#DCE9F8`）／相手＝左（白・枠 `#EEF1F4`）**。
- 各バブルは**時刻のみ**（日付はバブルに出さない＝日付区切りに集約）。
- **音声（E）**：`▶`（再生・青丸）＋簡易波形＋秒数（例 `0:07`）。録音/再生の挙動は実装側。
- **画像**：サムネ表示。「画像が送信されました」等のラベルは**付けない**（画像だけで自明）。

**システム行（出来事・B）＝会話バブルと別スタイル**
- **中央寄せ・淡色背景・ラインアイコン付き**。承認/準備/配達/到着/遅延/配送完了などの「出来事」を会話と明確に区別する。
- **アイコンはモノクロ線画（SVG・`currentColor`で行の色を継ぐ）。絵文字は使わない**（端末/フォント差で見た目が割れる・青系DSに馴染まない）。
  - 承認＝チェック／準備＝箱／配達中＝トラック／到着＝ピン／遅延＝時計（目安）。
- **色：通常＝青（`#0B5FB0`系・背景 `#EAF2FB`）／遅延（delay）＝amber（`#B36A00`・背景 `#FFF4E2`）**。遅延は割り込みなので暖色で目立たせる。
- 同じシステム行が**両者のチャットに出る**（農家がボタンを押す→レストランにも同じ行が流れる）。

**日付区切り（D）**
- 日付が変わる位置に**中央の区切り線**（例「6月4日（水）」）。曜日付き（カート/注文確認/Homeと統一）。

## 5. 配送ステータス送信（農家側のみ）

- ボタン：**準備開始 / 配送開始 / 少し遅れそうです / 到着**。押すとチャットに**システム行**として流れる（§4）＋**買い手5段ステッパーに反映**（§6）。
- **押下制御＝緩い誘導（B案）**：進行順（準備開始→配送開始→到着）で**次に押すべきボタンを強調**（青枠）。ただし**押下自体は自由（ロックしない）**＝鮮魚配達は段取りが前後する。
  - 済みのボタンは淡色＋取り消し線で「送信済み」を示す（参考表現）。
  - **「少し遅れそうです」は進行と無関係にいつでも押せる割り込み**（amber）。
  - **重複押し許容**（買い手には最新が反映）。
- レストラン側フッターにはこのブロックを**出さない**（A）。

## 6. ステータス↔買い手5段マッピング（farmer-order spec §5 と同一・再掲）

| 農家の状態・操作 | 買い手の表示（Home 5段） |
|---|---|
| 承認待ち | 農家確認中 |
| 承認 → 準備中／準備開始 | 出荷（準備） |
| 配送開始 | 配達中 |
| 少し遅れそうです | 配達中（＋「遅延の可能性」表示） |
| **到着** | **配達中のまま**＋ステッパー下のメッセージ行「📍到着しました・お受け取りください」＋プッシュ通知（ステッパーに到着ノードは作らない／"まもなく"は使わない） |
| 配送完了 | **完了** ← レビュー解禁 |

## 7. 通知

- **新着→相手にプッシュ通知**（農家送信→レストラン／レストラン送信→農家）。**メッセージ・配送ステータス更新・音声・画像すべて**対象。
- **集約（連投まとめ）はしない**（MVPは都度通知）。
- **既読表示（read receipt）はMVP見送り**（新着はプッシュで届く・既読管理は実装増）。
- やることリスト（✓✓）への積み込み（未読チャット等）は**横断で別途整理**（宿題）。

---

## 8. 連絡先マスク（中抜き対策）

- **本文の電話番号・SNS・URLを検知→伏字＋警告**（公開Q&Aと共通＝`fishlink-qa-spec.md` §6）。**チャット・公開Q&A 両方に適用**。
- 検知メッセージは `masked=true`・伏字を表示し、`rawBody`（原文）は**運営閲覧用に保持**。投稿時にも警告。
- 背景：**中抜き（プラットフォーム外の直接取引）対策**。最強の防御は**決済をFISHLINKが握ること**（レストラン→FishLink→農家へ送金＝直接支払いではない）。マスクは補助的抑止（完全には防げないが"公然の場"での連絡先交換を塞ぐ）。CAA価格表の番号は**仲買人**のもので農家番号は非公開＝中抜き防止の重要度は高い。

## 9. 📞通話ボタン（ヘッダー）

- **MVP＝実番号・限定露出（2026-06-11 確定）**：**📞は「注文がアクティブな間（承認後〜完了まで）有効・完了後は消す」**（"配達中"限定にしない＝手動ステータス未設定で電話できない問題を回避・露出を進行中注文に限定）。**実番号を恒久的に表示/発信するのは不可**（進行中だけの限定露出）。
- **マスク通話はMVPでは入れない（2026-06-11 判断）**：下道さん調査＝**PWAは「着信通知」は飛ばせるが「着信状態（鳴り続ける）」にできない＝通知ベース・iOS特に弱い→取りこぼし得る＝実質つながらない懸念**。Agoraでアプリ内マスク通話の実装自体は可能（音声・無料枠1万分/月・チャンネル=orderIdで番号不使用＝マスク構造的担保）だが、**「つながらないマスク通話」より「確実につながる実番号」が配達では実用的**＋**配達は対面で会う＝番号隠しの中抜き効果は配達場面では小さい**＋**中抜きの本丸は決済中継（ABA）**。→ 実番号・限定露出でMVP。
- **昇格条件**：Agoraマスク通話は**“着信のネイティブ化（CallKit級・鳴り続ける）”の目処が立った時にセットで昇格**＝その時「マスク＋確実につながる」が両立。会う前（Q&A等）の**テキスト連絡先マスクは別途維持**（§8）。

## 10. ライフサイクル・配送完了・遷移

**開く / 閉じる**
- **承認後（準備中〜）に開く。** 承認待ち中はチャット無し（事前の疑問は公開Q&A）。
- **配送完了後はユーザーにはチャットUIを閉じる。** 完了後カードは「配送状況を確認／配送管理へ」→「**支払い状況を見る（レストラン）／送金状況を見る（農家）**」に切替。
- **チャット履歴はサーバー保持（削除しない）。運営が問題報告対応時に閲覧できること＝必須**（紛争の証拠＝配達写真/到着時刻/約束事）。ユーザー向け完了後 read-only 表示は後フェーズ（MVP省略）。

**配送完了（農家のみ）**
- 「配送完了」＝農家の「配達した」申告。**確認ダイアログ＝自前モーダル**（誤タップ防止・承認と統一）。
  - 文言：「**配達完了にしますか？ レストランにお支払い（受け取り時のお支払い）のお知らせが届きます。**」（旧「受取確認を依頼するステータス」は受取確認ボタンが無く誤解→使わない）。
- → **レストランは「お支払い」画面へ**（受け取り時にQR（KHQR）で支払う＝支払い＝受領確認。専用の受取確認ボタンは無し）。
- ※配送ステータス送信ボタン（準備開始/配送開始/到着）は軽い操作で**ダイアログ不要**。
- 受け取り確認＝Grab/Uber流：レストランに完了ボタンを必須にしない。農家の配送完了で進み、**問題報告が無ければ一定時間で自動確定**→完了→精算・レビュー解禁（自動確定タイミングは精算フローで詰める）。

**問題報告はチャットに置かない**
- 問題報告は**注文カード（レストラン注文状況／農家注文確認）＋支払い画面**に集約。チャットはカードから1タップで戻れる＋入口を絞り運営負荷・煩雑さを回避。

| 要素 | 遷移先 |
|------|--------|
| レストラン：注文状況カード「配送状況を確認」 | 配送管理チャット（レストラン側） |
| 農家：注文確認カード・⑤「配送管理へ」 | 配送管理チャット（農家側） |
| ヘッダー「地図を見る」（農家側のみ） | 地図（配達先） |
| 農家：「配送完了」 | 自前モーダル → 確定で完了・レストランはお支払いへ・送金待ちへ |
| 完了後カード | 支払い状況（レストラン）／送金状況（農家） |

## 12. 表示ルール（色）

- **CTA＝青塗り `#0B5FB0`**（送信／配送完了）。オレンジは使わない（キャンペーン色と混線）。
- **システム行：通常＝青／遅延＝amber**。**緑は使わない**（緑はGAqP/成功/送料無料専用）。**赤はdanger専用**（チャット内では使わない＝問題報告はチャット外）。
- 価格・金額は青（全アプリ統一）。
