# FISHLINK 運営チャット（サポート）再設計 実装仕様

実装の正本は **`fishlink-support-chat-mockup.html`**（構造・レイアウト・配置はそこを参照）。
本書はHTMLから読み取れない情報＝**データモデル・振り分けロジック・FAQ管理・状態・通知・遷移・制約**を補う。

> 既存実装あり（マイページ「運営からのメッセージ」＝運営↔ユーザーのチャット／管理画面ユーザー詳細に返信欄）。本書は**再設計**（既存への修正＋FAQ管理を新設）。差分は §12。

### 実装方針の優先順位（他spec共通）
- 構成・データ・振り分け・状態・遷移 → 本書とモック通り。
- 見た目（色・タイポ）→ 既存DS準拠。**CTA・送信＝青／緑＝GAqP・成功専用／赤＝danger／amber＝注意**。本番デフォルト＝クメール語。

---

## 1. 概要

運営↔ユーザー（農家・レストラン）の**サポートチャット**。マイページ「運営からのメッセージ」から入る。

- **狙い＝「何でも来る窓口」にしない**：問い合わせ量が爆発すると少人数の運営がパンクするため、**カテゴリ選択を“振り分けの関所”**にして、**既存窓口＋FAQで自己解決**→それでもダメな時だけ運営チャットに通す（量を抑える）。L244（自己解決を先に促す＋解決しない物だけエスカレ）と一貫。
- **FAQは管理画面で運営が育てる**：問い合わせが来る→運営がそのQ&AをFAQに追加→次から自己解決→負担が減る（**自己改善ループ**）。
- 入口＝両ロールのマイページ「運営からのメッセージ」。

---

## 2. 画面構成

| 画面 | 内容 |
|------|------|
| ① お問い合わせ（カテゴリ選択） | 「どんなことでお困りですか？」＋カテゴリのタイル一覧（§3） |
| ② 振り分け（カテゴリ選択後） | そのカテゴリの **よくある質問（FAQ）＋関連窓口（ロール別の該当画面へ）＋「解決しない場合は運営に送る」**（§4） |
| ③ 運営チャット（ユーザー側） | **青の往復**チャット（§5） |
| ④ 管理画面 FAQ管理 | カテゴリ別Q&Aの追加/編集/公開/並べ替え/画像/言語（§7） |
| ⑤ 運営側 運営チャット | ユーザー管理→詳細の返信欄（往復・未対応/対応済・§8） |

---

## 3. カテゴリ（コード固定・ロール別出し分け）

- **コード固定の初期セット**（カテゴリは“振り分け先のロジック”を持つ＋小さく安定→コード管理。**育てるのはFAQだけ**＝管理画面で）。
- **共通6**：注文について／支払い・送金について／配送について／紹介・クーポンについて／アカウントについて／その他。
- **農家のみ +1**：出品について（レストランは出品しないので非表示）。GAqP・認証は「出品について」に含める。
- **「トラブル報告」はカテゴリに置かない**（問題報告フローと重複→削除。注文/配送/支払いの振り分け案内で「問題を報告へ」と誘導）。
- ※支払いラベルは**ロール別可**（農家＝「送金について」／買い手＝「支払いについて」）。

---

## 4. 振り分け（deflection・②）

カテゴリを選ぶと、いきなり運営に送らず：
1. **そのカテゴリの公開FAQ**を表示（§7。0件なら省略）。
2. **関連窓口ボタン＝ロール別の“該当画面”へ飛ばす**。カテゴリ選択時点では**どの注文/項目の話か分からない**ので、**特定の注文でなくリスト/ハブへ飛ばし、ユーザーがそこで該当を選ぶ**。
   | カテゴリ | 農家の飛び先 | レストランの飛び先 |
   |---|---|---|
   | 注文／配送 | 注文確認 | 注文状況 |
   | 支払い・送金 | 売上・取引／送金状況 | 注文状況（お支払い待ち）／支払い画面 |
   | 紹介・クーポン | マイページ（紹介） | マイページ（紹介） |
   | アカウント | 基本情報／パスワード回復 | 基本情報／パスワード回復 |
   | 出品 | 投稿一覧／出品 | （無し） |
   - 配送の連絡＝該当注文の**配送管理チャット**／問題＝該当注文の**問題を報告**（いずれも各注文ごと＝ロールの注文画面にある）。
3. それでも解決しなければ **「運営にメッセージを送る」→ ③運営チャット**。

---

## 5. 運営チャット（会話・③）

- **1本の連続チャット**（ユーザーごと1つ）。チケット型（件ごとスレッド）は将来。
- **青の往復**：自分＝青系バブル(右)／運営＝白バブル(左)。配送管理チャット（chat spec）の作りに寄せる。
- **カテゴリは“件の区切り”**：会話に「**— 配送について —**」のシステム行を入れて何の件か示す（トリアージにも効く）。**毎メッセージのタグは廃止**。別件は「別の件で問い合わせる」でカテゴリ選び直し→新しい区切り。
- **「ご利用にあたって」**（amber）＝返信に1〜2日／要点を簡潔に／緊急の問題は「問題を報告」から。維持。
- **画像＝複数枚添付可**（問題を複数スクショ/アングルで）。**音声は無し**（運営の負荷・トリアージしにくい・クメール音声の壁。声の受け皿は配送チャット側）。
- 日付区切り・時刻表示。

---

## 6. 通知

- **運営が返信 → ユーザーへプッシュ通知**（「運営から返信が来ました」→タップで開く）。1〜2日かかるので必須。
- 入口（マイページ）に**未読バッジ**。
- 連投集約はMVPは都度（chat specと同方針）。やることリスト（✓✓）連動は別途整理。

---

## 7. FAQ（データ＋管理・④）

**データモデル**
```js
faq = {
  id, category,            // カテゴリ（§3のコード固定値）
  question, answer,        // 言語別（下記）
  images,                  // 回答に添付（複数可）
  published,               // 公開/非公開（非公開=下書き/ネイティブチェック前）
  order,                   // 並び順（運営が手動）
  i18n,                    // { km:{question,answer}, en:{...}, ja:{...} }
  createdAt,
}
```
- **多言語**：質問・回答を**言語別に保持**（クメール/英/日）。既存の3言語切替に乗せる（§9）。
- **公開/非公開**：機械翻訳の下書き → **ネイティブチェック済みで公開**（非公開がチェック前を兼ねる）。

**管理画面 FAQ（管理画面ナビに「FAQ」を追加）**
- **カテゴリ＝ドロップダウン（選択式）**で切替（ツール画面なのでコンパクト）。
- **「編集する言語」タブ**（クメール/英/日）＝**編集するQ&Aの“中身”の言語**。**管理画面のUI言語（グローバル🌐）とは別物**（§9）。
- Q&Aリスト：**並べ替え（手動・ドラッグ）／公開・非公開トグル／編集／削除**。
- 編集：質問／回答／**画像（複数可）**／公開する。
- 後回し（MVP不要）：検索・閲覧数/役立った集計。

**ユーザー側表示**
- カテゴリ選択後（②）に**そのカテゴリの公開FAQをリスト表示**（質問タップで回答＋画像が展開）。**0件なら省略**（窓口ボタンのみ）。ユーザーの選択言語で表示。

---

## 8. 管理側 運営チャット（返信・⑤）

- 場所＝**管理画面 ユーザー管理 → ユーザー詳細**（UID/電話/拠点[地図を見る]/言語/評価/登録日＋BAN＋GAqP＋トラブル報告＋運営チャット を1画面集約＝維持）。
- **会話＝往復**：ユーザー＝白(左)／運営（自分）＝青(右)。**カテゴリは件の区切り**（毎メッセージタグ廃止）。
- **状態＝未対応/対応済 の2状態**：
  - **未対応**＝ユーザーの新着に未返信。**ユーザー一覧に未対応バッジ＋「未対応」フィルタ**で拾える。
  - 運営が返信→未対応が外れる。返信せず片付けたい時（重複/他で解決）→「**対応済にする**」で手動クリア。
  - 「対応中」は任意（複数人運営になったら追加）。
- 送信（Enter送信/Shift+Enter改行）→ §6でユーザーへpush。

---

## 9. 多言語

- **既存の3言語切替（クメール/英/日）に乗せる**（カテゴリ名・振り分け案内文・FAQ）。中身の翻訳はネイティブチェック（§7）。
- **2つの「言語」を混同しない**：
  - **🌐 グローバル＝管理者/ユーザーが見る UIの言語**。
  - **FAQの「編集する言語」タブ＝書くQ&Aの中身の言語**。分けることで「**日本語UIのままクメール語の答えを編集**」できる（非クメール話者でも全言語を管理可）。
- ※言語“切替の仕組み”は既にある。全UIのクメール語化“完成度”・地図言語同期・シェア文は別タスク（下道さん待ち②）だが、運営チャット/FAQは仕組みに乗るだけで今やってOK。

---

## 10. 色・表示ルール

- **CTA・送信＝青**（運営にメッセージを送る／送信／対応済にする）。**緑は使わない**（GAqP・成功・送料無料専用）。
- 窓口ボタン＝青枠／問題を報告＝赤枠（danger）。ご利用にあたって＝amber。
- バブル＝自分=青系／相手=白。
- **評価＝★N%（件数）**（管理側の「79.00 (19)」→「★79% (19)」・小数廃止）。
- 管理画面の言語バーなど**緑→青**。

| 項目 | 現状 | 修正後 |
