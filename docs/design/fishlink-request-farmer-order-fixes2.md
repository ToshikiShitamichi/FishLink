# 【実装依頼】売上・取引／購入・取引 の修正3点（2026-07-04）

下道さん

農家「**売上・取引**」と買い手「**購入・取引 / 注文状況**」を実機で確認して、修正が3点あります。①③は状態表示のバグ（確認中まわり）、②はラベルの誤解修正です。①は農家側、③は買い手側で、どちらも「問題報告で確認中の注文の見せ方」が揃っていない話です。

正本：モック `fishlink-farmer-order-mockup.html`（⑥売上・取引／⑧'送金状況＝確認中）・`fishlink-order-status-mockup.html`（買い手 購入・取引）／仕様書 `fishlink-farmer-order-spec.md`・`fishlink-order-status-spec.md`

## 🔴 ① 確認中（問題報告で送金保留）を「一覧」にも反映（農家・売上・取引）

**現状＝詳細だけ実装されていて、一覧が追随していない**：
- 詳細画面（送金状況）は問題報告時に「**確認中**」になっていて **OK** ✅
- でも一覧に戻ると、同じ注文の**バッジが「送金待ち」のまま** ❌
- サマリでも保留中の金額が「**送金待ち・本日中に送金予定**」に混ざっている ❌
  （例：問題報告済みの 48,963 KHR が受取額・送金待ちの両方に加算され、"本日入る額"を過大表示）

→ 運営がまだ確認中の金を、農家に「今日入ります」と表示してしまう状態です。

**直してほしいこと（お手本＝モック⑥ `fishlink-farmer-order-mockup.html` L602–629）**

| 箇所 | 直す内容 |
|---|---|
| **一覧カードのバッジ** | 問題報告済みの注文は「送金待ち」→「**確認中**」（濃オレンジ `#C2410C`）。金額表記を「**受取予定額（保留中）**」にし、1行「**⚠ 問題が報告されています（運営が確認中）**」を追加。タップで送金状況（確認中）へ。 |
| **上部サマリ** | 1行から**2行に分割**：<br>・`送金待ち ◯◯ KHR ・本日中に送金予定`<br>・`確認中 ◯◯ KHR ・問題報告あり（保留）`（濃オレンジ）<br>→ **保留分（確認中）を送金待ち＝"本日入る額"に混ぜない**。確認中が0なら確認中の行は非表示。 |

- データ：送金待ち＝`awaiting_payout` は維持、確認中＝`payout_holding` を別状態に（仕様書 §配送完了後・送金保留 / §7B に明記済み）。
- 買い手側（購入・取引）にはこの「確認中」は不要です（買い手は前払い済み＝送金保留は農家側の概念）。

## 🟠 ② 「平均単価」→「平均取引額」に表記変更（農家・買い手 両方）

サマリ3つ目の「**平均単価**」は、実際は **受取額（購入額）÷取引数＝1注文あたりの平均額**で、kgあたり単価ではありません。農家はCAA価格表の世界で「単価＝KHR/kg」で考えるため誤解を生みます（受取額は送料・手数料込みの手取りなので ÷kg でもズレます）。

- **ラベルだけ「平均取引額」に変更**（数字の計算は変えない）。
- 対象＝**農家 売上・取引**（`fishlink-farmer-order-mockup.html`）＋**買い手 購入・取引**（`fishlink-order-status-mockup.html`）の両方。
- ※実機の実ラベルは 農家＝「平均単価」／買い手＝「**平均購入単価**」。どちらも「**平均取引額**」に統一してください。

## 🔴 ③ 問題報告中の注文が「進行中」と「履歴」に二重表示されている（買い手）

**現状＝同じ注文が2か所に出て、状態も矛盾**（例：FL-260704-001 レッドティラピア）：
- **注文状況・進行中**：お届け済み＋「報告済み・対応中」＝**まだ終わってない扱い**
- **購入・取引・履歴**：「**取引完了**」＝**終わった扱い** ❌

問題を報告したのは買い手本人。返金の可能性がある未解決の注文なのに、履歴で「取引完了」と出て混乱します。

**直してほしいこと**

| 箇所 | 直す内容 |
|---|---|
| **購入・取引（履歴）** | 問題報告中（確認中）の注文は**履歴に出さない**。解決してから履歴へ（問題なし→取引完了／返金→ウォレット即返金）。 |
| **注文状況・進行中カード** | お届け済みで問題を報告したら、**バッジ「お届け済み」→「確認中」（濃オレンジ #C2410C）**。一言を「✓配達完了。問題がなければ数時間後に自動で取引完了します」→「**🚩 問題を報告しました。運営が確認しています**」に差し替え（報告後は自動完了しないので前の一言は矛盾）。「報告済み・対応中」チップはこの一言に統合。**カードタップで購入詳細（確認中）へ。** |

- **購入詳細の「確認中（問題が報告されています）」画面は既に実装済みでOK** ✅（新規不要・その状態を進行中カードと履歴の扱いに繋げるだけ）。
- お手本＝モック `fishlink-order-status-mockup.html`（進行中に「確認中」カードを追加済み）／仕様書 `fishlink-order-status-spec.md`（進行中の状態一覧・購入詳細・取引リストに明記済み）。
- 農家側①の確認中と対になる買い手版です。
