# FISHLINK 魚一覧画面 実装仕様

実装の正本は **`fishlink-list-mockup.html`**（構造・レイアウト・絞り込みUI・ソートUIはそこを参照）。
本書はHTMLから読み取れない情報＝**データモデル・絞り込み/ソート仕様・状態・遷移**を補う。

### 実装方針の優先順位（Home specと同じ）
- **構成・データ・絞り込み/ソート・遷移** → 本書とモックHTMLの通りに実装する。
- **見た目のスタイル**（色・余白・角丸・影） → **既存アプリのデザインシステムに準拠**。モックの色値は役割を示す参考値。

---

## 1. 概要

「魚から探す」やボトムナビ「魚一覧」から来る、商品（＝農家が出している魚）の一覧。
レストランが魚種・地域・価格・距離で絞り、ソートして仕入れ先を選ぶ画面。
ドメイン制約はHome spec §1と共通（養殖のみ / KHR / head/kg / GAqP / 供給不安定で複数農家から仕入れ）。

---

## 2. 画面構成

ヘッダー（sticky）→ ページタイトル「魚一覧」→ ソート＋絞り込みバー → 件数表示 → カードグリッド → ボトムナビ（fixed）。

- ヘッダー: ロゴ / 🌐言語切替 / 🔔通知(未読バッジ) / 👤プロフィール ＋ 検索バー「魚種・農家を探す」
- ボトムナビ: ホーム / 魚一覧(active) / カート / 注文状況
- 絞り込みはボトムシート（下からせり上がる）。適用中は「絞り込み」ボタンに適用数バッジ。
- **キャンペーンバナーは表示しない**（Home画面専用。仕様・理由は Home spec §2 参照）。魚一覧は比較・絞り込みの作業画面のためバナーは邪魔になり、割引はお会計（明細）に必ず再表示される（Home＝認知・明細＝回収）。

## 3. データモデル

```js
// 商品カード（Home spec の product と同形）
product = {
  id,
  fishName,           // "ヒレナマズ"
  sizeHeadPerKg,      // "10 head/kg"（文字列。範囲もある "6-8 head/kg"）
  priceKhrPerKg,      // 6300（カード表示単価, KHR/kg。22px/800/プライマリ青#0B5FB0で主役表示。緑にしない＝2026-06-03実装で価格が緑だった要修正）
  // ↑ カード表示価格 = 農家販売価格（魚単価+内臓処理）×(1+手数料率)。手数料率は現状5%だが変動しうる（例 5,500×1.05=5,775）
  // 内臓処理はデフォルト込み。レストランが注文時に外すと −500（注文画面で選択。カードには内臓処理の行は出さない）
  stockKg,            // 200（在庫, kg）
  distanceKm,         // 5.5（農家↔レストランの実距離。カード表示＆ソート用）
  // ↓ 配達3パラメータ（農家が投稿時に登録）。送料はレストラン負担
  deliveryFeePer10km, // 5000（送料単価, KHR/10km）
  freeDeliveryKm,     // 5（送料無料距離。これ以内は送料0）
  deliverableRadiusKm,// 30（配達可能距離。distanceKm > これ の商品は非表示＝§7配達範囲外）
  // 送料計算(A案): distanceKm<=freeDeliveryKm→0 / freeDeliveryKm<distanceKm<=deliverableRadiusKm→deliveryFeePer10km×(distanceKm/10)（0kmから比例）/ それ超→配達不可
  // ※送料は配達単位の固定額（注文kgに依らない）
  isFavorite,         // true/false（♡。右上トグル）
  orderedBefore,      // true/false（過去注文フィルタ用）
  province,           // "タケオ州"（地域フィルタ用。カード面には出さない）
  createdAt,          // 新着順ソート用
  farmer: { name, avatar, ratingPositivePct, ratingCount, gaqp }  // フッター: アバター＋名＋★好評価率%(件数) 例 "★62% (37)"
}
// ※GAqPは農家単位の認証（商品単位ではない）。farmer.gaqp=true ならその農家の全商品の画像左上に緑バッジ ✓ GAqP。
// ※商品カードに州名は表示しない（距離📍と役割が被る。地域は絞り込み条件として使う）
```

## 4. 絞り込み（ボトムシート）

| 条件 | UI | 値 |
|---|---|---|
| 魚種 | チップ（複数選択） | 12種＋**「その他」**から。選択でAND/OR→**OR**（いずれか該当）。「その他」は農家が出品時に12種以外で登録した投稿（CAA参考価格なし）に対応し、これらが魚種フィルタで埋もれないようにする。※固定12種＝CAA価格あり11種（11種目＝バラマンディ）＋**オニテナガエビ**（បង្កង／Giant freshwater prawn。CAA参考価格は無いが固定種に追加 2026-06-11・甲殻類だが「魚」扱い） |
| 地域 | チップ（複数選択） | province一致（OR）。チップは固定リストでなく**農家が実在する州を動的生成**（モックの4州は例示） |
| 価格帯 | スライダー（上限） | **0〜25000 KHR/kg（固定）**。max時「上限なし」（=2.5万以上、高級魚も全て含む）。上限は**動的にしない**（高級魚1件の出品で密集帯が間延びするのを防ぐ。出品は何日も続くため） |
| ♡お気に入り | チップ（トグル） | isFavorite=true のみ |
| ⏱過去注文 | チップ（トグル） | orderedBefore=true のみ |

- 「リセット」で全解除。「この条件で表示」で適用→シート閉じ→件数更新。
- **適用数バッジ**: 選択チップ数＋上限変更したスライダー数の合計を「絞り込み」ボタンに表示。
- **地域(province)の表記ルール**: 首都は「プノンペン」（州なし）、他州は「◯◯州」（例: タケオ州/カンダル州/バッタンバン州）。農家カード等の表示もこれに従う。

---

## 5. ソート

ドロップダウン。選択肢: `距離が近い順`（デフォルト） / `評価が高い順` / `価格が安い順` / `新着順`。
- 距離 = distanceKm 昇順
- 評価 = ratingPositivePct 降順（同率は件数多い順を推奨）
- 価格 = priceKhrPerKg 昇順
- 新着 = createdAt 降順

---

## 6. 表示モード

- 1列横長カード / 2列グリッド の切替（モックに両CSS併存。`.grid` / `.grid.cols-2`）。
- **デフォルト = 2列グリッド**（`.grid.cols-2`）。実装版は現在1列なので**2列に変更**する。
- 理由: Homeの商品カードは縦型（画像上→価格→農家フッター）。2列も縦型カード＝**Homeとカード言語が揃う**（1列横長は画像左で別形状になり不整合）。一覧性も高く写真で魚を比較しやすい。

## 7. 状態分岐

### 売切れ（stockKg = 0）
- **グレーアウトして一覧の末尾固定**（どのソートを選んでも在庫>0より後ろ）。非表示にはしない。
- カード上に「本日売切れ」表示。**CTA（カート追加・入荷通知）は付けない**。タップは商品詳細へは可、カート追加は不可。
- 理由: 供給不安定が前提なので「今日は無い」を正直に見せる。ただしリピート単位は魚種なので、商品単位の入荷通知は意思決定に効かず付けない（魚種まるごと0件の通知は将来の魚種レベル機能で検討＝§10）。

### 配達範囲外（distanceKm > deliverableRadiusKm）
- **ベースルール（常時適用）として一覧に出さない＝完全非表示**。フィルタやソートに関係なく除外。
- 理由: 売切れは「明日は買えるかも」だから残すが、配達範囲外は**何日経っても自分には届かない**ので残す価値がない。表示すると毎回「買えない商品だ」と確認する無駄足が出る。
- **距離スライダーは廃止**（§4の絞り込みに距離条件は無い）。理由: 配達可否がこのベースルールで保証され、表示される商品は全て「自分に届く」もの。残った中の「近い方がいい」はソート「距離が近い順」（§5）＋カードの📍距離表示で十分カバーでき、ハードに絞る距離スライダーは役割が被って不要。

### 送料の表示（一覧／詳細・注文）
- **一覧カード**: `distanceKm ≤ freeDeliveryKm` のときだけメタ行に **「送料無料」**（緑・太字）を表示。有料の場合は**金額を出さない**（📍距離で察せる）。
- 理由: 送料は配達単位の固定額で、**注文量によって実質負担(KHR/kg)が激変**する（200kg注文なら誤差、5kg注文なら激痛）。一覧段階では注文量が未定なので絶対額を出すと誤読を招く。一方「送料無料」は注文量に関係なく常に正しく、近場農家の正当な強みを訴求できる。
- **商品詳細・注文画面**: 送料単価(KHR/10km)・無料距離・**実際の注文量での総額**を正確に表示（量が決まる場所で判断させる）。

### 品薄
- **バッジは付けない**（在庫kgの数値表示のみで足りる）。

### 該当0件（絞り込み結果）
- 空状態表示（「条件に合う魚がありません」＋絞り込みリセット導線）。※文言はモック未実装、実装時に追加。

### 一覧の読み込み
- **「もっと見る」ボタン方式**（無限スクロールにしない）。
- 理由: 低速回線で通信を制御でき挙動が予測可能。
- **初期表示＝10件**（2列×5行＝約1.5画面ぶん）。**「もっと見る」で +10件ずつ**。数字（10／+10）は**調整可能パラメータ**（実データの件数感で初期8〜12・増分8〜12に微調整可）。
- 絞り込み・ソート・検索を変えたら**先頭からやり直し**（展開数リセット）。商品詳細等から戻った時は展開数を保持（下記）。

### 状態の保持（戻り時）
- 商品詳細などから**戻ってきた時は完全保持**：ソート・絞り込み・検索・「もっと見る」展開数・スクロール位置をすべて維持。
- 理由: 「絞り込む→候補を1件ずつ詳細で見る→戻る」の比較行動を繰り返すため、毎回リセットは苦痛。
- 例外: ボトムナビで別タブへ離脱→ホーム等から入り直した場合は初期状態にリセット。

## 8. 画面遷移

| 要素 | 遷移先 |
|---|---|
| 商品カード | **商品詳細画面**（未着手）。※一覧カードに「＋カート」等のクイック追加は**付けない**。カート投入は商品詳細経由のみ |
| 農家フッター（アバター/農家名）タップ | **農家詳細画面**（未着手）。カード本体＝商品詳細とは別導線。誤タップ防止のためフッターのタップ領域を明確に分ける |
| 検索バー | **この画面内で絞り込み（in-place）。別の検索結果画面は作らない** |
| ボトムナビ各タブ | 各画面 |

### 検索バーの挙動
- 対象: **魚種名・農家名の両方**を検索（プレースホルダ「魚種・農家を探す」通り）。
- 入力中に**サジェスト候補**を表示（マッチする魚種12種＋農家名）。候補タップで確定。
- 確定すると**現在の一覧グリッドをその場で絞り込む**（ソート・絞り込み条件と併用＝AND）。件数表示も更新。
- 別画面遷移はしない。クリアで全件に戻す。

## 9. 画面間の統一ルール（Home/魚一覧 共通・厳守）

1. **評価表記** = `★ N% (件数)` の％表示（例 `★62% (37)`）。5段階★は使わない。
   - **評価が0件（ratingCount=0）のときは％を出さない**。「新規」相当の表示にする（`★0% (0)` は出さない）。理由: 未評価の新規農家が「好評価率0%＝低評価」に誤認され、最初の注文が入らず立ち上がらないのを防ぐ。新規農家には不利のない中立表示を与える（Airbnb等の "New" バッジと同じ思想）。
2. **GAqPバッジ** = 画像左上の緑バッジ `✓ GAqP`。農家フッター内テキスト表示は廃止。
3. **サイズ・価格単位の文字色** = 在庫・距離と同じサブ文字色（モック参考 `#5A6470`）。薄グレー(`#9AA3B0`相当)は使わない。価格数字は22px/800/プライマリ色のまま。
4. **色の役割（緑→青の徹底・2026-06-11 要修正）**: **青＝価格／操作／リンク／CTA／選択・アクティブ状態。緑＝GAqP・成功・送料無料の専用色**。具体的に魚一覧で緑だった以下を**青に**＝**①ソートの選択ハイライト（距離が近い順 等）②絞り込みの「この条件で表示」ボタン③ボトムナビのアクティブ（魚一覧 等・全画面共通）**。理由＝緑をCTA/選択に使うとGAqPバッジ（✓GAqP・緑）の「認証」信号が弱まる＋「押せる＝青」の学習とブレる。緑は §9-2 のGAqPバッジと §7 の「送料無料」だけに残す。買い手アプリ全体に適用（横断チェック）。
