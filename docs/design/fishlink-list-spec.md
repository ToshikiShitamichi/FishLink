# FISHLINK 魚一覧画面 実装仕様

実装の正本は **`fishlink-list-mockup.html`**（構造・レイアウト・絞り込みUI・ソートUIはそこを参照）。
本書はHTMLから読み取れない情報＝**データモデル・絞り込み/ソート仕様・状態・遷移**を補う。

### 実装方針の優先順位（Home specと同じ）
- **構成・データ・絞り込み/ソート・遷移** → 本書とモックHTMLの通りに実装する。
- **見た目のスタイル**（色・余白・角丸・影） → **既存アプリのデザインシステムに準拠**。モックの色値は役割を示す参考値。

---

## 1. 概要

「魚から探す」やボトムナビ「魚一覧」から来る、商品（＝農家が出している魚）の一覧。
レストランが魚種・地域・価格・距離で絞り、ソートして仕入れ先を選ぶ画面。
ドメイン制約はHome spec §1と共通（養殖のみ / KHR / head/kg / GAqP / 供給不安定で複数農家から仕入れ）。

---

## 2. 画面構成

ヘッダー（sticky）→ ページタイトル「魚一覧」→ ソート＋絞り込みバー → 件数表示 → カードグリッド → ボトムナビ（fixed）。

- ヘッダー: ロゴ / 🌐言語切替 / 🔔通知(未読バッジ) / 👤プロフィール ＋ 検索バー「魚種・農家を探す」
- ボトムナビ: ホーム / 魚一覧(active) / カート / 注文状況
- 絞り込みはボトムシート（下からせり上がる）。適用中は「絞り込み」ボタンに適用数バッジ。

---

## 3. データモデル

```js
// 商品カード（Home spec の product と同形）
product = {
  id,
  fishName,           // "ヒレナマズ"
  sizeHeadPerKg,      // "10 head/kg"（文字列。範囲もある "6-8 head/kg"）
  priceKhrPerKg,      // 6300（単価, KHR/kg。22px/800/プライマリ色で主役表示）
  stockKg,            // 200（在庫, kg）
  distanceKm,         // 5.5
  gaqp,               // true/false（trueなら画像左上に緑GAqPバッジ）
  isFavorite,         // true/false（♡。右上トグル）
  orderedBefore,      // true/false（過去注文フィルタ用）
  province,           // "タケオ州"（地域フィルタ用。カード面には出さない）
  createdAt,          // 新着順ソート用
  farmer: { name, avatar, ratingPositivePct, ratingCount }  // フッター: アバター＋名＋★好評価率%(件数) 例 "★62% (37)"
}
// ※商品カードに州名は表示しない（距離📍と役割が被る。地域は絞り込み条件として使う）
```

---

## 4. 絞り込み（ボトムシート）

| 条件 | UI | 値 |
|---|---|---|
| 魚種 | チップ（複数選択） | 10種から。選択でAND/OR→**OR**（いずれか該当） |
| 地域 | チップ（複数選択） | province一致（OR）。チップは固定リストでなく**農家が実在する州を動的生成**（モックの4州は例示） |
| 価格帯 | スライダー（上限） | 0〜10000 KHR/kg。max時「上限なし」 |
| 距離 | スライダー（上限） | 0〜100km。max時「上限なし」 |
| ♡お気に入り | チップ（トグル） | isFavorite=true のみ |
| ⏱過去注文 | チップ（トグル） | orderedBefore=true のみ |

- 「リセット」で全解除。「この条件で表示」で適用→シート閉じ→件数更新。
- **適用数バッジ**: 選択チップ数＋上限変更したスライダー数の合計を「絞り込み」ボタンに表示。
- **地域(province)の表記ルール**: 首都は「プノンペン」（州なし）、他州は「◯◯州」（例: タケオ州/カンダル州/バッタンバン州）。農家カード等の表示もこれに従う。

---

## 5. ソート

ドロップダウン。選択肢: `距離が近い順`（デフォルト） / `評価が高い順` / `価格が安い順` / `新着順`。
- 距離 = distanceKm 昇順
- 評価 = ratingPositivePct 降順（同率は件数多い順を推奨）
- 価格 = priceKhrPerKg 昇順
- 新着 = createdAt 降順

---

## 6. 表示モード

- 1列横長カード / 2列グリッド の切替（モックに両CSS併存。`.grid` / `.grid.cols-2`）。
- **デフォルト = 2列グリッド**（`.grid.cols-2`）。実装版は現在1列なので**2列に変更**する。
- 理由: Homeの商品カードは縦型（画像上→価格→農家フッター）。2列も縦型カード＝**Homeとカード言語が揃う**（1列横長は画像左で別形状になり不整合）。一覧性も高く写真で魚を比較しやすい。

---

## 7. 状態分岐

### 売切れ（stockKg = 0）
- **グレーアウトして一覧の末尾固定**（どのソートを選んでも在庫>0より後ろ）。非表示にはしない。
- カード上に「本日売切れ」表示。**CTA（カート追加・入荷通知）は付けない**。タップは商品詳細へは可、カート追加は不可。
- 理由: 供給不安定が前提なので「今日は無い」を正直に見せる。ただしリピート単位は魚種なので、商品単位の入荷通知は意思決定に効かず付けない（魚種まるごと0件の通知は将来の魚種レベル機能で検討＝§10）。

### 品薄
- **バッジは付けない**（在庫kgの数値表示のみで足りる）。

### 該当0件（絞り込み結果）
- 空状態表示（「条件に合う魚がありません」＋絞り込みリセット導線）。※文言はモック未実装、実装時に追加。

### 一覧の読み込み
- **「もっと見る」ボタン方式**（無限スクロールにしない）。実装版の現状に準拠。
- 理由: 低速回線で通信を制御でき挙動が予測可能。初期表示件数・追加件数は実装版の値を踏襲。

### 状態の保持（戻り時）
- 商品詳細などから**戻ってきた時は完全保持**：ソート・絞り込み・検索・「もっと見る」展開数・スクロール位置をすべて維持。
- 理由: 「絞り込む→候補を1件ずつ詳細で見る→戻る」の比較行動を繰り返すため、毎回リセットは苦痛。
- 例外: ボトムナビで別タブへ離脱→ホーム等から入り直した場合は初期状態にリセット。

## 8. 画面遷移

| 要素 | 遷移先 |
|---|---|
| 商品カード | **商品詳細画面**（未着手）。※一覧カードに「＋カート」等のクイック追加は**付けない**。カート投入は商品詳細経由のみ |
| 農家フッター（アバター/農家名）タップ | **農家詳細画面**（未着手）。カード本体＝商品詳細とは別導線。誤タップ防止のためフッターのタップ領域を明確に分ける |
| 検索バー | **この画面内で絞り込み（in-place）。別の検索結果画面は作らない** |
| ボトムナビ各タブ | 各画面 |

### 検索バーの挙動
- 対象: **魚種名・農家名の両方**を検索（プレースホルダ「魚種・農家を探す」通り）。
- 入力中に**サジェスト候補**を表示（マッチする魚種10種＋農家名）。候補タップで確定。
- 確定すると**現在の一覧グリッドをその場で絞り込む**（ソート・絞り込み条件と併用＝AND）。件数表示も更新。
- 別画面遷移はしない。クリアで全件に戻す。

---

## 9. 画面間の統一ルール（Home/魚一覧 共通・厳守）

1. **評価表記** = `★ N% (件数)` の％表示（例 `★62% (37)`）。5段階★は使わない。
2. **GAqPバッジ** = 画像左上の緑バッジ `✓ GAqP`。農家フッター内テキスト表示は廃止。
3. **サイズ・価格単位の文字色** = 在庫・距離と同じサブ文字色（モック参考 `#5A6470`）。薄グレー(`#9AA3B0`相当)は使わない。価格数字は22px/800/プライマリ色のまま。
