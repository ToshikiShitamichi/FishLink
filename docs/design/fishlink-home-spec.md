# FISHLINK Home画面 実装仕様

実装の正本は **`fishlink-home-mockup.html`**（構造・レイアウト・コンポーネント配置はそこを参照）。
本書はHTMLから読み取れない情報＝**データモデル・状態分岐・遷移・ドメイン制約**のみを補う。

### 実装方針の優先順位（重要）
- **構成・データ・状態・遷移**（画面構成 / セクション順 / カードの種類とデータ項目 / 注文ステータスや辞退の挙動 / 遷移先）
  → **本書とモックHTMLの通りに実装する**。今回新しく設計し直した部分。
- **見た目のスタイル**（色・タイポgrafi・余白・角丸・影など）
  → **既存アプリのデザインシステムに準拠**する。モックHTMLの色値・トークンは**参考/プレースホルダー**であり、そのまま使う前提ではない。既存DSの近い色・コンポーネントに置き換えること。

## 1. 概要・ドメイン制約（実装前提）

カンボジア向け **B2B養殖魚マーケットプレイス**（レストラン＝買い手 ⇄ 養殖農家＝売り手）。

- **養殖のみ**。天然/活魚のバッジは作らない。
- 通貨は **KHR**（リエル）。3桁区切り表示（例 `504,000`）。
- 魚のサイズは **head/kg**（1kgあたりの尾数。例 `6-8 head/kg`）。重量(g/kg)表記は使わない。
- **GAqP**（Good Aquaculture Practice）認証は**農家の属性**。商品・農家に紐づくバッジ。
- 供給が不安定（小規模農家が頻繁に売切れ）→ レストランは**複数農家から仕入れる**前提。
  リピートの単位は「農家」ではなく「**魚種**」。だから注文は農家に**辞退されうる**（後述）。
- 基本魚種は11種固定: ライギョ / ヒレナマズ / レッドティラピア / ナイルティラピア /
  シルバーバーブ / スポットパンガシウス / パンガシウス / ジャイアントスネークヘッド / バラマンディ / キノボリウオ / カエル
  - **バラマンディ**（英 Asian Sea bass / クメール ត្រីឆ្ពង់ / 学名 Lates calcarifer）は **CAA参考価格表に追加されたため固定種化**（2026-06-01）。CAA価格が付いた＝「その他」ではなく固定種。

## 2. 画面構成

ヘッダー（sticky）→ 検索バー → **キャンペーンバナー（任意）** → 5セクション（縦に並ぶ／各セクション内は横スクロール）→ ボトムナビ（fixed）。

| # | セクション | 表示するカード | 空状態ルール |
|---|---|---|---|
| — | キャンペーンバナー | 告知バナー（検索バー直下） | **キャンペーン中のみ表示。無ければ非表示** |
| 1 | 進行中の注文 | 注文カード | **進行中ゼロなら セクションごと非表示** |
| 2 | 魚から探す | 魚タイル（11種＋「その他」） | 常に全12タイル表示（空状態なし）。末尾に「その他」（11種以外の出品＝CAA参考価格なしへの導線。魚一覧フィルタの「その他」チップと整合） |
| 3 | 近くの農家の魚 | 商品カード | — |
| 4 | 新着の魚 | 商品カード | — |
| 5 | おすすめ農家 | 農家カード（評価順） | — |

### キャンペーンバナー（買い手向け）
- 内容 = **プラットフォーム取引手数料の割引キャンペーン**告知。対象は買い手＝レストラン。
- **手数料の仕組み**: 通常は手数料（現状5%。**変動・未確定**）が**表示価格に内包**され、買い手は手数料を項目として見ない。キャンペーン中は**お会計（明細）に割引行として `-5%相当` が出る**＝買い手は「割引された」と体感できる。
  - **カードの表示価格は据え置き**（手数料込みのまま）。割引は**明細でのみ反映**（カード一覧には反映しない）。
- **割引% = 通常率 − キャンペーン率**（＝明細の割引行と一致）。例: 5%→0%なら5%引き、5%→2.5%なら2.5%引き。
- **文言**（明細の割引と一致させる）: サブは必ず「魚代から{割引%}%割引 〜{期限}」（割引は**魚代ベース**＝配送料は対象外。手数料は魚代に乗るため。「お会計から」は計算実態とズレるので使わない）。メインは {キャンペーン率=0なら「手数料が無料」／半額なら「手数料半額」／他「手数料{割引%}%OFF」}。
  - **率はデータ化**（運営が設定した通常率・キャンペーン率を変数で流す）。モックの「無料・5%」は全額免除の一例でハードコードしない。
- **データ源＝運営画面（料金設定）を参照（割引%・終了日とも。ハードコード禁止）**:
  - 運営画面のキャンペーン設定 = `serviceRatePct`（サービス料率・通常率。例5.0）／`campaignEnabled`（有効化トグル）／`campaignDiscountPct`（キャンペーン割引率＝魚代から引く%。例2.5。＝本書の「割引%」に一致）。
  - **追加要件: 運営画面に `campaignEndDate`（終了日）フィールドを新設**（開始日は持たない＝有効化トグルが実質の開始操作。開始日を足すと「ONだが開始前」の曖昧状態が生まれ複雑化するため、予約投入が必要になるまで持たない）。
  - バナーのサブ「魚代から{割引%}%割引 〜{期限}」の **{割引%}=`campaignDiscountPct`、{期限}=`campaignEndDate`** を表示（手打ち値を出さない）。
- **表示条件 = `campaignEnabled === true` かつ 今日 ≤ `campaignEndDate`**。終了日を過ぎたら自動で非表示（運営が消し忘れても勝手に止まる）。
- 配置 = 検索バー直下。**期間中のみ表示**、期間外は要素ごと非表示（進行中の注文と同じ流儀）。
- ×（pclose）で閉じられる。閉じた状態の保持有無は実装判断（推奨: そのキャンペーンIDは再表示しない）。
- タップでキャンペーン詳細へ（詳細画面は未着手）。
- バナー形式の理由: 手数料は全商品横断の取引条件 → 特定の商品カードに付けると「その商品だけ安い」と誤読される。期間限定なので常設セクションよりバナーが「今だけ」を伝えやすい。
- **表示はHome画面のみ**（魚一覧など他画面には出さない）。理由: Home＝入口で一度告知すれば認知は足りる／魚一覧は比較の作業画面でバナーが邪魔／割引は全注文共通で明細に必ず再表示される（Home＝認知・明細＝回収）。
  - **【2026-06-03 要修正】バナーが「魚一覧に紐付き」で実装されていた→魚一覧から削除した結果どこにも出ない状態。バナーを Home（検索バー直下）に紐付け直す。表示条件＝バナー表示ON かつ campaignEnabled かつ 今日≤campaignEndDate。管理画面「キャンペーンバナー表示」の説明文「レストランの魚一覧の上部に…」を「Home画面の上部に…」へ修正。**

- ヘッダー要素: ロゴ / 🌐言語切替 / 🔔通知(未読バッジ) / ✓✓やることリスト / 👤プロフィール
- ボトムナビ: ホーム / 魚一覧 / カート / 注文状況

### 評価・GAqP表示ルール（商品カード／おすすめ農家カード共通・魚一覧 spec と統一）

1. **評価が0件（ratingCount=0）のときは％を出さない**。「新規」相当の表示にする（`★0% (0)` は出さない）。理由: 未評価の新規農家が「好評価率0%＝低評価」と誤認され立ち上がらないのを防ぐ（Airbnb等の "New" バッジと同じ思想）。
2. **GAqPバッジ** = 画像左上の緑バッジ `✓ GAqP`（商品カード・おすすめ農家カードで文言統一）。「GAqP認証済み」等の別表記は使わない。**サイズはコンパクトに**（小さいピル状）。**おすすめ農家カードで大きい丸シールにしない**＝商品カードと同サイズ・同形（2026-06-03 実装で農家カードのバッジが大きすぎた→要修正）。

## 3. データモデル（JSオブジェクト形状）

```js
// 注文カード（セクション1）
order = {
  id,
  fishName,          // "ナイルティラピア"
  sizeHeadPerKg,     // "6-8 head/kg"（文字列。範囲もある）
  farmerName,        // "サンサン養殖場"
  qtyKg,             // 80（注文量, kg）
  totalKhr,          // 504000（合計金額, KHR。3桁区切りで表示）
  status,            // 下記 status enum
  eta,               // "本日 16:00頃" 等。status="confirming"のときは null
}

status = "ordered"      // 注文済
       | "confirming"   // 農家確認中（在庫確認の承認待ち）
       | "shipped"      // 出荷
       | "delivering"   // 配達中
       | "done"         // 完了
       | "declined"     // 辞退（農家が在庫切れで断った）

// 商品カード（セクション3・4）
product = {
  id,
  fishName,
  sizeHeadPerKg,
  priceKhrPerKg,     // 6300（単価, KHR/kg。22px/800/青で主役表示）
  stockKg,           // 120（在庫, kg）
  distanceKm,        // 4.2
  gaqp,              // true/false（trueなら画像左上に緑GAqPバッジ ✓ GAqP）
  farmer: { name, ratingPositivePct, ratingCount }  // フッター: アバター＋名＋★好評価率%(件数)。例 "★96% (32)"。ratingCount=0 は「新規」表示（§評価・GAqP表示ルール）
}
// ※商品カードに州名は出さない（距離と役割が被るため）

// 農家カード（セクション5）
farmer = {
  id,
  name,
  province,          // "タケオ州"。表記ルール: 首都は「プノンペン」(州なし)、他州は「◯◯州」
  distanceKm,        // 8.5
  ratingPositivePct, // 96（好評価率%。表示は "★96%"）
  ratingCount,       // 32（評価件数。表示は "(32)"。0 のときは「新規」表示＝§評価・GAqP表示ルール）
  gaqp,              // true/false（trueなら緑バッジ ✓ GAqP）
  fishTags: [..],    // 取扱魚種タグ（例 ["ティラピア","ナマズ","ライギョ"]）
}

// 魚タイル（セクション2）固定11種
fishSpecies = { name, icon }   // データ項目は名前のみ。農家数等は出さない
```

## 4. 状態分岐

### 注文カードのステータス（5段ステッパー + 辞退）
- ステッパー段: `注文済 → 農家確認中 → 出荷 → 配達中 → 完了`。
  - 完了済みの段＝緑、現在の段＝青リング、未到達＝グレー。
- `status="confirming"` のとき: ETA欄は `⏳ 農家の在庫確認を待っています`（到着予定が未確定なため）。
- `status="declined"`（辞退）:
  - カードを**赤枠**にし、**進行中の先頭に固定**（対処が必要なため最優先）。
  - 「農家確認中」の位置を**赤＋白✕印**で停止表示し、ラベルを「辞退」に。
  - 警告 `⚠️ {farmerName}が在庫切れで辞退しました`（赤背景）。
  - 主役ボタン `他の農家を探す →` を表示（遷移は §5）。**ボタンの色は青**（主CTA。緑にしない）。
  - **辞退カードが消えるタイミング（2026-06-03 確定）**: 先頭固定で残すが**放置で無限に溜めない**。①「他の農家を探す」から**代替注文を入れたら自動でクリア**（解決済み）／②**手動の「×（閉じる）」でも消せる**（別ルートで手配済み等）。消えた辞退は**注文状況の「辞退済み」に記録**（履歴保持）。
- 完了(`done`)になった注文は進行中から**即消す**（Home上は完了を残さない。完了の閲覧は「注文状況」タブ）。**レビュー促し（完了時プッシュ＋注文状況の「レビュー待ち」導線）はレビュー送信フロー設計時に詰める。**

## 6. 補足（モックHTML参照項目 / 見た目は既存DSに置換）

以下のうち**寸法・配置はモックHTMLが正本**。**色・スタイルは既存アプリのデザインシステムに合わせる**（モックの色値は役割を示す参考値）。
- レイアウト寸法: カード幅＝注文 300px / 商品 220px / 農家 240px。横スクロールは1.5枚見え想定。
- **色の「役割」**（具体値ではなく用途。既存DSの対応色に置換すること）:
  - 価格・主役アクション = プライマリ**青** `#0B5FB0`（モック参考）。**緑は使わない**。
  - GAqPバッジ = 認証/成功**緑**（GAqP・成功・送料無料 専用）
  - ★評価 = アクセント/警告系の黄
  - 辞退・警告 = エラー色（赤・danger 専用）
  - **【2026-06-03 Home実装で緑になっていた要修正】商品カードの価格／進行中の注文の合計／「他の農家を探す」ボタン（辞退時の主CTA）＝すべて緑→青に。価格・CTAに緑を使わない（緑＝GAqP/成功/送料無料のみ）。**
- 価格スタイル: 大きめ(22px相当) / 太字 / プライマリ色、単位は小さい薄字。
- カードは2種類のみ（商品カード・農家カード）＋注文カード。再注文ボタンや鮮度バッジのバリアントは作らない。
