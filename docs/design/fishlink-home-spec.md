# FISHLINK Home画面 実装仕様

実装の正本は **`fishlink-home-mockup.html`**（構造・レイアウト・コンポーネント配置はそこを参照）。
本書はHTMLから読み取れない情報＝**データモデル・状態分岐・遷移・ドメイン制約**のみを補う。

### 実装方針の優先順位（重要）
- **構成・データ・状態・遷移**（画面構成 / セクション順 / カードの種類とデータ項目 / 注文ステータスや辞退の挙動 / 遷移先）
  → **本書とモックHTMLの通りに実装する**。今回新しく設計し直した部分。
- **見た目のスタイル**（色・タイポgrafi・余白・角丸・影など）
  → **既存アプリのデザインシステムに準拠**する。モックHTMLの色値・トークンは**参考/プレースホルダー**であり、そのまま使う前提ではない。既存DSの近い色・コンポーネントに置き換えること。

## 1. 概要・ドメイン制約（実装前提）

カンボジア向け **B2B養殖魚マーケットプレイス**（レストラン＝買い手 ⇄ 養殖農家＝売り手）。

- **養殖のみ**。天然/活魚のバッジは作らない。
- 通貨は **KHR**（リエル）。3桁区切り表示（例 `504,000`）。
- 魚のサイズは **head/kg**（1kgあたりの尾数。例 `6-8 head/kg`）。重量(g/kg)表記は使わない。
- **GAqP**（Good Aquaculture Practice）認証は**農家の属性**。商品・農家に紐づくバッジ。
- 供給が不安定（小規模農家が頻繁に売切れ）→ レストランは**複数農家から仕入れる**前提。
  リピートの単位は「農家」ではなく「**魚種**」。だから注文は農家に**辞退されうる**（後述）。
- 基本魚種は10種固定: ライギョ / ヒレナマズ / レッドティラピア / ナイルティラピア /
  シルバーバーブ / スポットパンガシウス / パンガシウス / ジャイアントスネークヘッド / キノボリウオ / カエル

## 2. 画面構成

ヘッダー（sticky）→ 検索バー → 5セクション（縦に並ぶ／各セクション内は横スクロール）→ ボトムナビ（fixed）。

| # | セクション | 表示するカード | 空状態ルール |
|---|---|---|---|
| 1 | 進行中の注文 | 注文カード | **進行中ゼロなら セクションごと非表示** |
| 2 | 魚から探す | 魚タイル（10種固定） | 常に全10種表示（空状態なし） |
| 3 | 近くの農家の魚 | 商品カード | — |
| 4 | 新着の魚 | 商品カード | — |
| 5 | おすすめ農家 | 農家カード（評価順） | — |

- ヘッダー要素: ロゴ / 🌐言語切替 / 🔔通知(未読バッジ) / ✓✓やることリスト / 👤プロフィール
- ボトムナビ: ホーム / 魚一覧 / カート / 注文状況

## 3. データモデル（JSオブジェクト形状）

```js
// 注文カード（セクション1）
order = {
  id,
  fishName,          // "ナイルティラピア"
  sizeHeadPerKg,     // "6-8 head/kg"（文字列。範囲もある）
  farmerName,        // "サンサン養殖場"
  qtyKg,             // 80（注文量, kg）
  totalKhr,          // 504000（合計金額, KHR。3桁区切りで表示）
  status,            // 下記 status enum
  eta,               // "本日 16:00頃" 等。status="confirming"のときは null
}

status = "ordered"      // 注文済
       | "confirming"   // 農家確認中（在庫確認の承認待ち）
       | "shipped"      // 出荷
       | "delivering"   // 配達中
       | "done"         // 完了
       | "declined"     // 辞退（農家が在庫切れで断った）

// 商品カード（セクション3・4）
product = {
  id,
  fishName,
  sizeHeadPerKg,
  priceKhrPerKg,     // 6300（単価, KHR/kg。22px/800/青で主役表示）
  stockKg,           // 120（在庫, kg）
  distanceKm,        // 4.2
  gaqp,              // true/false（trueなら画像左上に緑GAqPバッジ）
  farmer: { name, ratingPositivePct, ratingCount }  // フッター: アバター＋名＋★好評価率%(件数)。例 "★96% (32)"
}
// ※商品カードに州名は出さない（距離と役割が被るため）

// 農家カード（セクション5）
farmer = {
  id,
  name,
  province,          // "タケオ州"。表記ルール: 首都は「プノンペン」(州なし)、他州は「◯◯州」
  distanceKm,        // 8.5
  ratingPositivePct, // 96（好評価率%。表示は "★96%"）
  ratingCount,       // 32（評価件数。表示は "(32)"）
  gaqp,              // true/false
  fishTags: [..],    // 取扱魚種タグ（例 ["ティラピア","ナマズ","ライギョ"]）
}

// 魚タイル（セクション2）固定10種
fishSpecies = { name, icon }   // データ項目は名前のみ。農家数等は出さない
```

## 4. 状態分岐

### 注文カードのステータス（5段ステッパー + 辞退）
- ステッパー段: `注文済 → 農家確認中 → 出荷 → 配達中 → 完了`。
  - 完了済みの段＝緑、現在の段＝青リング、未到達＝グレー。
- `status="confirming"` のとき: ETA欄は `⏳ 農家の在庫確認を待っています`（到着予定が未確定なため）。
- `status="declined"`（辞退）:
  - カードを**赤枠**にし、**進行中の先頭に固定**（対処が必要なため最優先）。
  - 「農家確認中」の位置を**赤＋白✕印**で停止表示し、ラベルを「辞退」に。
  - 警告 `⚠️ {farmerName}が在庫切れで辞退しました`（赤背景）。
  - 主役ボタン `他の農家を探す →` を表示（遷移は §5）。
- 完了(`done`)になった注文は進行中から**即消す**（Home上は完了を残さない。完了の閲覧は「注文状況」タブ）。

## 5. 画面遷移（各タップ先）

| 要素 | 遷移先 |
|---|---|
| 魚タイル（セクション2） | その魚種を**今日出している農家一覧**（本命画面・未着手） |
| 注文カード（通常） | 注文詳細画面（未着手） |
| 注文カードの「他の農家を探す →」（辞退時） | その魚種の農家一覧（魚タイルと同じ遷移先） |
| 商品カード | **商品詳細画面**（未着手） |
| 農家カードの「農家を見る →」 | 農家詳細画面（未着手） |
| 各セクション見出しの「もっと見る ›」 | 該当一覧画面 |
| セクション1見出しの「注文状況 ›」／ボトムナビ「注文状況」 | 注文状況一覧（完了注文もここで閲覧。**再注文ボタンは無し**） |

## 6. 補足（モックHTML参照項目 / 見た目は既存DSに置換）

以下のうち**寸法・配置はモックHTMLが正本**。**色・スタイルは既存アプリのデザインシステムに合わせる**（モックの色値は役割を示す参考値）。
- レイアウト寸法: カード幅＝注文 300px / 商品 220px / 農家 240px。横スクロールは1.5枚見え想定。
- **色の「役割」**（具体値ではなく用途。既存DSの対応色に置換すること）:
  - 価格・主役アクション = プライマリ色（モック参考: `#0B6FB8`）
  - GAqPバッジ = 認証/成功色（モック参考: `#1FA463`）
  - ★評価 = アクセント/警告系の黄（モック参考: `#F0A500`）
  - 辞退・警告 = エラー色（モック参考: `#E5484D`）
- 価格スタイル: 大きめ(22px相当) / 太字 / プライマリ色、単位は小さい薄字。
- カードは2種類のみ（商品カード・農家カード）＋注文カード。再注文ボタンや鮮度バッジのバリアントは作らない。
