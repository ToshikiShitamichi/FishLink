# FISHLINK 農家側 投稿一覧画面 実装仕様

実装の正本は **`fishlink-posting-list-mockup.html`**（構造・レイアウト・カード・アクションバー・グルーピングはそこを参照）。
本書はHTMLから読み取れない情報＝**データモデル・状態・遷移・状態保持**を補う。

### 実装方針の優先順位（買い手側 Home/魚一覧 spec と同じ）
- **構成・データ・状態・遷移** → 本書とモックHTMLの通りに実装する。
- **見た目のスタイル**（色・余白・角丸・影） → **既存アプリのデザインシステムに準拠**。モックの色値は役割を示す参考値（プライマリ青 `#0B5FB0` / GAqP緑 `#2E7D32` / サブ文字 `#5A6470`）。

---

## 1. 概要

農家（売り手）が自分の出品を管理するダッシュボード。販売中／売切れ／出品停止のステータスごとに自分の投稿を一覧し、補充・編集・停止・再開・削除を行う。
ドメイン制約は買い手側と共通（養殖のみ / KHR / head/kg / GAqP / 供給不安定で売切れが頻発）。

**買い手一覧との根本的な違い**: 買い手は「買える物だけ見たい」→売切れは末尾・配達範囲外は非表示。農家は「自分の商売を管理する」→**売切れこそ機会損失として目立たせる**（買い手の流儀を持ち込まない）。

---

## 2. 画面構成

ヘッダー（sticky）→ ページタイトル「投稿一覧」→ ステータス別グループ（カードリスト）→ ボトムナビ（fixed）。

- ヘッダー: ロゴ / 🌐言語切替 / 🔔通知(未読バッジ) / ✅やることリスト(未対応バッジ) / 👤プロフィール
  - **🔔通知と✅やることリストの役割分担は未確定（§9）**。中身の画面も未着手。
- ボトムナビ: 投稿一覧(active) / 出品する / 注文確認
  - **新規出品はボトムナビ「出品する」に一本化**。右下FABは付けない（重複のため廃止）。農家の日常作業は既存投稿の管理であり、新規出品を画面の主役にしない。
  - **ナビの色（確定 2026-06-19）**: アクティブ＝青 `#0B5FB0`、非アクティブ＝ニュートラルグレー。**中央「➕出品する」を緑にしない**（緑＝GAqP専用。買い手側で確定の「緑→青」系統ルールを農家ナビにも適用。実装は中央が緑になっている → 青/ニュートラルへ）。
- カードは**1列**（横長の管理カード）。買い手の2列グリッド＝写真で比較するブラウジング用途と違い、こちらは1枚ずつ状態とアクションを確実に扱う管理用途。

## 3. データモデル

```js
// 農家自身の投稿。買い手側 product と同じ実体を、農家視点のフィールドで持つ
posting = {
  id,
  fishName,           // "ヒレナマズ"（12種から選択 or その他）
  fishKind,           // "ヒレナマズ" | "その他"（12種固定＋「その他」のselect。自由入力ではない。固定12種=CAA11種[11種目=バラマンディ]＋オニテナガエビ[CAA参考価格なし]）
  sizeHeadPerKg,      // "10 head/kg"（文字列。範囲もある "6-8 head/kg"）
  priceKhrPerKg,      // 5775（単価, KHR/kg。22px/800/プライマリ色＝青で主役表示）
  gutFeeKhrPerKg,     // 500（内臓処理オプションの追加料金。カードに「内臓処理 +500 KHR/kg」と表示）
  stockKg,            // 200（在庫, kg。注文で自動減算。0で自動「売切れ」）
  deliverableRadiusKm,// 20（配達できる範囲。投稿時に農家が登録。買い手側で distanceKm>これ の商品は非表示）
  status,             // "selling" | "soldout" | "stopped" → §4
  createdAt,          // 出品日時
  imageUrl,           // 商品画像
  farmer: { name, avatar, gaqp }  // gaqp=農家単位（取得するとこの農家の全投稿に緑✓GAqPバッジ。商品単位ではない）
}
```

- **status は stockKg と stopped 操作から導出**: 出品停止中なら `stopped`、在庫0なら `soldout`、それ以外 `selling`。
- **CAA参考価格**: **CAA価格あり11種**には出品/編集画面で「相場はこれくらい」と参考価格を表示し値付けを助ける（**オニテナガエビ・その他は参考価格なし＝ボックス非表示**）。投稿一覧カードには出さない。※固定12種＝CAA11種（11種目＝バラマンディ）＋オニテナガエビ（បង្កង／Giant freshwater prawn。CAA参考価格は無いが固定種に追加 2026-06-11・甲殻類だが「魚」扱い）

## 4. 投稿ステータス（3状態）

| 状態 | 条件 | バッジ | 買い手側での見え方 |
|---|---|---|---|
| 販売中 | stockKg > 0 かつ停止していない | 緑「販売中」 | 通常表示 |
| 売切れ | stockKg = 0（**自動**） | オレンジ「売切れ」 | グレーアウト末尾固定・非表示にしない（買い手 spec §7） |
| 出品停止 | 農家が**手動**で停止 | グレー「停止中」 | **一覧から完全非表示**（買い手 spec §7） |

- **在庫0→自動で「売切れ」**（自動で出品停止にはしない／投稿は残す）。
- **持続投稿（ライフサイクル）**: 日次リセットなし・自動期限切れなし。在庫は注文で自動減算→0で自動売切れ→農家が補充（編集で数量入れ直し）で販売中に復活。鮮度の自動期限切れ機構は今は作らない。
- **出品停止 ⇄ 再開**: 停止＝買い手から完全非表示（売切れのグレー残しと違い、農家の意図的な取り下げなので隠す）。登録内容（在庫/価格等）は保持し、ワンタップで再開。想定用途＝今日配達不可／全量売約済／価格見直し／季節休み。

## 5. 並び順＝ステータスでグルーピング

セクション見出し＋件数で固定順に並べる（ソートUIは持たない）。

```
● 販売中 (n)
● 売切れ・要補充 (n)
● 出品停止 (n)
```

- **発想転換**: 買い手は売切れを末尾に隠すが、農家にとって売切れ＝補充すれば売れる物＝**機会損失**なので「要補充」として目立つ位置（販売中の直後）に置き、気づかせる。
- 投稿が少ない初期はグループが1〜2個でも成立。投稿増加を見越してグルーピングを採用。

## 6. カード構成とアクション

カード = サムネ（GAqP農家なら左上に緑✓GAqPバッジ）＋魚名＋ステータスバッジ／`サイズ head/kg ・ 在庫 N kg`／**価格（青・22px/800）**＋「内臓処理 +500 KHR/kg」／下部アクションバー。

- **価格は青（プライマリ）に統一**。緑はGAqP専用（買い手/農家 共通の統一ルール）。
- **内臓処理**は農家が投稿時に追加料金を登録。カードには `内臓処理 +500 KHR/kg` と表示（買い手は注文画面で選択）。

### アクションバー（状態別・4ボタン）

| 状態 | ボタン構成 |
|---|---|
| 販売中 | ✏編集 / 💬質問 / **⏸出品停止（赤/danger）** / ⋮ |
| 売切れ | **📦在庫を補充（青/primary）** / 💬質問 / ⏸出品停止（赤） / ⋮ |
| 停止中 | ✏編集 / 💬質問 / **▶再開（青/primary）** / ⋮ |

- **ぱっと見の混乱対策**: ①停止中カードは**全体グレーアウト**（grayscale＋opacity）で状態を色で即判別 ②向きのあるボタン（停止/再開）は**アイコン＋色**で示す ③バッジは念押しで残す。
- **💬質問ボタンの未回答バッジ（確定 2026-06-19）**: その投稿に**対応すべき質問スレッドがある場合のみ**、「💬質問」ボタンに**件数バッジ（赤丸・白数字 `#E5453F`）**を出す（例「💬 質問 ②」）。0なら**出さない**（中身ゼロで枠だけ出さない＝0件バグ回避）。色はヘッダーの🔔・✓✓の赤丸通知バッジと同じ赤＝「対応が必要」のサイン。
  - **スレッドの状態で扱いを分ける（消えるタイミングが2系統）**：

    | スレッドの状態 | バッジに数える | 消えるタイミング |
    |---|---|---|
    | **要返信**（農家がまだ一度も返信していない＝買い手の番） | ○ | **農家が返信したら**（開くだけでは消えない＝放置防止） |
    | **回答済み**（最後の投稿が農家） | × | — |
    | **未読フォロー**（農家が返信した後に買い手が再投稿し農家が未読。例「ありがとう」） | ○ | **農家がそのスレッドを開いたら（既読）**（返信不要・幽霊バッジ化を防ぐ） |

  - **質問ページを開くと**：未読フォローは開封で消える／**要返信は開いただけでは残る**（返信して初めて消える）。＝開いた後のバッジは「本当に答えるべき質問の数（＝要返信）」に収束。
  - **割り切り**：農家が一度答えた後の買い手の追加質問も“未読フォロー”扱い（開けば消える・要返信には戻さない）。「質問か挨拶か」は機械判定しない＝**初回質問の取りこぼしだけを強く守る**設計。
  - **3か所の役割分担**（同一ではない・関連する別集合）：**カードのバッジ＝要返信＋未読フォロー**（新着ぜんぶ／開くと要返信だけに収束）／**ヘッダー✓✓やることリスト＝要返信のみ**（“答えるべき質問”だけ積む・挨拶はToDoにしない・失注防止）／**🔔通知＝買い手の新着ごと**（既読で消える）。
  - 実装：投稿ごとに「要返信スレッド数」と「未読フォロースレッド数」をカウントし合算をバッジに。既読/返信でそれぞれ減算。
- **📦在庫を補充 ＝ 編集画面へのショートカット**（在庫フィールドにフォーカス）。補充は結局「編集画面で在庫を再入力」なので、売切れカードでは「✏編集」を「📦在庫を補充」に**置換**（ボタンは増やさない＝重複ゼロ）。価格やサイズも同じ編集画面で直せる。

### ⋮メニュー（削除）

- **削除**: 販売中は直接削除不可 → **先に出品停止してから削除**。確認ダイアログに「取引履歴からは確認できます」と明記（過去注文の記録は消えない）。
- **確認は自前モーダル（確定 2026-06-19）**: `confirm()` のブラウザ標準ダイアログ（「○○.web.app の内容」とドメインが出る）は使わない → **アプリ内の自前モーダル**にする。承認モーダル・カート削除と同じ共通改善（横断で `confirm()` を自前モーダルへ統一する一部）。
  - 文言＝維持：タイトル「本当に削除しますか？」／本文「削除すると投稿一覧に表示されなくなります。」＋補足（薄色）「過去の取引履歴からは引き続き確認できます」。
  - ボタン＝**「削除する」赤/danger（#E5453F）** ＋ **「キャンセル」ニュートラル**（標準の「OK（紫）」は廃止。削除＝破壊的操作なので確定ボタンは赤・出品停止の赤と一貫。ラベルは「OK」でなく動詞「削除する」で明確化）。
  - **配置＝中央アラート（確定 2026-06-19）**: ボトムシートにはしない。役割を分ける＝**ボトムシート＝入力/選択（絞り込み・配送設定）／中央アラート＝二択の確認・破壊的操作**。確認系は中央＋背景暗転で一拍止めるのが正（iOS/Android のネイティブ確認も中央／下スワイプで誤って閉じない／承認モーダルも自前・中央で一貫）。

### ✏編集

- 注記「編集すると、これから入る**新しい注文**に反映されます。**現在進行中・完了済みの注文**表示には影響しません」＝編集は未来の注文のみに効く。

## 7. 画面遷移

| 要素 | 遷移先 |
|---|---|
| ✏編集 / 📦在庫を補充 | **編集画面**（補充は在庫フィールドにフォーカス）。未着手 |
| 💬質問 | **質問画面（公開Q&A）**。未確定・後で詰める（§9） |
| ⏸出品停止 / ▶再開 | その場でステータス変更（画面遷移なし）。停止は確認ダイアログ推奨 |
| ⋮ → 削除 | **自前モーダル**（赤「削除する」/ニュートラル「キャンセル」）→ 削除（販売中は不可、停止後のみ） |
| ボトムナビ「出品する」 | **出品フロー（新規投稿）**。別画面・後で詰める |
| ボトムナビ「注文確認」 | 注文管理画面。未着手 |

## 8. 状態の保持

- 編集・補充・停止/再開などから戻った時は一覧の表示状態（スクロール位置・展開）を保持。
- 操作（補充→販売中、停止→停止中）の結果はグループの再配置に即反映。
