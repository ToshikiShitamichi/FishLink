# 【実装依頼】農家 出品フォーム（魚を投稿する）の修正3点（2026-07-05）

下道さん

農家の出品フォームを実機で確認して、修正が3点あります。正本＝モック `fishlink-posting-create-mockup.html`／仕様書 `fishlink-posting-create-spec.md`。

## 🟡 ① 内臓処理の注記が価格モデルと矛盾
現状の注記「**内臓処理を希望する注文に加算されます**」＝オプトイン（デフォルトなし・希望したら加算）に読めます。でも実際は「販売価格＝魚単価＋内臓処理」＝**デフォルト込み**（レストランが注文時に外すと −500）。フォーム内で矛盾。

→ 注記を **「内臓処理込みの価格です。レストランが不要なら注文時に外せます（−500 KHR/kg）」** に変更。

## 🟠 ② 金額入力を100リエル単位に（魚単価・内臓処理・10kmあたり送料）
100リエル丸めルール（`fishlink-request-amount-rounding.md`・payment-spec §4.2）の「**単価・送料は100刻み**」がこのフォームに未反映。

- **魚単価**にヘルパー「**100リエル単位で入力してください（例：7,500）**」＋placeholderを「例：7,500」に。
- **検証**：投稿時に100の倍数でなければインラインエラー「**100リエル単位で入力してください**」（既存の必須未入力と同じ赤インライン方式）。
- **自動で丸めない**（農家の価格を勝手に変えない＝低リテラシーの"勝手に変わった"不安を避ける。農家は元々丸めて入れるのでほぼ発動しない）。
- 対象＝魚単価・内臓処理（500）・配送の10kmあたり送料。

## 🟠 ④ 内臓処理「する／しない」トグルを新設
**内臓処理をやらない農家（丸魚のみ販売）に対応**（現状は500プリフィルで全農家が提供する前提になっている）。

- 内臓処理の欄に **「する／しない」トグル**（既定＝**する**・淡い青の選択状態＝目立たせすぎない）。
  - **する**：料金入力（500・100刻み）＋デフォルト込み（①の注記）。販売価格＝魚単価＋内臓処理。
  - **しない**：料金欄を消す・**販売価格＝魚単価**・注記「内臓処理なし（丸魚のみ）で販売します。買い手は内臓処理を選べません」。
- データ：`gutFeeKhrPerKg` を **null＝提供しない（しない）** で表現。
- お手本＝モックの ①②（する）・④（しない）状態。

### ④の波及（内臓処理「しない」＝`gutFeeKhrPerKg=null` の見せ方）
- **買い手・商品詳細**：価格直下の「内臓処理込み」ラベルを出さない → **「丸魚のみ（内臓処理なし）」**（お手本＝`fishlink-product-detail-mockup.html` ③）。
- **買い手・注文フロー（商品詳細・カート）**：「内臓処理あり／なし」の選択肢を出さない（丸魚のみ）。
- **農家・投稿一覧**：カードの「内臓処理 +500 KHR/kg」を出さない → **「丸魚のみ（内臓処理なし）」**（お手本＝`fishlink-posting-list-mockup.html` 販売中カード）。

※買い手/農家側モックの調整は後追い（sweep時）。実装は先に対応いただけると助かります。

## 補足
