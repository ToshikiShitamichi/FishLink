## まず見てほしい：feasibility（仕様書 §9）
これが成立するかで、機能をやるか決まります。**FBのリールが現地で回ってる＝見込みはある**が、要判断：
- **帯域**：農村の低速回線。短尺（5〜15秒）・低解像度・自動圧縮でFBリール相当に抑えられるか。
- **アップロード**：端末側で圧縮して送る／撮影＋ギャラリー選択。
- **保存・配信**：動画ストレージ／CDN／ストリーミング（運用コスト含む）。**「1出品に最大N本」保持**＝上限Nでストレージを有界化。1本≒5〜10MB仮定で、出品10個の農家の最悪ケース ≒ 10出品×10本×〜10MB ≒ **約1GB/農家**。**重ければN=10→5で半減**。→ **Nの現実的な値を含め、感触を教えてください。**
- **再生**：カルーセル/全画面の自動再生（ミュート）・スクロール先読み。

→ **ここが「いけそう」か「厳しい」か、感触を先に教えてください。** 厳しければ規模を落とす/見送るを一緒に考えます。

## 機能の狙い
- **「魚は動き」＝跳ねてる/活〆の短尺動画で鮮度を証明**。写真は盛れるが動画は盛れない＝リモートB2Bの最大の信頼ギャップを崩す。
- **需要は実証済み**：FBのリールで現に注文が入っている。農家も既にFB用に動画を撮っている＝習慣を移すだけ。
- 動画は**任意スタート**（写真は必須のまま）＝出品ハードルは上げない。
- ポートフォリオ＝過去動画を溜めて「この農家、ずっと本物を上げてる」という**時系列の信頼アーカイブ**にする（1本の宣伝で終わらせない）。

## 実装の全体像（設計は確定済み）

### 1. 農家が動画を上げる
- **出品フォームに「🎬動画（任意）」**（写真は必須のまま・撮影/ギャラリー・「動画があると売れやすい」nudge）。
- **「🎬リールを投稿」＝いつでも投稿**（出品する＋で「魚を出品する／リールを投稿する」の2択・在庫がある出品を選んで投稿・出品ゼロなら「この動画で新しく魚を出品する」＝動画を持って出品フォームへ）。
- **差し替え＝アーカイブ行き（物理削除しない）**：同じ出品に新しい動画を上げると、**カード/Homeに出る最新1本（featured）が入れ替わり、古い動画はその出品の動画リストに降りて残る**。UI文言は「更新」。
- **上限＝出品ごと最新N本（推しN=10）**：N+1本目で最古を物理削除（storage解放）。**Nは下道さんのストレージ感で最終決定（10→5に下げ可）**。
- **④「この魚の動画」画面**（投稿一覧の「🎬動画を確認・更新」ボタン等から着地・mock reel-post ④）＝その出品の動画を**新しい順にリスト表示・各行＝サムネ＋日付（「2時間前」等）＋🗑個別削除**。**featured を示すバッジ・場所ラベルは付けない**（新しい順で先頭＝最新が自動決定・農家が管理不要。説明文1行「新しい順。一番新しい動画が自動で買い手に一番見せられます。古い動画も消えずに残ります（最大N本・超えると古いものから自動で消えます）」）。〔↻新しい動画を投稿〕。個別削除は撮り直しミス等を消せる（公開ポートフォリオの必須機能）。最新を削除したら次に新しいのが自動繰り上げ。**re-pinは無し（代表＝常に最新）**。動画ソースは撮影/ギャラリー（プレビュー上は「↻ 動画を変更」）。
- **出品の削除 vs 停止で、農家が動画の残す/消すを制御**：
  - **停止** → 動画は保持（生産者ページのアーカイブに残る）。
  - **完全削除** → **その出品の動画も一緒に物理削除（道連れ）**。→ 削除確認モーダルに「削除すると…**この魚に投稿した動画もすべて削除されます**。残したい場合は「停止」を」を追加（mock反映済み）。
  - ＝「残したいなら停止・要らないなら削除」で農家に伝わる。orphan（飛び先のない動画）は作らない。

### 2. 商品に動画が付く（買い手）＝2階層
- **商品詳細「🎬 この魚の動画（◯本）」** ＝ その出品の動画を新しい順・横カルーセル（最大N本）。各サムネに「○時間前/○日前」。0本なら非表示。
- **生産者ページ「🎬 動画（◯本）＋すべて見る」** ＝ その農家の**全出品の全保持動画**を新しい順で合算・横カルーセル→「すべて見る」で縦3列グリッド。0本なら非表示。
  - **「販売中」バッジ**＝紐づく出品が販売中×在庫あり（**配達圏内は無視**）。売切れ/停止の出品の動画＝バッジ無しのアーカイブ。
  - **タップ**：販売中→全画面→「この魚を見る」で商品詳細（購入導線）／バッジ無し→全画面再生のみ＋「この魚は現在販売されていません」（mock `fishlink-reel-fullscreen-mockup.html`）。
- カード／Homeで動画がある出品に「🎬動画」バッジ（最新1本＝featured）。サムネは当面**写真を流用**。

### 3. Homeで発見（買い手）
- 最上部「🎬新着リール」カルーセル → 全画面リールフィード（縦スクロール・最新から）。**並び＝配達圏内×在庫あり×新着順**。0本ならセクション非表示。社交アイコン無し（質問は商品詳細のQ&Aへ）。
- **Homeフィードは1出品につきfeatured 1本だけ**（1農家1魚がフィードを埋めないように）。

## データ・表示条件（仕様書 §7）
- **動画は独立テーブル推奨**：`reel_videos { id, listingId, videoUrl, thumbUrl, postedAt, fishType }`。listingId必須。`fishType` はdenormalizeでポートフォリオ表示を速く。
- **featured（カード/Home/商品詳細の先頭に出る1本）＝その出品の `postedAt` 最大**。
- **表示サーフェスと条件**：
  - **Home新着リール／カード🎬バッジ** ＝ `featured && 販売中 && 在庫>0 && 配達圏内`（1出品1本）。
  - **生産者ページ ポートフォリオ** ＝ その農家の全出品の全保持動画を `postedAt` 降順で合算。販売中×在庫あり→「販売中」バッジ（配達圏内無視）／それ以外はバッジ無しアーカイブ。
  - **商品詳細「この魚の動画」** ＝ その出品の保持動画を `postedAt` 降順（最大N本）。
- **上限N本**：出品ごとに超過分の最古を物理削除。**停止/売切れ→保持**（アーカイブに残る・再開でfeatured復活）。**完全削除→道連れ削除**。

## 補足
- これは**修正でなく新機能**。他の依頼（丸め・内臓処理・レビュー導線・納品日時等）より重い。
- 鮮度表記＝「○時間前に投稿」（事実。検証できない claim は不可）。
- モデレーション（動画内の連絡先等）は将来（仕様書 §10）。
