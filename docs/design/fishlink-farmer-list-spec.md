# FISHLINK 農家一覧画面 実装仕様（おすすめ農家「もっと見る」の着地）

実装の正本は **`fishlink-farmer-list-mockup.html`**（構造・レイアウト・ソート/魚種チップUIはそこを参照）。
本書はHTMLから読み取れない情報＝**データモデル・並び/フィルタ仕様・状態・遷移・画面の位置づけ**を補う。

### 実装方針の優先順位（他specと同じ）
- 構成・データ・並び/フィルタ・遷移 → 本書とモックHTMLの通りに実装する。
- 既存の農家カード（Home「おすすめ農家」のカード）を**そのまま流用**する。

---

## 1. 概要

- **入口＝Home「☆ おすすめ農家」セクションの「もっと見る ›」**。タップでこの画面に遷移。
- 目的＝**評価の高い農家を発見する**リスト（「おすすめ＝評価が高い順」）。農家カードを評価順に縦1列で並べる。
- **🔴 現状はミスマッチ**＝「もっと見る」が**魚一覧（商品カードのリスト）に着地**している（農家を見たいのに商品が出る）。これを**この農家一覧に直す**（§10）。
- **位置づけ＝子画面**（タブではない。§2）。
- **農家ブラウズは設計上は副次的**（リピート単位＝魚種であって農家ではない）。よって**軽い画面**にとどめ、重い絞り込みは付けない。魚種フィルタだけ付ける（§5の理由）。

---

## 2. 画面構成

ヘッダー（子画面）→ ソート＋魚種チップ行 → 件数表示 → 農家カード縦1列 → ボトムナビ。

- **ヘッダー**＝**戻る矢印（←）＋タイトル「⭐ おすすめ農家」**（＋🌐言語切替）。戻る＝Homeへ。
- **ボトムナビ＝ホームがアクティブ**（Homeから開く子画面のため。**魚一覧タブを点灯させない**＝今のミスマッチの一因）。
- **ソート**＝ドロップダウン（§4）。
- **魚種チップ行**＝横スクロールのチップ列（§5）。**文字のみ**（アイコンは付けない）。ルール＝「Home『魚から探す』タイル＝アイコン＋名前（発見の入口）／絞り込み系チップは文字だけ」。魚一覧の絞り込みチップも文字＝統一。
- **農家カード＝縦1列（全幅）**。2列にしない（カードが背が高く情報が多く、2列だとタグ/ボタンが潰れる。魚一覧が2列なのは商品カードが小さいから）。

### 農家カード（Home「おすすめ農家」カード流用）
カバー写真（GAqP農家は左上に緑「✓ GAqP」バッジ）＋円アバター（未設定はデフォルト）＋農家名＋「州・地区 ・ km」＋評価＋取扱魚種タグ＋「農家を見る →」（青枠ボタン）。
- 評価表記＝`★ N% (件数)`。**レビュー0件＝「新規」**（★%を出さない・L94の新規農家ルール）。
- 取扱魚種タグ＝その農家が扱う魚種（魚種フィルタ選択中は**該当タグを青で強調**）。

## 3. データモデル

```js
// 一覧の各農家
farmerListItem = {
  farmerId,
  name,                 // 表示名（農家名）
  avatarUrl,            // 未設定はデフォルトアイコン
  coverUrl,             // 養殖環境カバー写真（任意）
  province, district,   // 「州・地区」表記（首都=プノンペン/他=◯◯州）
  distanceKm,           // 自分との距離
  gaqp,                 // true なら ✓GAqP バッジ
  ratingPositivePct,    // 取引満足度の好評価率（生産者ページのheadlineと同じ軸）
  ratingCount,          // レビュー件数。0 なら「新規」表示
  fishKinds: [],        // 取扱魚種（魚種フィルタ・タグ表示の元）。固定12種＋その他
  deliverableRadiusKm,  // この農家が配達できる距離（§6の表示判定に使用）
}
```
- `ratingPositivePct` の軸＝**取引の満足度**（生産者ページ §4 と同じheadline／全画面統一）。
- `fishKinds` の定義＝その農家が**扱う魚種**（現在＋過去の出品から。今日売切れでもレパートリーに含む＝§5の「レパートリー」の意味）。

---

## 4. 並び（ソート）

ドロップダウン。選択肢は**2つだけ**＝`評価が高い順`（デフォルト） / `距離が近い順`。
- 評価が高い順＝`ratingPositivePct` 降順。**新規（ratingCount=0）は末尾**（実績がないため・ただし隠さず表示）。同率は件数多い順を推奨。
- 距離が近い順＝`distanceKm` 昇順（評価は無関係）。
- 価格順などは付けない（価格は商品ごとで農家単位だと意味が薄い）。

---

## 5. 魚種フィルタ（このフィルタだけ付ける）

- **UI＝上部の魚種チップ横スクロール行**（ボトムシートにしない＝1次元のみなので過剰）。固定12種（＋必要なら「その他」）。文字のみ。
- **挙動＝タップでその魚を扱う農家に絞り込み・もう一度タップで解除**（単一選択）。選択中チップは青、該当農家カードの**取扱魚種タグも青で強調**。
- **判定＝`fishKinds` に選択魚種を含む農家**を表示。
- **なぜ魚種だけ付けるか**＝この画面の目的（信頼できる農家を見つける）＋模型思想（リピート単位＝魚種 L8）に最も合う唯一のフィルタ。**魚種タイル経由の商品一覧とは別の仕事**＝商品一覧＝「**今日**その魚を出してる出品（今日の在庫）」／農家一覧＋魚種フィルタ＝「その魚を**扱っている農家**（レパートリー＝今日売切れでも常連供給先）」。供給不安定だからこそ「その魚を頼れる農家は誰か」を知る価値がある。地域・価格は付けない。

---

## 6. 状態分岐

### 配達範囲外の農家＝非表示
- **自分に配達できる農家だけ表示**（`distanceKm ≤ deliverableRadiusKm`）。買えない遠方農家（例：3679km）を出しても無駄足。魚一覧で商品が配達範囲外を非表示にするベースルール（list-spec §7）と同じ思想。

### 新規農家（ratingCount = 0）
- ★%を出さず「**新規**」表示。評価順では末尾だが**隠さない**（新規農家が埋もれて立ち上がらないのを防ぐ・L94）。

### 魚種フィルタで0件
- 空状態「**◯◯を扱う農家はいません**」＋ボタン（青）「**すべての農家を見る**」（＝魚種フィルタ解除）。魚一覧の0件空状態と同発想。

### 読み込み
- 農家が多い場合は魚一覧と同じ「もっと見る」方式に準ずる（MVPは件数が少ないので一覧表示でも可・実装判断）。

## 8. 色（統一ルール）

- ソートの選択／魚種チップの選択・該当タグ強調／「農家を見る →」ボタン＝**青**（操作・選択・CTA）。
- GAqPバッジ＝**緑**（認証専用）。評価★＝amber。
- 緑をCTA/選択に使わない（青＝操作の徹底）。
