# 【実装依頼】農家「注文確認」再設計 ＋「売上・取引」新画面（2026-06-11）

下道さん

農家アプリの**「注文確認」まわりを大きく作り替えました**。既存実装への再設計です。ポイントは2つ：(1) 注文確認リストを「やることが一目で分かる」形（A案）に、(2) 散らばっていた取引履歴と売上集計を**「売上・取引」1画面に統合**。

仕様の正本：
- 仕様書：**`fishlink-farmer-order-spec.md`**（§2B 注文確認リスト／§7B 売上・取引 が新設・§10 差分表に「2026-06-11再設計」）
- モック：**`fishlink-farmer-order-mockup.html`**（① 注文確認リスト[A案]／②③④⑤ 既存維持／⑥ 売上・取引[通常]／⑦ 売上・取引[辞退・キャンセル選択時]）

## 全体像（なぜ変えるか）
- 旧注文確認リストは**承認待ちの全詳細がインライン展開**＋配達日グルーピングで、肝心の**承認期限（過ぎると自動辞退＝失注）が埋もれていた**。→ 低リテラシー農家が「今やること」を一目で掴める**優先フィード型**に。
- **取引履歴**（注文確認のアコーディオン）と**売上確認**（マイページ）が別々で、過去の取引が2か所に散っていた。→ **「売上・取引」1画面に統合**（迷わない）。

## 修正② 売上・取引（新画面）＝モック⑥⑦ / spec §7B

注文確認の「取引履歴を見る」と、マイページの「売上確認」が、**この同じ1画面に着地**します。

| 要素 | 内容 |
|------|------|
| フィルタ | **「過去1か月 ▾」ドロップダウン＋「辞退・キャンセル」チップ** の2つだけ。期間＝本日/過去1週間/過去1か月/日付指定/すべて |
| サマリ | **受取額（手取り＝手数料を引いた後・青）／取引数／平均単価**。**送金待ちが>0の時だけ**「うち送金待ち ◯◯ ・ 本日夕方に送金予定」を小さく表示 |
| 取引リスト | **軽いカード**＝状態バッジ（送金待ち=amber／取引完了=グレー）・日時・魚名・数量・**レストラン名（青→レストランページ）**・金額＋`›`。**アクションは置かない**。新しい順・もっと見る |
| カードのタップ | **送金状況画面へ**（後述・送金状況/レビュー/問題報告はそこ） |
| 「辞退・キャンセル」選択時（⑦） | お金サマリを**「成立しなかった注文 N件（売上には含まれません）」**に切替＋リスト＝**辞退済み（赤・自分が断った＋理由）／キャンセル（灰・買い手が承認前にキャンセル）**。受取額・取引数に含めない。辞退とキャンセルは**1チップにまとめる**（3つに増やさない） |

> **＋ 買い手のキャンセル受領（承認前のみ）**：買い手は**農家の承認前なら注文をキャンセルできる**（承認後は不可）。キャンセル時＝**農家へプッシュ通知＋承認待ちから消す（承認期限停止）＋競合ロック**（同一注文で承認とキャンセルは二重成立しない・先に押した方が成立）。取引履歴に「キャンセル」（辞退＝農家都合とは別ラベル）。詳細は買い手依頼文 `fishlink-request-order-status.md` と対。

## 修正③ 送金状況画面に追記（既存・精算 spec）

軽いリストからアクションを外した分は、**既存の「送金状況」画面に集約**（新規画面は作りません）。

- **レビューを送信**＝この画面に置く（**副CTA・白地青枠**・**支払い確認後に解禁**）。
- **問題を報告**＝この画面に置く（**赤**・報告できる期間だけ＝精算の自動確定窓と連動）。

## 修正④ マイページ入口リネーム

- マイページの「**売上確認**」→「**売上・取引**」にリネーム（画面タイトルと一致）。サブ「売上金額・取引履歴」はそのまま。
- 注文確認側の「取引履歴を見る」は**そのまま**（動作リンクとして自然）。
