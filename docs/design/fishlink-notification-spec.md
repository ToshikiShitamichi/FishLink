# FISHLINK プッシュ通知 実装仕様

版: v1.0 / 2026-06-27

本書は**アプリ横断のプッシュ通知の単一の真実（SoT）**。発火条件・文言・タップ先（ディープリンク）・通知許可フローを定義する。お金まわりの通知は `fishlink-payment-spec.md` §7 にも要約があるが、**詳細は本書が正本**。関連mock：`fishlink-push-permission-mockup.html`（許可前のソフト確認）。

---

## 1. 概要・スコープ
- 対象＝買い手（レストラン）・農家への全プッシュ通知＋通知許可の取得フロー。
- 通知の本体UI（ロック画面/通知センター）は**OSが描画**＝こちらはタイトル/本文/アイコン/ディープリンクを渡すのみ。カスタムUIは**許可前のソフト確認**だけ（§6）。
- **スコープ外＝専用「やることリスト」（✓✓）は作らない**（2026-06-27確定）。理由＝対応すべき項目は「注文状況の対応が必要」＋各画面バッジ＋プッシュで足り、専用集約は二重で過剰（大手も利用者向けには専用やることリストを持たず、注文/取引リスト自体＋プッシュで対応）。**ヘッダーの✓✓アイコンは外す（🔔通知だけ残す）＝ヘッダー整理でまとめて実施（別タスク）**。

## 2. 設計原則
- **3層**：① 取引ライフサイクル（必須・都度・確実）／② コミュニケーション（会話・新着で都度）／③ 成長系（任意・最小限）。
- **① 取引系＝都度・確実**（お金/取引が動いた通知は落とさない＝案Bの安心の核）。**② 会話＝新着で都度**（将来、連投は集約検討）。**③ プロモの一斉送信はMVPでは送らない**（通知疲れ防止）。
- **タップで関係する画面に直接ディープリンク**（ホームに開いて探させない）＝全通知共通。
- **基本オン**（§6）。文言は§7。

## 3. ① 取引ライフサイクル通知

### 3-1. 買い手（レストラン）への通知
| 場面 | 文言（例） | タップ先 |
|---|---|---|
| 農家が承認 | 「大星農場が注文を承認しました」 | 注文状況（その注文） |
| 辞退／承認期限切れの自動辞退 | 「大星農場が辞退しました。51,775 KHRをウォレットに返金しました」 | 注文状況の辞退カード |
| お支払い未確認 | 「入金を確認できませんでした。もう一度お支払いください」 | お支払い未確認カード |
| 配送開始 | 「大星農場が配達に向かいました」 | 配送管理（チャット） |
| お届け（配送完了） | 「大星農場がお届けしました」 | 配送管理／お届け済み |
| 受取の問題の返金 | 「6,825 KHRをウォレットに返金しました」 | ウォレット |
| 取引完了（レビュー誘導） | 「お取引が完了しました。農家を評価しましょう」 | レビュー画面 |
| 出金完了 | 「23,500 KHRを返金しました」 | ウォレット |

### 3-2. 農家への通知
| 場面 | 文言（例） | タップ先 |
|---|---|---|
| 新しい注文（前払い済み） | 「新しい注文が届きました（承認期限 15:24）」 | 承認待ち詳細 |
| 承認期限10分前 | 「承認期限まであと10分です」 | 承認待ち詳細 |
| 承認期限切れの自動辞退 | 「承認期限が過ぎ、注文が自動的に辞退になりました」 | 取引履歴 |
| 問題が報告された | 「問題が報告されています（送金確認中）」 | 送金状況（確認中） |
| 送金済（レビュー誘導） | 「46,525 KHRを送金しました。取引相手を評価しましょう」 | 送金状況／レビュー |

### 3-3. ルール
- **A 複数農家＝農家ごとに都度（まとめない）**。注文は農家ごと独立なので、どの農家がどうなったか各通知で明確。
- **文言に必ず農家名を入れる**（単数でも複数でも統一）。お金が動く通知は**農家名＋金額**。
- **B 配送ステータスのプッシュ＝配送開始・少し遅れそう・到着 の3つだけ**（準備開始は送らない＝チャットのシステム行にのみ出す）。買い手の行動が変わる節目だけ。固定（運営ON/OFF設定は作らない）。
- **C 承認期限切れの自動辞退は農家にも1通**（失注に気づき次回早く承認・無言で消すより本人のため）。

## 4. ② コミュニケーション通知
| チャネル | 場面 | 文言（例） | タップ先 |
|---|---|---|---|
| 配送チャット | 相手の新メッセージ（文字/画像/音声） | 「大星農場さんからメッセージが届きました」 | 配送管理（チャット） |
| 運営チャット | 運営の返信 | 「運営からメッセージが届きました」 | 運営チャット |
| Q&A（農家へ） | 自分の出品に新しい質問 | 「新しい質問が届きました」 | その出品のQ&A |
| Q&A（質問者へ） | 自分の質問に農家が返信 | 「質問に返信が来ました」 | そのQ&Aスレッド |

### ルール
- **2者間だけに通知**（Q&Aは質問者↔農家のみ・他の閲覧レストランには送らない）。
- 文言＝**送り主＋「メッセージが届きました」**。中身プレビューはMVPでは出さない（シンプル＆安全）。
- **連投はMVPは都度**（将来、◯分内の複数は1通に集約）。
- **A フォアグラウンド抑制**：その会話を開いている間はプッシュを出さない（画面内に新着が出るため）。別画面/アプリ閉では鳴る。
- **B 配送チャット通知は配送完了で止まる**：配送チャットは承認後〜配送完了だけ開く（完了でUI閉・履歴はサーバー保持）。完了後の連絡は運営チャットに集約（直接やり取り＝中抜き防止）。
- **C 二重通知回避**：出金完了は①のお金通知1本にし、運営チャットの定型返信（②）ではプッシュを重ねない。

## 5. ③ 成長系通知
| 場面 | 文言（例） | タップ先 | 送る？ |
|---|---|---|---|
| 紹介ボーナス獲得（農家） | 「紹介ボーナス5,000 KHRを獲得しました（次回送金に上乗せ）」 | 紹介ページ | ✅ |
| クーポン獲得（買い手） | 「2,500 KHRクーポンをもらいました」 | 紹介ページ／ウォレット | ✅ |
| クーポン期限リマインド | 「クーポンの期限が近づいています（あと3日）」 | カート／紹介ページ | ✅ |
| 運営プロモ（一斉送信） | 「手数料ゼロキャンペーン中！」等 | Home | ❌ MVPは送らない |

### ルール
- **「自分に価値が増えた／期限が近い」だけ送る**（紹介ボーナス・クーポン獲得・期限）。プロモの一斉送信はMVPはHome/アプリ内バナーのみ（将来オフ設定とセットで追加）。
- **A クーポン期限リマインド＝期限3日前に1回**（未使用の人だけ・しつこくしない）。
- **B 「もらいました」の伝え方**：被紹介者（招待された新規）は登録直後でアプリを見ている → **アプリ内表示**（プッシュ冗長）／紹介元（招待した側）は相手の初回取引完了後でアプリ外 → **プッシュ**。原則＝通知は"アプリ外にいそうな時"だけプッシュ。

## 6. 通知許可フロー（基本オン）
mock: `fishlink-push-permission-mockup.html`（買い手版/農家版）

- **基本オン**（種類別オプトインは作らない＝迷わせない）。
- **二段構え**：いきなりOS許可を出さず、**自前のソフト確認（アプリ内）を挟む**。
  1. **自前ソフト確認**（中央モーダル）：🔔「通知をオンにしますか？」＋**役割別文言**（買い手＝「農家の承認や配達状況をお知らせします」／農家＝「新しい注文が来たらすぐお知らせします」）。〔オンにする〕（青・目立つ）／〔あとで〕（地味なテキスト）。〔オンにする〕の下に小さく**「※ 次の画面で『許可』を選んでください」**（OSダイアログの文言は変えられないので自前側で誘導・低リテラシー対策）。
  2. 〔オンにする〕を押した人にだけ**OSダイアログ**が出る（その瞬間に出る）。〔あとで〕では出さない（再度聞けるようOSの一発勝負を温存）。
- **聞くタイミング**：基本は**登録完了の直後**にソフト確認。〔あとで〕の人だけ、**買い手＝初回注文後／農家＝初回出品後**に再度ソフト確認（価値の瞬間に繰り返す＝オン率最大化）。
- **OSダイアログのタイミングはアプリ実装で制御可能**（＝「通知を要求する処理」を呼ぶ場所で決まる。現状は登録時に自動で呼んでいる→ソフト確認の後に呼ぶよう移す）。
- **OSダイアログは実質1回きり有効**（拒否されると以後はOS設定からのみ）。
- **強制許可はNG**（OS/ストア規約・ダークパターン回避）。良いタイミング×ソフト確認×役割別文言×何度でも聞ける、で実質オン率を最大化。
- 通知オフの手段＝MVPはアプリ内設定を作らない（OS設定で全オフは元々可＝安全弁）。

> ⚠️ **要確認（下道さん）**：FISHLINKはWebアプリ（web.app）なので、Webプッシュの実現方式（FCM/Webプッシュ／PWA前提か・**iOS SafariはPWAをホーム画面に追加しないとWebプッシュ不可**等）を確認。タイミング制御の前提として要。

## 7. 文言の原則
- ① 取引系＝**農家名を必ず入れる**（単数/複数統一）。お金通知は**農家名＋金額**（誰の分がいくら）。
- ② 会話系＝**送り主＋「メッセージが届きました」**（中身プレビューなし）。
- タップ先は**対応すべき画面に直接**。
- **多言語**（日本語/クメール/英語）＝ユーザーの言語設定に合わせる（他spec共通）。

## 8. 実装メモ
- ディープリンク先は §3〜§5 の「タップ先」列。
- 連投＝都度（MVP）。フォアグラウンド抑制（開いている会話は出さない）。出金完了の二重通知回避（①優先）。
- 配送ステータスのプッシュ対象は固定3つ（配送開始/遅れ/到着）。
