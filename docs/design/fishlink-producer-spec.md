# FISHLINK 生産者ページ（農家プロフィール）実装仕様

実装の正本は **`fishlink-producer-mockup.html`**（構造・レイアウト・配置はそこを参照）。
本書はHTMLから読み取れない情報＝**データモデル・評価指標・空状態・遷移・制約**を補う。

> 既存実装は構成が良好（評価★%・3軸メトリクス・GAqP・出品中の魚・レビューコメント・養殖環境）。差分は §9。

### 実装方針の優先順位（他specと同じ）
- 構成・データ・指標・状態・遷移 → 本書とモック通り。
- 見た目（色・タイポ・アイコン）→ 既存DSに準拠。価格＝青／GAqP・成功＝緑／danger＝赤。

## 1. 概要

買い手が**商品詳細の農家名タップ**で開く、農家のプロフィール／実績ページ。信頼判断（評価・GAqP・養殖環境）と、その農家の**出品中の魚**の回遊に使う。

- ドメイン制約は共通（KHR / head/kg / GAqP=農家単位 / 供給不安定）。
- **農家の連絡先（電話番号等）は出さない**（中抜き防止＝Q&A spec §6と整合。問い合わせは商品ごとの公開Q&A経由）。

## 2. 画面構成（上から）

ヘッダー（戻る＋ロゴ＋言語）→ タイトル「生産者ページ」→ プロフィール → 生産者の一言（任意）→ 認証情報（GAqP農家のみ）→ 養殖環境（任意）→ **🎬 動画（任意・ポートフォリオ）** → レビュー（3軸メトリクス）→ 出品中の魚 → レビューコメント。

| 要素 | 内容 | 空/条件 |
|------|------|--------|
| プロフィール | アバター／農家名／**★好評価率%（件数）**／📍州・地区／**取引◯件 ・ YYYY年M月〜** | アバター未設定＝**デフォルトアイコン**（非表示にしない）／場所は**テキストのみ・Mapリンク無し**／取引数・登録時期＝レストランページと対称（§4） |
| 生産者の一言 | 農家の一言（**中立グレーのボックス**） | **未記入ならセクションごと非表示**。緑にしない（緑＝GAqP/成功専用・3軸をニュートラル化したのと同じ理由） |
| 認証情報 | **薄緑のボックスで囲む**（✓ GAqP 緑ピル＋説明文を1つの認証カードに） | **GAqP農家のみ表示**。非認証は非表示。ヘッダー下のcompactバッジは出さない（ボックスに一本化）。**緑はGAqPの正しい色＝ここは緑でOK**（一言は灰／3軸はニュートラル＝緑をGAqPに集約） |
| 養殖環境 | 農家の池/環境の写真。**複数枚＝横スワイプのカルーセル＋ドットインジケーター**（商品詳細の写真カルーセルと統一・縦積みにしない） | **未登録ならセクションごと非表示** |
| 🎬 動画（ポートフォリオ） | この農家の**全動画**を新しい順・横カルーセル＋「すべて見る」。**販売中バッジ**の出し分けあり（§4B） | **0本ならセクションごと非表示**（養殖環境と同じ空状態）。`fishlink-reels-spec.md` v2.0 |
| レビュー | 「レビュー（◯件）」＋**3軸メトリクス**＋「レビューを見る」 | §4 |
| 出品中の魚 | **標準商品カード**の2列グリッド | §5 |
| レビューコメント | 最新3件＋「もっと見る（残り◯件）」 | §6 |

  ratingCount,                // 総レビュー件数。0→「新規」
  tradeCount,                 // 完了取引の累計（信頼ブロック・「取引◯件」）。0→「新規」

  hitokoto,                   // 生産者の一言。空→非表示
  envImageUrls,               // 養殖環境の写真。空→セクション非表示

  listings: [ /* §5：standard 商品カード（在庫0は末尾グレー） */ ],
  reviews: [ { reviewerName, thumb /* good|bad */, text, createdAt } ],  // §6
}

## 4. 評価（★% と 3軸メトリクス）

- **ヘッダーの★%＝「取引の満足度」の良かった率**（3軸のうち取引満足度を headline に採用。全画面のカード★%と同じ軸で統一）。
  - 表示 `★{取引満足度%}（{総レビュー件数}件）`。**0件＝「新規」**（％を出さない）。
  - 理由：取引の満足度＝買い手の総合的な「満足したか」の判定で、単一の信頼シグナルに最適。品質/配送だけだと偏る。内訳は本ページの3軸で確認できる（総合＝headline・内訳＝プロフィール）。
- **3軸メトリクス**＝取引の満足度／魚の品質／配送時間 の**良かった率%**。「レビュー（◯件）」ブロックに3つ並べる。**3つとも同じニュートラルカード**（色をバラバラにしない＝青/緑/amberの意味色と衝突させない）。
- **信頼ブロック（★%の下の1行）**＝`取引◯件 ・ YYYY年M月〜`（取引数＝完了取引の累計＝主役の信号／登録時期＝副次）。**0件＝「新規」**。レストランページの信頼ブロックと対称。**“率”（支払い遅延率等）は出さない**（小サンプルで誤誘導・レストランページ §5 と同思想）。
- 各軸%の算出方法・各レビューが3軸を個別評価するか／コメントの単一👍👎が取引満足度かは、**レビュー送信フロー（別画面・未着手）設計時に確定**（本書では headline=取引満足度 のみ確定）。

## 4B. 🎬 動画（ポートフォリオ）

`fishlink-reels-spec.md` v2.0（2026-07-09 ポートフォリオ化）の**生産者ページスコープ**。mock＝`fishlink-producer-mockup.html`（インラインの動画カルーセル）／`fishlink-producer-reels-mockup.html`（＋「すべて見る」グリッド）。

- **中身＝この農家の全出品の全保持動画を `postedAt` 降順で合算**（＝出品ごとの最大N本の合算・reels-spec §7）。役割＝「この農家、ずっと本物の魚を上げてる」の**実績アーカイブ**（Homeリール＝最新1本の宣伝、とは役割が別）。
- **見出し＝「🎬 動画（◯本）」＋「すべて見る」**。トップは**縦サムネの横カルーセル**、「すべて見る」→**別画面の縦3列グリッド（新しい順）**の全動画画面（＝インライン展開ではない・動画が増えても一画面で見渡せてスケールする。実機がインライン展開なら別グリッド画面に直す＝2026-07-12）。
- **カルーセルの各動画カードに農家名は出さない**（＝農家自身のページなので冗長。Homeリールカードの流用で農家名が付いていたら外す・2026-07-12）。残す＝サムネ＋▶＋秒数／魚名／価格／販売中の表示。
- **「販売中」バッジの出し分け**：動画の紐づく出品が**販売中×在庫あり→青「販売中」バッジ**（**配達圏内は無視**＝買い手ごとにバッジがチラつくのを防ぐ）。**売切れ/停止の出品の動画＝バッジ無し**（過去実績のアーカイブとして出す）。
- **タップ挙動**：**販売中→全画面再生→「この魚を見る」で商品詳細（購入導線）**／**バッジ無し（売切れ/停止）→全画面再生のみ＋「この魚は現在販売されていません」**（買えないので商品詳細へ飛ばさない）。※出品の完全削除で動画も消える（reels-spec §6）ので、飛び先のないorphan動画は存在しない。
- **0本ならセクションごと非表示**（養殖環境と同じ空状態ルール）。
- サムネ各種メタ＝魚種名＋「○時間前/○日前」（reels-spec §8）。

## 5. 出品中の魚（グリッド）

- **標準の商品カード**（魚一覧 spec と同じ）を2列グリッドで表示。価格は**画像下**（オーバーレイにしない）。
  - **全部同じ農家**なので、商品カードの**農家フッター（アバター＋名＋評価）は省略**（冗長）。
  - 表示：画像＋魚名＋サイズ＋**価格（青・買い手向け単一の正＝手数料込み・内臓処理込み）**＋**在庫**。
  - **在庫は価格下のメタ行**（`在庫 ◯kg`）に出す。**画像オーバーレイにしない**（魚一覧の標準カードと同じ＝在庫はメタ行）。※売切れの「売切れ」バッジは画像上に残してよい（状態バッジ）。
  - この農家のページなので **距離・送料は出さない**（同一農家＝距離は共通でヘッダーに既出）。
- **価格は全画面で単一の正**（カード／詳細／カート／質問／生産者で同一商品＝同額）。生産者グリッドが農家価格を参照していたら買い手価格参照に修正。
- **売切れ（在庫0）＝グレーアウトして末尾固定**（魚一覧 spec §7と同じ）。「売切れ」バッジ。
- タップでその出品の商品詳細へ。

## 6. レビューコメント

- **最新3件を表示**＋**「もっと見る（残り◯件）」でインライン展開**（公開Q&Aの「もっと見る」と同パターン）。**専用の全レビュー画面は作らない**（将来レビューが大量になったら👍/👎・軸フィルタ付き画面に格上げ）。
- 「レビュー（◯件）」ブロックの「レビューを見る」＝**このレビューコメント節へスクロール**（同ページ内アンカー。現状維持）。
- 各コメント＝評者名／**線画スマイリー**（笑顔＝良かった／困り顔＝残念だった。**絵文字👍👎は使わない**＝レビュー入力 spec と統一）／日付／本文。新しい順。
- **コメント件数 ≠ レビュー件数**：「レビュー（◯件）」＝レビュー総数（評価の母数・★%の分母）／「レビューコメント（◯件）」＝**テキストコメントを書いた件数**（全員がコメントするわけではない＝コメント数 ≤ レビュー数）。**「もっと見る（残り◯件）」はコメント件数基準**。
- 初期表示件数（3）は調整可能パラメータ（後で3〜5に微調整可）。

## 10. 農家側の編集画面（マイページ →「生産者ページ」）

本ページ（買い手が見る公開ページ）の**中身を農家が編集する管理画面**＝農家マイページの「生産者ページ」メニュー。※本格的な仕様（プロフィール全体）はプロフィールレビュー（後日）で。ここは色・構成の確定事項のみ記録。

- **構成**：上部「**公開ページをプレビュー**」（自己プレビュー＝§の公開ページをread-onlyで表示＋バナー「👁 これは買い手から見えるあなたのページです（編集は「生産者ページ」から）」）／顔写真（写真を選択・削除）／生産者の一言（textarea）／養殖環境（**複数写真を ✕削除＋＋追加で管理**）／キャンセル・保存。
  - ※バナーの「編集は〇〇から」は**マイページの実メニュー名に合わせる**（農家＝「生産者ページ」／レストラン＝「レストラン情報」）。旧「プロフィールから」は実在しないメニュー名で紛らわしいため廃止（2026-07-05）。実機（農家編集画面）はモック未作成＝spec正本＋下道さんに文言修正を依頼。
- **色（確定 2026-06-16）**：**「編集」ボタン＝青**（紫にしない）／**「保存」ボタン＝青**（緑にしない・CTA青／キャンセルはグレー）／「公開ページをプレビュー」＝情報系の淡色でOK。＝送金先情報の「編集 紫→青／保存 緑→青」と同じ青ルール。
- 用語「**顔写真**」（農家）はレストラン側「アイコン写真」と文脈が違うため揃えない（農家＝顔で信頼／店＝看板）。
- 実装OK＝公開ページをプレビュー／自己プレビューバナー／養殖環境の複数写真管理（✕・追加）。要修正＝編集/保存ボタンの色（青へ）。
