【モック＝取引一覧の "2軸ステータス" 版（1-6）】 前払い化で、取引は「配送の進行」と「お金（入金・送金）」が並行して進むため、ステータスを2軸に分けています。 ＝1件が「配送＝配送開始／入金＝入金確認待ち」のように2つ同時に持てる（1本の直線では表せなかった状態）。フィルタも2軸＝運営のTo-Do（入金確認待ち・農家送金待ち）が絞れます。
※入金確認待ちは「注文（前払い）した瞬間」に始まる＝農家の承認を待たない（配送軸の「農家確認中」と同時スタート）。農家が承認すると配送軸が動き、運営が着金を確認すると入金軸が動く＝別タイミングで進む。※状態名・色は叩き台。
