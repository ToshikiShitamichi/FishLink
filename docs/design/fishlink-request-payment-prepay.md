## 全体像（新しいお金の流れ）
```
① 注文時にレストランが前払い（KHQR・ウォレット残高があれば自動充当＋残額KHQR）
② FISHLINKが預かる（エスクロー）・運営が入金を照合
③ 即農家へ → 農家が承認 / 辞退
④ 配送 → 配送完了 → 数時間後に自動で取引完了
⑤ 取引完了後、農家へ送金（手数料引き・配送完了後＝不変）
　 ※辞退 / 受取の問題 / 承認前キャンセル → 買い手へ「即ウォレット返金」
```
**ポイント**：払う瞬間が「受取時 → 注文時」に早まっただけ。**農家の受取額・送金ロジック・タイミング（配送完了後）は不変**。返金は銀行振込でなく**ウォレット残高に即返金**（次回注文で自動充当）。

## 1. 買い手｜支払い（前払い・KHQR）＝**新規**
モック：`fishlink-payment-prepay-mockup.html`（6状態）

- 注文確定時に**前払い**。構成：注文内容の確認 → 農家ブロック（read-only）→ お支払い明細（**合計（カートより）→ 💰ウォレット利用 → お支払い額**）→ 🔒エスクロー安心文 →〔KHQRで支払う ◯KHR〕。
- **ウォレット充当＝既定で全額自動**＋〔変更〕で金額指定（0＝使わない）。残高で全額まかなえれば **KHQR不要**〔注文を確定する〕。
- **⑤KHQR画面に支払い手段3ボタン**：📋金額をコピー（額の横）／🏦銀行アプリで開く（主役）／🖼QR画像を保存。お支払い額の下に差引1行「合計 ◯ − 💰ウォレット ◯ を差し引き済み」。
- 〔支払いました〕→ 確認モーダル → **即農家へ**（状態＝支払い確認中・運営が並行で入金確認）。

## 2. 買い手｜ウォレット＝**新規**
モック：`fishlink-wallet-mockup.html`（4状態）・入口＝マイページ「💰ウォレット 残高◯」行

- **残高（大きく青）**＋ナッジ「💡 次回の注文で自動的に使われます／支払い時に使う金額を選べます」。残高0では空状態。
- **利用履歴4種**：辞退の返金（+緑）／受け取りの問題の返金（+緑）／注文に使用（−）／残高の返金=出金（−）。各行＝日付・理由・相手（農家名 or FL番号）・金額。
- **〔残高の返金を申請〕**（控えめリンク・残高>0時のみ）→ 確認シート（返金額＝残高ぜんぶ／返金先／1〜3営業日）→〔運営に申請する〕→ **運営チャットへ（残高は即減算）**。返金先未登録なら〔返金先を登録する〕（JIT）・申請ボタンは無効。

## 共通
- 色ルール：**金額＝青 `#0B5FB0`／割引（キャンペーン・クーポン）＝amber／手数料・送料＝中立グレー／CTA＝青／緑はGAqP・成功・「返金済の安心」専用／赤はdanger専用**。全画面に**FL番号**。
- **返金は即ウォレット**（辞退・承認前キャンセルは自動／問題報告は運営判断後）。「お金が動いた」通知（返金・送金）は確実に出す。
- **クーポンはカート全体に1回**（農家に割り当てない）。複数農家の1農家辞退＝その農家の小計を満額返金・割引は残った農家に残る／全農家辞退＝全額返金＋クーポン未使用で戻る（仕様書 §4.3）。
- 出金（残高の返金）は**全額のみ・運営チャット申請・申請で残高即減算**（仕様書 §6.4）。

## 確認したいこと（重要）
1. **決済の自動化（fast-follow）**：今は個人口座QRで運営が手動照合。前払いだと件数が増えるので**入金の自動照合が必須**。ABA PayWayマーチャント／Bakong で〈動的QR＋入金webhook＋送金API〉が組めるか（技術・費用・審査）。※代表（橘）からもABAへ問い合わせます。
2. **預かり金（前払い金・ウォレット残高）の規制・信用面**：カンボジアの電子マネー／決済ライセンス等に当たるか＝法務・運営で要確認（実装前提に影響）。
3. **レストランが前払いを嫌がらないか**：設計では即ウォレット返金・出金申請・エスクロー安心文で緩和。実店舗数店にヒアリング予定。
