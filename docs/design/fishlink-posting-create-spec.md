# FISHLINK 農家側 出品フォーム画面 実装仕様

実装の正本は **`fishlink-posting-create-mockup.html`**（構造・レイアウト・セクション・入力欄・配送サマリー・ボトムシートはそこを参照）。
本書はHTMLから読み取れない情報＝**データモデル・必須/任意・デフォルト値・状態分岐・遷移・状態保持・価格モデル**を補う。

### 実装方針の優先順位（他specと同じ）
- **構成・データ・状態・遷移** → 本書とモックHTMLの通りに実装する。
- **見た目のスタイル**（色・余白・角丸・影） → **既存アプリのデザインシステムに準拠**。モックの色値は役割を示す参考値（プライマリ青 `#0B5FB0`＝価格/CTA / GAqP・参考価格緑 `#2E7D32` / サブ文字 `#5A6470` / danger赤 `#E5453F` / 警告アンバー `#B36A00`+`#FFF7E6`）。

---

## 1. 概要

農家（売り手）が**魚1種類を1投稿として出品する**フォーム。投稿一覧画面（`fishlink-posting-list-spec.md`）のボトムナビ「出品する」から遷移する。
ドメイン制約は共通（養殖のみ / KHR / head/kg / GAqP は農家単位 / 供給不安定で売切れ頻発 → 同じ魚を複数農家から買う＝**繰り返し単位は魚種**）。

**設計の核**: 農家は同じ魚を何度も投稿する。魚ごとに変わらない情報（＝配送条件）を毎回入力させない。**配送は農家ごとの設定**として一度きり登録し、全投稿に自動適用する（§5）。

**画面の性格**: ヘッダーは「← ＋ 魚を投稿する」だけ（FISHLINKロゴ・ボトムナビなし）＝集中タスク画面。完了/離脱で投稿一覧へ戻る。

---

## 2. 画面構成

ヘッダー（戻る＋タイトル）→ スクロールするフォーム本体（セクション縦積み）→ 固定CTA「投稿する」（青）。

セクションは固定順（モック準拠）:

1. **基本情報** — 魚種 / サイズ / 在庫数量 / 写真
2. **販売価格** — 魚単価 / ＋内臓処理 / 販売価格 計算ボックス / 価格の式 / CAA参考価格ボックス
3. **配送**（レストラン負担・全出品に共通）— 配送設定の有無で表示が分岐（§5）
4. **取引条件**（任意）— 最低注文量
5. **商品の説明**（任意）— ガイド＋虚偽注意（B案）＋テキストエリア

- CTA「投稿する」は**青 `#0B5FB0`**（価格と統一。発注/購入のオレンジは買い手側用途なので使わない）。

## 3. データモデル

投稿で生成・更新される実体は2つ: **posting（魚1種＝1投稿）** と **農家の配送設定（farmer単位・1個）**。

```js
// 投稿（魚1種）。投稿一覧spec §3 の posting と同一実体
posting = {
  id,
  fishKind,        // "ヒレナマズ" | … | "バラマンディ" | "その他"（CAA標準11種固定＋「その他」のselect。自由入力不可）必須。※11種目=バラマンディ（CAA価格表に追加・固定種化 2026-06-01）
  fishName,        // 表示名。標準種は fishKind と同じ。「その他」選択時のみ自由入力（将来）
  sizeHeadPerKg,   // "6-8"（head/kg。範囲も可。文字列）必須
  stockKg,         // 50（在庫kg。注文で自動減算→0で自動「売切れ」）必須
  imageUrls,       // 写真（複数枚可）必須・最低1枚
  priceKhrPerKg,   // 5000（魚単価, KHR/kg）必須 ← ここが農家の入力値
  gutFeeKhrPerKg,  // 500（内臓処理の追加料金, KHR/kg）デフォルト500・編集可
  minOrderKg,      // 任意。これ未満の注文は受付不可。空＝制限なし（null）
  description,     // 任意。商品の説明。空可
  status,          // "selling" 等は投稿一覧spec §4 で導出
  createdAt,
  farmer: { name, avatar, gaqp }  // gaqp=農家単位
}

// 農家の配送設定（farmer単位で1つ。全postingに共通適用）
farmerDelivery = {
  deliverableRadiusKm,  // 30（配送可能距離。これより遠いレストランには非表示）必須
  deliveryFeePer10km,   // 5000（10kmあたりの送料。レストラン負担）必須
  freeDeliveryKm        // 5（送料無料距離。空=0=無料圏なし）任意
}
```

- **配送は posting に持たせず farmer 単位**（§5）。買い手側 product 表示時に farmerDelivery を参照して配送可否・送料を計算する。
- **販売価格はフォームに保存しない**（魚単価＋内臓処理から都度算出する導出値）。

### 価格モデル（確定 2026-05-31）

```
販売価格（農家受取ベース） ＝ 魚単価 ＋ 内臓処理
  例：5,000 ＋ 500 ＝ 5,500 KHR/kg

買い手カード表示価格 ＝ 販売価格 ×（1 ＋ 手数料率）
  例：5,500 × 1.05 ＝ 5,775 KHR/kg
```

- **手数料率は現状5%だが変動しうる**（固定値としてハードコードしない）。
- **内臓処理はデフォルト込み**。レストランが注文時に外すと −内臓処理（注文画面で選択）。
- **このフォームには手数料・買い手表示価格を出さない**（農家視点を混乱させないため）。価格ボックスに出すのは「魚単価＋内臓処理＝販売価格」まで。

## 4. 必須/任意 とデフォルト値

| フィールド | 必須/任意 | 初期値 | 理由 |
|---|---|---|---|
| 魚種 | 必須 | 未選択（「選択してください」） | 選ばせる |
| サイズ head/kg | 必須 | 空（placeholder 例：6-8） | 投稿ごとに異なる |
| 在庫数量 kg | 必須 | 空（placeholder 例：50） | 投稿ごとに異なる |
| 写真 | 必須・最低1枚 | 空 | 信頼の土台 |
| 魚単価 | 必須 | **空**（placeholder 例：5,000） | 古い価格を使い回すと害。意識的に入力させる |
| ＋内臓処理 | 任意（実質既定運用） | **500 をプリフィル** | ほぼ全農家共通の慣行値。編集可 |
| 配送可能距離 | 必須 | 初回のみ空（placeholder 例：30） | 配送設定の核（§5） |
| 10kmあたりの送料 | 必須 | 初回のみ空（placeholder 例：5,000） | 同上 |
| 送料無料距離 | 任意 | 空（placeholder「0（無料圏なし）」） | 無料圏を作る農家だけ入力。空＝全配達課金 |
| 最低注文量 kg | 任意 | 空（placeholder「指定なし」） | 設けない農家が多い。空＝制限なし |
| 商品の説明 | 任意 | 空（文例は**placeholder のみ**） | §6 |

**デフォルト値の原則**: 安全な唯一の意味があるときだけ空（最低注文量＝空＝制限なし）。使い回すと害になる値（魚単価）は空＋必須で都度入力を強制。慣行が固まっている値（内臓処理500）はプリフィル。

### 必須未入力時のバリデーション（インラインエラー方式・確定 2026-06-01）
- 「投稿する」押下時に必須が空なら、**空の各必須フィールド直下に赤メッセージ「○○を入力してください」＋枠を赤**にする。
- **最初のエラー欄へ自動スクロール**する。
- **トップに単一エラーバナーを出す方式は使わない**（1件ずつしか直せず、該当欄から遠くて不親切）。出すなら「未入力の項目があります」の総括のみで、個別指摘は各フィールドで行う。
- **「投稿する」は常に押下可**（無効化しない＝押せない理由が分からないため）。押下時に検証する。
- 任意項目（内臓処理500／送料無料距離／最低注文量／商品の説明）はエラー対象外。
- 販売価格の計算ボックスは、**魚単価が未入力のとき「0 KHR/kg」ではなく「魚単価を入力してください」**（空状態表示）にする。

## 5. 配送＝農家ごとの設定（A案・確定）と初回/2回目以降の分岐

配送3パラメータは魚ごとに変わらない農家の属性。**一度設定すれば全投稿に自動適用**、変更も1箇所で全投稿に反映する。

> **採用理由（B案＝毎回プリフィルとの比較）**: B案は「前回値を持ってくる」だけなので、範囲を変えたいとき全投稿を編集して回る必要がある。A案は farmerDelivery を1つ持ち、変更が全投稿に一括反映される＝一元管理。配送条件は本質的に魚ではなく農家に紐づく属性なのでA案が正しい。

### 初回（farmerDelivery 未設定）— モック①

- 配送セクションに**3つの入力欄**を表示: 配送可能距離[必須] / 10kmあたりの送料[必須] / 送料無料距離[任意]。
- ヘルパー「この配送条件は**すべての出品に共通**で保存されます。次回からは入力不要です。」
- 「投稿する」で posting と同時に **farmerDelivery を保存**。

### 2回目以降（farmerDelivery 設定済み）— モック②

- 配送の入力欄は**消す**。代わりに適用される設定を**サマリー1行**で表示:
  `🚚 30kmまで配達 ・ 5km以内は送料無料 / 10kmあたり 5,000 KHR` ＋ 右に **〔変更〕**リンク。
- ヘルパー「配送条件は全出品に共通です。「変更」するとすべての出品に反映されます。」
- このフォームでは配送を再入力させない（投稿は farmerDelivery を参照するだけ）。

### 〔変更〕＝ボトムシート（A案・確定）— モック③

- 〔変更〕タップで**ボトムシートが下から出現**。背後の入力中フォームは**保持されたまま**上に重なる（rgba(0,0,0,0.4) オーバーレイ）。
- 上部に警告（アンバー）: 「⚠️ 配送条件は全出品に共通です。変更すると現在の **N件** すべてに反映されます。」N＝現在の投稿数。
- 3フィールドは現在の farmerDelivery を**プリフィル**（30 / 5,000 / 5）。
- アクション: 〔キャンセル〕（グレー・破棄して閉じる）/ 〔保存して全出品に反映〕（青・farmerDelivery 更新→全投稿に即反映→シートを閉じてサマリー行を更新）。
- **買い手側フィルターのボトムシートUIと同じ作法**（一貫性）。

### 配送料の計算（A案・確定／買い手側で適用）

```
distanceKm ≤ freeDeliveryKm        → 送料 0
freeDeliveryKm < distanceKm ≤ radius → 送料 = deliveryFeePer10km × (distanceKm / 10)（0からの比例）
distanceKm > deliverableRadiusKm   → 配達不可（買い手側で非表示）
```

- 送料は**配達1回の固定額・注文量では変わらない**（量依存の段階送料は作らない）。
- 「大口注文は送料無料」は**未実装・農家ヒアリング後に判断**（任意1フィールドで対応しうるが、B2Bでは手数料が大口で相対的に小さく、最低注文量で小口も制御できるため必要性は弱い）。

---

## 6. 商品の説明（B案・確定）

- セクションは**任意**。
- 上部に書く観点のガイド（品質/安全性/餌/池の状態/おすすめ）を箇条書きで提示。
- その下に**虚偽注意（B案）**: アンバーの `warn-note`「⚠️ 事実だけを書いてください（虚偽は信頼を損ないます）。」
- 文例は**placeholder のみ**で表示（プリフィルしない）。プリフィルすると農家がそのまま投稿し、事実でない記述＝虚偽表示になりうるため。

## 7. 画面遷移

| 要素 | 遷移/挙動 |
|---|---|
| ← 戻る | 投稿一覧へ戻る（入力破棄。下書き保存は未実装） |
| 〔変更〕（配送） | ボトムシートを開く（§5）。フォーム入力は保持 |
| シート〔キャンセル〕 | 変更を破棄してシートを閉じる |
| シート〔保存して全出品に反映〕 | farmerDelivery 更新→全投稿に即反映→閉じてサマリー更新 |
| 投稿する（CTA） | バリデーション→ posting 保存（初回は farmerDelivery も保存）→投稿一覧（販売中）へ |

---

## 8. 状態の保持

- 〔変更〕ボトムシートを開閉しても、フォームに入力中のデータ（魚種・価格等）は保持する。
- 初回投稿で farmerDelivery が保存されたら、以降の出品フォームは自動で「2回目以降」表示（サマリー行）になる。
- farmerDelivery の変更は、保存と同時に既存の全 posting の買い手側表示（配送可否・送料）に反映される。
