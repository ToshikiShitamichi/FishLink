# FISHLINK レストランページ（レストランプロフィール）実装仕様

実装の正本は **`fishlink-restaurant-mockup.html`**（構造・レイアウト・配置はそこを参照）。
本書はHTMLから読み取れない情報＝**データモデル・評価指標・信頼ブロック・空状態・遷移・制約**を補う。

> 生産者ページ（`fishlink-producer-spec.md`）の**対**。既存実装あり（構成は良好）。差分＝要修正は §12。

### 実装方針の優先順位（他specと同じ）
- 構成・データ・指標・状態・遷移 → 本書とモック通り。
- 見た目（色・タイポ・アイコン）→ 既存DSに準拠。価格＝青／GAqP・成功＝緑／danger＝赤／キャンペーン・割引＝amber。

---

## 1. 概要

農家が**承認待ち詳細（注文確認）でレストラン名タップ**で開く、レストランのプロフィール／実績ページ。生産者ページのミラー。

- **用途＝農家が「誰に魚を回すか＝未払い／受取トラブルのリスク判断」**。承認の意思決定を支える信頼確認画面。
- 「公開」＝世界でなく**相手の農家に見える**意味（承認時に見る）。
- ドメイン制約は共通（KHR / head/kg / 供給不安定）。
- **レストランの連絡先（電話番号等）は出さない**（中抜き防止＝Q&A spec §6と整合）。
- **地図は出す**（§7。生産者ページはテキストのみだが、レストランページは農家向け＝配達先の地図なので地図あり＝非対称は原則どおり）。

---

## 2. 画面構成（上から）

ヘッダー（戻る＋ロゴ＋言語）→ タイトル「レストラン情報」→ プロフィール（アイコン＋店名＋★%＋メタ行）→ 地図 → お店の紹介（任意）→ お店の様子（任意）→ レビュー（3軸メトリクス）→ レビューコメント。

| 要素 | 内容 | 空/条件 |
|------|------|--------|
| プロフィール | **アイコン写真**（円形アバター）／店名／**★好評価率%（件数）**／メタ行＝**📍地区 ・ 取引N件 ・ YYYY/M〜** | アイコン未設定＝**デフォルトアイコン**（非表示にしない）。★0件＝「新規」。信頼ブロック＝§5 |
| 地図 | 埋め込み地図＋「Google Mapsで開く」 | **常に表示**（新規でも・場所は登録済み）。§7 |
| お店の紹介 | レストランの自己紹介（中立ボックス） | **未記入ならセクションごと非表示** |
| お店の様子 | 店舗の外観/内観の風景写真 | **未登録ならセクションごと非表示** |
| レビュー | 「レビュー（◯件）」＋**3軸メトリクス**＋「レビューを見る」 | §4。0件＝空状態 |
| レビューコメント | 最新3件＋「もっと見る（残り◯件）」 | §8。0件＝空状態 |

※**生産者ページから落とす要素**：出品中の魚（レストランは売らない）／養殖環境／GAqP認証（農家単位の属性）／生産者の一言（→お店の紹介に差し替え）。

## 3. データモデル

```js
restaurant = {
  id, name, avatarUrl,         // avatarUrl 無→デフォルトアイコン（店アイコン）
  province, district,          // 「タケオ州」「Samraong」。メタ行にテキスト表示
  lat, lng,                    // 地図ピン用（§7。地図は出す）
  ratingPositivePct,           // ★%＝総合評価の良かった率（§4）
  ratingCount,                 // 総レビュー件数。0→「新規」
  tradeCount,                  // 取引完了の累計件数（§5）。0→「新規」
  registeredAt,                // 登録時期。表示「YYYY/M〜」（§5）
  introText,                   // お店の紹介。空→セクション非表示
  sceneImageUrls,              // お店の様子の写真。空→セクション非表示
  metrics: { overallPct, communicationPct, receivingPct },  // 3軸（良かった率%・§4）
  reviews: [ { reviewerName /* 農家名 */, thumb /* good|bad */, text, createdAt } ],  // §8
}
```

---

## 4. 評価（★% と 3軸メトリクス）

- **ヘッダーの★%＝「総合評価」の良かった率**（3軸のうち総合評価を headline に採用。生産者ページの「取引の満足度」に相当）。
  - 表示 `★{総合評価%}（{総レビュー件数}件）`。**0件＝「新規」**（％を出さない・★は非赤）。
- **3軸メトリクス＝総合評価／やり取り／受取対応**（農家→レストラン評価の軸＝レビュー spec）。各**良かった率%**を「レビュー（◯件）」ブロックに3つ並べる。
  - **3軸は色をニュートラル統一**（薄グレーカード＋濃色数字）。青/緑/amber等の意味色を軸に割り当てない。
- 各軸%の算出・各レビューが3軸を個別評価するか／コメントの単一スマイリーがどの軸か は**レビュー入力フロー（`fishlink-review-spec.md`）に準拠**。

---

## 5. 信頼ブロック（取引数＋登録時期）★本ページの核

農家のリスク判断（未払い／受取トラブル）を支える基本情報。メタ行に1行：**「📍地区 ・ 取引 N件 ・ YYYY/M〜」**。

- **取引数（tradeCount）＝取引完了した注文の累計件数**（辞退・キャンセルは数えない）。**主役の信号**。
- **登録時期（registeredAt）＝アカウント作成**。表示「YYYY/M〜」（例 2025/12〜）。副次。
- **取引0件＝「新規」**（★%も「新規」表示。L94の新規農家と同思想）。
- **“率”指標（支払完了率／未払い率等）は出さない**。理由：
  1. 小サンプルで率は誤誘導（3件中1遅延＝67%でも「危ない客」と過剰表示→流通が死ぬ）。
  2. 取引数＝“生き残り”シグナルが率より頑健（未払い常習なら垢バン／取引続かない）。
  3. 危険客は農家に自衛させず**運営がアカバンで止める**設計（精算 spec・与信を握るのがMoat）。
  4. 受取トラブルの主観評価は3軸「受取対応」に既出。
  - **未払い等のデータは裏で貯めるがページ非表示**。必要になったら“率”でなく「⚠️過去に支払い遅延あり」の二値フラグ＋ダブルブラインド/晒し上げ議論（§13）と一緒に後で。
- **リアルタイムの「今支払い遅延中／問題報告中」警告も出さない**（運営アカバンで担保。考慮の上で非表示）。

---

## 6. 写真（アイコン＋お店の様子の2枠）

生産者ページ（アバター＋養殖環境写真）と対称。

- **アイコン写真（円形アバター）**＝店のロゴ/看板。**常に表示**（未設定はデフォルトアイコン）。
- **お店の様子（風景写真）**＝外観/内観。**任意・未登録ならセクションごと非表示**（生産者の養殖環境と同ルール）。
- 写真の登録・編集はレストランの**プロフィール画面**側（別途）。

---

## 7. 地図

- **埋め込み地図＋「Google Mapsで開く」を表示**。
- 理由＝**「誰向けの画面か」で決まる**：生産者ページ＝買い手向け→農家の場所を隠す（私的＋中抜き）／**レストランページ＝農家向け→レストランは公開店舗＋配達先**なので地図を見せる。`fishlink-chat-spec`（配送管理）§「農家側だけ地図リンクを出す＝レストランの配達先を開く用」と一貫＝“レストランの地図は農家向けツール”。
- カンボジアは距離10kmでも川/悪路でアクセス困難があり、承認時の配達可否判断に**kmより地図が実用的**。
- **新規レストランでも地図は表示**（場所は登録済み）。

---

## 8. レビューコメント

- **最新3件を表示**＋**「もっと見る（残り◯件）」でインライン展開**（生産者ページ・公開Q&Aと同パターン）。専用の全レビュー画面は作らない（将来レビュー大量で格上げ）。
- 各コメント＝**評者名（農家名）**／**良かった（笑顔）or 残念だった（困り顔）**／日付／本文。新しい順。
- **アイコンは線画スマイリー（笑顔/困り顔）**。**絵文字（👍👎）は使わない**（端末差/DS不一致＝レビュー入力 spec §5と統一）。良かった＝緑／残念だった＝赤。
- **連絡先マスク＋モデレーション**を適用（Q&A spec §6共通ロジック・両方向）。
- 「レビューを見る」＝このレビューコメント節へスクロール（同ページ内アンカー）。
- 初期表示件数（3）は調整可能パラメータ。

---

## 9. 自己プレビュー（レストラン本人ビュー）

- レストラン本人が**プロフィールから「農家に見えているあなたのページ」をread-onlyで確認**できる。自分の評判を把握→支払い/受取の振る舞いを自己修正するナッジ（農家に“率”を見せない代わりに本人に評判を見せて行動を正す）。生産者ページ（農家の自己プレビュー）と対称。
- 上部に案内バナー「**👁 これは農家から見えるあなたのページです（編集はプロフィールから）**」。
- **評者（農家）名は伏せず非匿名**（農家が見る①と同一）。報復の本丸対策は名前マスクでなく**ダブルブラインド（将来）**＋当面は運営監視（§13）。
- 編集は本ページでなくプロフィール画面側。

## 11. 制約

- **連絡先非露出**：レストランの電話番号・SNS等は本ページに出さない（中抜き防止）。
- **地図は出す**（例外＝農家向け＝配達先。生産者ページの「テキストのみ」とは§7の原則により非対称）。
- アクセス経路はMVP＝**承認待ち詳細からの1経路**（＋本人はプロフィールから自己プレビュー）。配送中/完了後など他経路は将来。

## 12. 既存実装との差分（要修正）

| 項目 | 旧 | 修正後 |
|------|----|--------|
| 信頼ブロック（取引数・登録時期） | **無し** | メタ行に「取引 N件 ・ YYYY/M〜」を**追加**。取引0件＝新規。**“率”は出さない**（§5） |
| 写真 | カバー写真1枚のみ | **アイコン写真＋お店の様子の2枠に分離**（様子は未登録で非表示） |
| 3軸メトリクスの色 | 軸ごとに橙/青/緑でバラバラ | **ニュートラル統一**（薄グレー＋濃色数字） |
| レビューコメント | 全件インライン | **最新3件＋もっと見るで展開** |
| レビューのアイコン | 👍/👎 絵文字 | **線画スマイリー（笑顔/困り顔）**・絵文字不可 |
| ★の色 | 赤 | **非赤**（赤=danger専用。生産者ページの★と揃える） |
| ★%表記 | `★ 93% 43件` | **`★93%（43件）`**（カッコ付き全画面統一） |
| お店の紹介（未記入） | （要確認） | **セクションごと非表示** |
| タイトル | 「レストラン情報」 | 維持可（生産者ページ「生産者ページ」との命名統一は軽微・後で） |

> 修正不要（実装OK）：**地図あり**（農家向け＝配達先・§7で正しい）／3軸の軸（総合評価/やり取り/受取対応）／連絡先非露出／全体の構成。
>
> **生産者ページへの波及（共通修正）**：(a)レビューの👍絵文字→線画スマイリー（生産者モックも👍のまま）(b)取引数を生産者ページにも対称追加。＝生産者ページの追加修正としても依頼に含める。
