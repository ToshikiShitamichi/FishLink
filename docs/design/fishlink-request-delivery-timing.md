# 【実装依頼】納品日・時間帯の選択ルールと承認期限（2026-07-05）

下道さん

カートの「納品日・時間帯」の選択ルールと、農家の「承認期限」を整理しました。**特に "夜間に注文すると、農家が寝てる間に承認期限が切れて自動辞退される" 問題の解消**が主目的です。正本＝`fishlink-cart-spec.md` §7・`fishlink-farmer-order-spec.md`（承認期限）。

## 変更点

### ① 最短リードタイム＝8h（現状維持／要確認）
注文から **8時間後** 以降のスロットのみ選択可（それ未満はグレーアウト）。※実機が既に8hなら変更不要。運営パラメータ。

### ② 最長リードタイム＝14日（新規）
**14日より先のスロットは選択不可**。理由＝生鮮＋供給不安定で先ほど確度が落ちる／CAA連動価格が古くなる／前払いが長く拘束される。宴会等の先の大口は運営相談ベース。運営パラメータ。

### ③ 承認期限を「注文＋1h」→「注文＋1.5h・夜間リセット」に（重要）
- **承認期限 ＝ 注文 ＋ 1.5h**。
- **ただしその期限が夜間（21:00–5:00）に入る場合は、翌朝5:00起点にリセット（＝6:30）**。
- 狙い＝旧「注文＋1h」だと、例えば **23:00注文 → 0:00に自動辞退 → 農家は承認できたのに朝には注文が消えてる**。夜リセットで「寝てる間に良い注文が勝手に切れる」を防ぐ。
- 夜間注文は物理的に即承認できないので、**"翌朝まで待てる"のが最善**。買い手には「農家の確認は朝になります」等を出せると尚良し（任意）。
- 10分前リマインドpush・期限切れ→自動辞退＋ウォレット返金 は従来どおり。

### ④ スロット選択可否に「承認期限＋準備4h」を追加（最重要）
**選べるスロット ＝ 「承認期限 ＋ 農家準備4h」以降**。これ1本で下記が自動で出ます：

| 注文タイミング | 早朝（例7:00）配達 |
|---|---|
| **前日など早め** | 承認期限が早い → **7:00もOK**（農家は前日から朝の準備を計画できる） |
| **夜間（21時以降）** | 承認期限が翌朝6:30 → ＋4h＝10:30 → **早朝は不可・最速11時頃**（農家が朝起きてから準備＝間に合わない） |

＝「早く頼めば早く届く／夜に頼むと朝イチは無理」が仕組みから自然に出る。**固定の早朝フロアや夜専用ガードは不要**。準備4h＝池揚げ〜処理〜配達（〜30km）の運営パラメータ（将来は距離連動）。

### backstop（既存・変更なし）
無理なスロットを選んでも、農家が承認しなければ**自動辞退＋ウォレット返金**。お金は必ず戻る。

## パラメータ一覧（すべて運営で調整可）
- 最短リード：8h／最長リード：14日／承認期限：注文＋1.5h／夜間帯：21:00–5:00（翌朝起点5:00）／農家準備：4h

## 補足
- 時間帯スロット（1時間刻み6:00–18:00）自体は変更なし。無理な早朝枠は④のルールが弾く。
- 他の依頼（丸め・内臓処理・レビュー導線等）とは別件です。
