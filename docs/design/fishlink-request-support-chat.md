# 【実装依頼】運営チャット（サポート）再設計 ＋ FAQ管理（新設）（2026-06-12）

下道さん

マイページ「運営からのメッセージ」（運営↔ユーザーのサポートチャット）を再設計しました。既存実装への修正＋**FAQ管理の新設**です。狙いは「**問い合わせが増えても運営が回るように、自己解決で量を絞る**」こと。

仕様の正本：
- 仕様書：**`fishlink-support-chat-spec.md`**
- モック：**`fishlink-support-chat-mockup.html`**（①カテゴリ選択／②振り分け／③ユーザー側チャット／④管理FAQ／⑤運営側チャット）

## 全体像（なぜ変えるか）
- 旧運営チャットは**一方通行に見える（緑バブルばかり）／毎メッセージにカテゴリタグ／入力UIが二重**で素っ気ない。
- さらに**自由な問い合わせ窓口は量が爆発して運営がパンク**する。→ **カテゴリ選択を“振り分けの関所”**にして、**FAQ＋既存窓口で自己解決→それでもダメな時だけ運営チャット**に通す。
- **FAQは管理画面で運営が育てる**（問い合わせ来る→Q&A追加→次から自己解決＝自己改善ループ）。

|------|------|--------|

| カテゴリの付き方 | 毎メッセージにタグ（冗長） | **「— 配送について —」の件の区切りに1回**（毎メッセージタグ廃止）。別件は「別の件で問い合わせる」で選び直し |

| 画像 | 1枚 | **複数枚 添付可** |

## 修正② FAQ管理（新設）＝モック④ / spec §7

**管理画面のナビに「FAQ」を追加**。運営がカテゴリ別にQ&Aを足して育てる。

- **カテゴリ＝ドロップダウン（選択式）**で切替。
- **Q&A：追加／編集／削除／公開・非公開／並べ替え（手動）**。
- **回答に画像（複数可）**（手順を step1→2→3 と画像で見せられる＝低リテラシーに効く）。
- **多言語**：質問・回答を**言語別に保持**（クメール/英/日・既存の3言語切替に乗せる）。**「編集する言語」タブ**で中身の言語を切替＝**管理画面のUI言語（🌐）とは別**（日本語UIのままクメール語の答えを編集できる）。機械翻訳の下書き→**ネイティブチェック済みで公開**（非公開がチェック前を兼ねる）。
- データ：`faq { カテゴリ, 質問, 回答, 画像[複数], 公開, 並び順, 言語別(km/en/ja), 登録日時 }`。

**ユーザー側**：カテゴリ選択後に**そのカテゴリの公開FAQをリスト表示**（質問タップで回答＋画像展開・0件なら省略・ユーザーの言語で表示）。

## 修正③ 振り分け（カテゴリ選択後）＝モック② / spec §4

カテゴリを選んだら、いきなり運営に送らず：
1. **そのカテゴリのFAQ** を表示
2. **関連窓口ボタン＝ロール別の“該当画面”へ飛ばす**。カテゴリ選択時点では**どの注文の話か分からない**ので、**特定注文でなくリスト/ハブへ飛ばし、そこで該当を選ぶ**：

   | カテゴリ | 農家 | レストラン |
   |---|---|---|
   | 注文／配送 | **注文確認** | **注文状況** |
   | 支払い・送金 | 売上・取引／送金状況 | 注文状況（お支払い待ち）／支払い画面 |
   | 紹介・クーポン | マイページ（紹介） | マイページ（紹介） |
   | アカウント | 基本情報／パスワード回復 | 基本情報／パスワード回復 |
   | 出品 | 投稿一覧／出品 | （無し） |

3. それでも解決しなければ「**運営にメッセージを送る**」→ 運営チャット

**カテゴリ**＝コード固定の初期セット（共通6＝注文/支払い・送金/配送/紹介・クーポン/アカウント/その他＋**農家のみ出品**。**トラブル報告は削除**＝問題報告フローへ）。GAqPは出品に含める。

- **CTA・送信＝青**（緑は使わない＝GAqP・成功・送料無料専用）。窓口＝青枠／問題を報告＝赤枠／注意＝amber。
