- **購入履歴**（注文状況）と**購入確認**（マイページ）が別々で重複。→ **「購入・取引」1画面に統合**。

## 修正① 注文状況（既存画面の再設計）＝モック① / spec §2B

| 項目 | 現状 | 修正後 |
|------|------|--------|
| 全体構造 | お支払い待ち/完了済み/辞退済みアコーディオン | **上から：対応が必要（お支払い待ち＋辞退）→ 進行中 →「購入履歴を見る」1行** |
| お支払い待ち | 赤いカード・赤ボタン | **urgent枠＋金額＋「お支払いへ進む」＝青**。**赤は「⏰ 支払期限を過ぎています」注記だけ** |
| 辞退された注文 | 緑「魚一覧から再注文する」 | **「同じ魚を再注文する」＝青**（その魚種を今出している**別農家へ**＝リピートは魚種単位）＋**「× 閉じる」（小）**。再注文/×で対応が必要から外れ、購入・取引の記録へ |
| 進行中 | （混在） | 農家確認中（⏳農家が在庫を確認しています・ボタンなし）／出荷準備・配達中・到着（「配送状況を確認」→チャット） |
| 履歴 | アコーディオン3本 | **「🧾 購入履歴を見る（完了 N・辞退・キャンセル N）›」1行**（→購入・取引） |
| 空状態 | — | 0件セクションは非表示／対応中ゼロ＝「✓ 対応が必要な注文はありません」＋履歴行は残す |
| 日付 | 「6/13」 | **「6/13（金）」曜日付き** |

**＋ 農家確認中のキャンセル（新規挙動）**：農家がまだ承認していない注文は**買い手がキャンセル可**（承認後は不可）。
- キャンセル時＝**農家へプッシュ通知＋農家の承認待ちから消す（期限停止）**。
- **競合ロック**＝同一注文で**承認とキャンセルは二重成立しない**（先に押した方が成立）。
- 記録＝両者の履歴に**「キャンセル」**で残す（**辞退とは別ラベル**：辞退＝農家都合／キャンセル＝買い手都合）。

## 修正② 購入・取引（新画面）＝モック②③ / spec §7

注文状況の「購入履歴を見る」と、マイページの「購入確認」が、**この同じ1画面に着地**します。

| 要素 | 内容 |
|------|------|
| フィルタ | **「過去1か月 ▾」＋「辞退・キャンセル」チップ**の2つ。期間＝本日/過去1週間/過去1か月/日付指定/すべて |
| サマリ | **購入額（払った合計・青）／取引数／平均単価**。（農家の「うち送金待ち」に当たる行は無し） |

| 「辞退・キャンセル」選択時（③） | サマリを**「成立しなかった注文 N件」**に切替＋リスト。**辞退済み（赤）＝農家に断られた＋理由／キャンセル（グレー）＝自分でキャンセル**。1チップにまとめ、バッジで区別 |

## 共通仕様（買い手 = 農家のミラー）

- **CTAは青**（お支払いへ進む／再注文／配送状況を確認／レビュー送信＝副／支払い状況を見る）。赤は支払期限超過の注記だけ。
- **農家名タップ＝生産者ページ**。価格＝青・割引＝amber・受取完了/送料無料＝緑。
- 決済はFishLink経由（中抜き対策）。
