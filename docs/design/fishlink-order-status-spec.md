# FISHLINK 買い手（レストラン）注文状況 / 購入・取引 実装仕様

実装の正本は **`fishlink-order-status-mockup.html`**（構造・レイアウト・配置はそこを参照）。
本書はHTMLから読み取れない情報＝**データモデル・状態・対応フロー・遷移・制約**を補う。

> 農家側 `fishlink-farmer-order-spec.md`（注文確認）の**買い手版・対**。注文確認まわりの再設計（A案＋取引履歴統合）を買い手にミラー。

### 実装方針の優先順位（他spec共通）
- 構成・データ・状態・遷移 → 本書とモック通り。
- 見た目（色・タイポ）→ 既存DS準拠。**CTAは青 `#0B5FB0`**。価格＝青／緑＝GAqP/成功/送料無料／**赤＝danger（支払期限超過の注記など）**／amber＝割引・キャンペーン。

---

## 1. 概要

買い手（レストラン）が**自分の注文の状態を確認し、必要な対応（支払い・再注文）をする**画面群。ボトムナビ「注文状況」。

- 注文は**農家ごと独立**（複数農家にまたがる買い物は農家ごとに別注文）。
- **決済はFISHLINK経由**（KHQRで FishLink へ支払い→農家へ送金）＝中抜き対策の本丸。
- ドメイン制約共通（KHR / head/kg / 供給不安定 / **リピートは魚種単位**）。

## 1.5 【案B 前払い対応＝2026-06-26】

> お金のロジックの正本は `fishlink-payment-spec.md`（案B v2.0・特に §3 状態モデル）。本specは注文状況画面のUI/挙動に限定。mockは `fishlink-order-status-mockup.html`（4画面）が正本。

**対応が必要（旧「お支払い待ち＋辞退」→ 案Bで再構成）**
- ❌ **「お支払い待ち」カードは廃止**（前払いで成立＝未払い注文は存在しない。「支払期限を過ぎています」赤注記・〔お支払いへ進む〕も廃止）。
- ➕ **「お支払い未確認」カード**（例外＝運営が入金を確認できない時だけ）：「入金を確認できませんでした」＋〔もう一度支払う〕青＋〔運営に問い合わせる〕。**自動キャンセルしない**＝運営が照合/督促で解決（payment-spec §4.1）。
- **辞退カード**：手動辞退＋**承認期限切れの自動辞退**（「農家から返答がなく自動的に辞退になりました」）。両方に**緑の返金行「✓ ◯◯KHRをウォレットに返金しました（次回使えます）」**＋〔同じ魚を再注文する〕。
- （問題報告フォローも対応が必要に入りうる。窓口は配達カード／お届け済みの〔問題を報告〕）

**進行中（待つだけ）**
- ➕ **「お支払い確認中」**（払った直後・⏳運営が入金を確認しています＝旧お支払い待ちの正常版）。
- **農家確認中**：農家の承認前なので**〔注文をキャンセル（ウォレットに返金）〕**可（控えめリンク）→ **キャンセル確認モーダル**（中央・「お支払い済みの◯KHRはウォレットに返金されます」金額緑・〔やめる〕グレー/〔キャンセルする〕赤）。承認後は不可＝〔問題を報告〕に切替。
- **配送中**：〔配送状況を確認〕＋**〔問題を報告〕**。
- ➕ **「お届け済み」**（緑バッジ・「✓配達完了。問題がなければ数時間後に自動で取引完了します」＋〔問題を報告する〕）。
- ➕ **「確認中」**（お届け済みで**問題を報告した後**の状態）＝**濃オレンジバッジ「確認中」（#C2410C）＋「🚩 問題を報告しました。運営が確認しています」**。お届け済みの「問題がなければ数時間後に自動で取引完了します」の一言は**消す**（報告後は自動完了しないため矛盾）。ボタンは付けない（報告は済＝待つだけ）。カードタップで**購入詳細（確認中）**へ。**運営が解決するまで進行中に残す＝終わっていないため`購入・取引`（履歴）には出さない**（履歴は取引完了/辞退/キャンセルのみ）。解決後に履歴へ移動（問題なし→取引完了／返金→ウォレット即返金＝辞退カードと同流儀）。※農家側「売上・取引」の確認中と対（farmer-order-spec §配送完了後・送金保留）。データ＝`payout_holding` に相当する買い手側フラグ（問題報告中）。

**お金の見え方** — 返金は銀行でなく**ウォレットに即返金**（payment-spec §4.5）。〔問題を報告〕は trouble-report フローの起点（配送完了＋4hの窓内）。

> 下の §2B〜§10（お支払い待ち前提の記述）は案Bで上記に置換。データモデルの `awaiting_payment` は廃止し、`payment_confirming / payment_unconfirmed / approval_pending / declined(auto含む) / preparing / delivering / delivered / completed / refunded` 等に（payment-spec §3・§5）。

**購入詳細（購入・取引のタップ先）＝read-only に置換**（旧「お支払い」画面＝受取時払いのKHQR支払い・支払期限超過バナー・支払い完了 は**廃止**）。前払い済みなので**払う操作は無し**。構成＝注文内容／現在の状態（お届け済み／**確認中（問題が報告されています）**／完了 等）／**お支払い（read-only）＝合計（支払い済み）＋内訳（魚代・送料・キャンペーン割引・💰ウォレット利用・お支払い額）＋「✓ お支払い済み（前払い）」**／状態に応じたアクション（お届け済み＝〔問題を報告〕窓内／**確認中＝アクションなし（"報告済み・対応中"の状態表示のみ／このあとについて＝「問題の確認が済むと、問題がなければ完了します。内容によっては一部・全額がウォレットに返金される場合があります」）**／完了＝**〔同じ魚を再注文〕（青・主）＋〔レビューを送信〕（副）**）。mock＝`fishlink-order-status-mockup.html` ⑤（お届け済み）・⑤'（完了）。辞退→ウォレット返金は注文状況の辞退カードで既出のため購入詳細では作らない。**〔同じ魚を再注文〕の遷移先＝その魚種の出品一覧（魚から探す相当・辞退カードの再注文と同じ流儀）**＝供給不安定で同じ出品が今日あるとは限らない／リピートは魚種単位のため。前回の農家/出品が今日無ければ一覧に出ないだけ（他の農家から選べる）・今日誰も出していなければ一覧は空状態・**ボタンは常に押せる（無効化不要）**。クーポンを使った注文は内訳の「🏷キャンペーン割引」が「🎟クーポン割引」に変わる（併用不可でどちらか一方）。

## 2B. 注文状況リスト（A案・優先フィード型）

**画面の正体＝「やることキュー」**。最優先＝**お支払い（KHQR）を見逃させない**（農家の「承認」に当たる買い手の最重要アクション）。

**構成（上から）**：
1. **対応が必要**＝赤見出し。買い手が今やること：
   - **お支払い待ち**（最重要・urgent枠）：魚名/サイズ/数量・農家・FL・納品日時・**支払金額**・**お支払いへ進む（青）**。支払期限超過は「⏰ 支払期限を過ぎています」**赤注記**（赤はこの注記だけ）。
   - **辞退された注文**：「農家に断られました。別の農家から同じ魚を再注文できます」＋**「同じ魚を再注文する」（青）**＋**「× 閉じる」（副・小）**。
2. **進行中**＝待つ状態。
   - **農家確認中**（承認待ち）：状態文「⏳ 農家が在庫を確認しています」。**ボタンなし**（待つ）。※キャンセルは §6。
   - **出荷準備／配達中／到着**：「配送状況を確認」（→配送管理チャット）。
3. **履歴**＝「**🧾 購入履歴を見る（完了 N ・ 辞退・キャンセル N）›**」の**1行リンク**（→②購入・取引）。

**空状態**：0件セクションは見出しごと非表示／対応中ゼロ＝「✓ 対応が必要な注文はありません」（赤くしない）＋購入履歴行は残す／完全新規＝「まだ注文がありません」＋魚を探す導線。

**カード本体タップ＝主アクションへ**：お支払い待ち→お支払い／辞退→再注文 or 詳細／進行中→配送状況／購入履歴カード→支払い状況（§7）。

## 3. データモデル

```js
buyerOrder = {
  id,                 // "FL-260612-001"
  fishName, sizeHeadPerKg, qtyKg, gutIncluded,
  farmer: { name, ratingPositivePct, province, district },
  deliveryDate, deliverySlot,
  amountKhr,          // 買い手が払う金額（手数料込み・内臓処理込み・送料込み＝§4）
  payDeadline,        // 支払期限（運営パラメータ）
  status,             // 下記
  declineReason,      // 辞退時：農家の構造化理由
}

status = "awaiting_farmer" // 農家確認中（承認待ち）
       | "preparing"       // 出荷準備（承認後）
       | "delivering"      // 配達中
       | "arrived"         // 到着
       | "awaiting_payment"// お支払い待ち（受取＝KHQR支払い）
       | "paid"            // 支払い済み（→送金/完了）
       | "done"            // 完了（取引完了）
       | "declined"        // 辞退（農家都合）→ 対応が必要（再注文）→ 記録
       | "cancelled"       // キャンセル（買い手都合・承認前のみ）→ 記録
```

## 5. ステータスと Home 5段ステッパーのマッピング

注文状況の状態は **Home の進行中ステッパー（注文済→農家確認中→出荷→配達中→完了）** と単一の真実を共有し、表示文言だけ役割別。到着＝配達中のまま＋「📍到着しました・お受け取りください」メッセージ＋push（農家 spec §5と対）。お支払い＝受取時（到着後）。

## 6. 対応フロー（買い手）

- **お支払い**：到着・受取時に **KHQR で FishLink へ支払う**（＝受領確認）。「お支払いへ進む」→支払い画面（精算 spec）。期限超過は督促push＋赤注記（自動キャンセルはしない＝魚は渡済み・与信問題、精算 spec §9）。
- **辞退された注文への対応**：
  - **「同じ魚を再注文する」**＝魚一覧でなく**その魚種を今出している別農家**へ（「魚から探す→農家一覧」相当・リピートは魚種単位）。新規注文を作成。
  - **「× 閉じる」**＝今は再注文しない時、対応が必要から外す。
  - どちらでも → **購入・取引の「辞退」記録**へ移動（Home L195：辞退カードは先頭固定＋再注文/×で片付け→注文状況「辞退済み」に記録、と整合）。
- **農家確認中のキャンセル（§6 重要）**：
  - **承認前は可／承認後は不可**（承認後は問題報告/連絡で対応）。「農家確認中」カードに「注文をキャンセル」（軽い導線）。
  - キャンセル時＝**農家へプッシュ通知**＋**農家の承認待ちリストから消す（承認期限停止）**。
  - **競合ロック**＝同一注文に対し**承認とキャンセルは二重成立しない**（先に押した方が成立。買い手キャンセルが先→農家の承認は弾く／農家承認が先→キャンセル不可）。
  - **記録**＝買い手・農家とも取引履歴に **「キャンセル」状態で軽く記録**（**辞退とは別ラベル**＝辞退＝農家都合／キャンセル＝買い手都合）。

## 7. 購入・取引（購入履歴＋購入額集計の統合画面）②③

注文状況「購入履歴を見る」とマイページ「購入確認」の**両入口が着地**（過去の購入が1か所）。農家「売上・取引」の対。

**構成**：
1. **フィルタ＝期間ドロップダウン「過去1か月 ▾」＋「辞退・キャンセル」チップ**の2つ。期間＝本日/過去1週間/過去1か月/日付指定/すべて。
2. **購入額サマリ**：**購入額（払った合計・青）／取引数／平均取引額（＝購入額÷取引数・1注文あたりの平均）**。※「平均単価」とは呼ばない（kgあたり単価と誤解される・農家側の売上サマリと同じ理由で統一）。※農家の「うち送金待ち」に当たる行は**出さない**（買い手の未払い＝お支払い待ちは①対応が必要に既出）。
3. **取引リスト＝軽い**：状態バッジ（取引完了＝グレー）・日時（曜日付き）・魚名(size)・数量・**農家名（青→生産者ページ）**・**支払額**＋`›`。アクションは置かない。新しい順・もっと見る。**※問題報告中（確認中）の注文はここに出さない**（＝終わっていない・注文状況の進行中に確認中で残る）。履歴に載るのは解決後（取引完了／辞退／キャンセル）。

**「辞退・キャンセル」チップON時（③）**：サマリを**「成立しなかった注文 N件（購入には含まれません）」**に差し替え＋リスト：
- **辞退済み（赤バッジ）**＝農家都合＋理由（「農家の辞退・理由：在庫が足りない」）。
- **キャンセル（グレーバッジ）**＝買い手都合（「自分でキャンセル（農家の承認前）」）。
- **1チップにまとめる**（別チップにしない＝3つに増やさない・「成立しなかった注文」で1カテゴリ）。

**取引カードのタップ先＝既存「支払い状況」画面（精算 spec）を流用**（新規画面を作らない）：
- 支払い状況画面＝注文サマリ＋支払金額内訳＋現在の状態。
- ここに **レビュー送信（副CTA・白地青枠・支払い後に解禁）** と **問題を報告（赤・報告できる期間だけ＝精算の自動確定窓と連動）** を追加。
- ＝軽いリストから外したアクションをここに集約。

## 8. 表示ルール

- **CTA青**：お支払いへ進む／同じ魚を再注文する／配送状況を確認（青枠）／レビュー送信（副・白地青枠）／支払い状況を見る。**赤は「支払期限を過ぎています」注記だけ**（ボタンに使わない）。
- **支払金額・購入額＝青**（買い手が払う価格の主役）。割引＝amber／緑＝GAqP・送料無料・受取完了バッジ。
- **辞退済み＝赤バッジ／キャンセル＝グレーバッジ**（「辞退・キャンセル」1カテゴリ内で区別）。
- **納品日は曜日付き「6/13（金）」**（全画面統一）。
- **農家名タップ＝生産者ページ**。
- 評価＝`★N%（件数）`（小数なし）。
