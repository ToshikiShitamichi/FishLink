## ①＋④ サムネ＆タップ直後の黒画面 ＝ セットで解決
- **①今のサムネ＝出品の"写真"を流用**しているが、**動画の1コマ（フレーム）をサムネにしたい**（TikTok/IG/FB同様＝中身が正直に伝わりタップを誘う）。
- **④リールをタップして全画面を開いた瞬間が黒い**（動画未読込のため）。
- **解決＝投稿時に動画から1コマを切り出して小さい静止画を作る（クライアント側 canvas 等・サーバ変換なし・クライアント圧縮と同時にやれば軽い）**。その静止画を：
  - **新着リールカルーセルのサムネ**に使う（①）
  - **全画面を開いた瞬間の背景(poster)**に敷く → その上で動画を読み込む（④の黒画面が消える）
- ＝1手で①④両方解決。**【2026-07-15＝実装してほしい・①④はセット・写真流用はしない】** 実機はまだ①写真流用／④黒画面のまま。**サムネもposterも"動画の1コマ（フレーム）"にしてほしい**（写真は使わない）。
  - **理由（写真posterがダメ）**：写真をposterに敷くと、動画が再生し始めた瞬間に**写真→動画（別の絵）にガクッとジャンプ**する。**動画フレームをposterにすればposter→動画が同じ絵でシームレス**（TikTok/IG/FBはこれ）。＝写真流用は見た目が繋がらないので不可。
  - **実装（いけるはず）**：`<video>`にローカルファイルを読み込み→seekして`<canvas>`に描画→`toBlob()`で小さいJPEG、を**既にやっているクライアント圧縮（「動画を処理中」）と同じタイミング**でやれば軽い（ローカルファイルなのでCORS問題なし）。iOS Safariの注意点（playsinline/読み込み待ち）はあるが対応可能なはず。
  - その1枚のフレーム画像を **①サムネ ＋ ④全画面を開いた瞬間のposter** に兼用（＝タップ直後の黒画面も消える）。
  - **①の適用範囲＝リールのサムネが出る全箇所**：Home新着リールのカルーセル／生産者ページの「🎬動画」ポートフォリオカルーセル／その〔すべて見る〕グリッド画面／商品詳細「この魚の動画」／（あれば）その他一覧。＝**どこも今は写真流用なので、全部を動画の1コマに**（実機で確認済み＝カルーセルもグリッドも写真のまま）。**動画フレームを出す共通処理を1つ**にして全箇所で使う。
  - **難しければ相談してください**（技術的な壁があれば一緒に考える。ただし写真流用での代替はしない方向）。
  - **🔴【2026-07-22 再レビュー＝まだ効いていない・診断済】**：数日前に着手いただいたが、**実機で①がまだ効いていません**。切り分けのため**新しく動画を1本投稿**して確認したところ、**新規投稿のサムネも写真のまま**でした（Home新着リールのカルーセル）。→ **「過去投稿に遡及していない」問題ではなく、フレーム抽出そのものが動いていない**（未実装 or 抽出が失敗してフォールバックしている）と判明。以下を確認してほしい：
    - **(a) CORS で canvas が汚染されていないか（最有力）**：**アップロード後のリモート動画URL**からフレームを抜くと、canvasがtaintされ `canvas.toBlob()`/`toDataURL()` が SecurityError で失敗→写真にフォールバックします。**必ずアップロード前のローカルファイル（`URL.createObjectURL(file)` の blob: URL）から抽出**してください（クライアント圧縮と同じ場所＝同一オリジン扱いでtaintされない）。
    - **(b) iOS Safari の seek/描画タイミング**：`video.muted=true` ＋ `playsinline` を付け、**`loadeddata` を待つ→`currentTime` を設定→`seeked` イベントを待ってから** `drawImage(video, …)` する。これを守らないと**黒/空フレーム**になります。復号が進まない場合は一瞬 `play()` する。`display:none` の video は描画できないので**画面外配置（visibility/off-screen）**で。
    - **(c) 失敗時のログ**：抽出に失敗したら**写真に黙ってフォールバックせず**、コンソール等に理由（例外内容）を出してほしい（今は「なぜ写真のままか」が外から分からない）。
    - ＝上記(a)(b)を直せば動くはずです。難しければ一緒に実装を見ます。

## ⑦【最優先・機能の生命線】実機スマホで動画が読み込めない／再生開始が激遅
**【2026-07-15 実機スマホ（iPhone・モバイル回線）で確認＝これは"あると嬉しい"でなく"直さないと機能が成立しない"】** 弱い回線のスマホ（＝本番のターゲット端末＝カンボジアの農家/レストラン）で、**全画面リールの動画がなかなか出てこない／黒いまま止まって見える**。ここが直らないとリール機能自体が使えません。**最優先で見てください。**
原因はだいたい次の3つ（**いずれもコスト分析に対策が書かれている項目**なので、"どれがまだ未対応か"の確認です）：
1. **MP4が faststart（progressive）になっているか＝最有力**：moov atom が**末尾**だと再生前に全ファイルDLが必要で激遅。**先頭（faststart）**にすれば頭の一部DLで即再生開始。→ **アップロード/保存時にfaststart化してますか？**（分析の「MVPは progressive MP4」に対応）
2. **クライアント圧縮の出力サイズ**：投稿時に「動画を処理中…」が出る＝**端末側の処理（圧縮）は入っている模様**。→ **その結果の実ファイルサイズは？（目標1本2〜6MBに収まってる？）** 圧縮はしてても出力が大きいと遅い。
3. **配信がRange/206（部分取得）に対応しているか**：非対応だとプレイヤーが頭だけ先読み再生できず全DLする。→ **206/Range対応してますか？**（分析で「現行SWが206/Rangeをキャッシュできない＝要対処」と既出の件は解決済み？）
- 上記の**根っこ（faststart/サイズ/Range）を直すのが本命**。
- **あわせて＝ローディング/失敗状態を堅くする**：実機では**くるくるが出る時（読込中）と、真っ黒で何も出ない時（止まって見える）が混在**していた。→ **①poster（動画の1コマ）を必ず敷く（黒くしない）②読み込み中は必ずローディング表示③読み込めなかったら「読み込めませんでした・再試行」を出す（永久に黒のまま放置しない）**。＝根っこが直るまでの間も、止まって見えないように。

### 🟢🔴【2026-07-22 再レビュー＝新規は解決・"既存動画"が残課題（遡及）】
- **新規投稿の動画＝再生OK ✅**（例：2時間前投稿のパンガシウス＝ちゃんと再生・自動再生も動く）。**⑦の根っこ修正が新規アップロードに効いています。** ＋**失敗時の「読み込めませんでした ↻再試行」UIも実装済 ✅**（上記の③）。
- **ただし修正前に投稿した"既存動画"は、まだ読み込めないものがある ❌**（例：5日前投稿のヒレナマズ＝「読み込めませんでした」のまま）。→ **⑦の直しが"新規アップロードのエンコード（faststart等）"に効いていて、既にアップロード済みの古いファイルには届いていない**と考えられます。
- **必要な対応＝既存動画の遡及（どちらかで）**：
  1. **既存の動画ファイルを新パイプラインで再処理（faststart化などの再エンコード）する一括migration**、または
  2. **配信側（サーバ/ストレージ）でRange/206・faststart相当を古いファイルにも効かせる**（再エンコード不要で全ファイルが即再生できるならこちらが軽い）。
- ＝**「どちらの直し方だったか（クライアント側エンコード or 配信側）」を下道さんに確認**し、既存動画にも効く形にしてほしい。本番でも「昔の投稿が見られない」は困るため。
- **🔎 追加の切り分け（重要）＝同じ古い動画が「PCでは再生できる・スマホ（iOS）では読み込めない」**：
  - **PCで再生できる＝動画ファイル自体は無事**（壊れていない・消えていない）＝**農家の再投稿は不要**。
  - **iOSだけ失敗＝モバイル特有の再生要件**：iOS Safariは **Range/206対応＋faststart（moov先頭）** に厳しい（Range非対応だと再生を拒否／moov末尾だと遅い回線で頭出しできない）。デスクトップは寛容なので通る。
  - **新規がスマホでOKなのは、新規がfaststart/Range対応で保存・配信されているから**。古い動画にその対応が及んでいないだけ。
  - **→ 直し方の確定：既存動画に faststart＋Range/206 を効かせれば直る。ファイルは生きているので再アップロード不要。まず「配信側でRange/206を古いファイルにも効かせる」（再エンコード不要なら最軽量）→ それでも頭出しできないファイルだけ faststart化の再エンコードmigration。**

## ⑪ 出品フォーム 動画欄の nudge 文言を修正
- 現：「泳いでる魚・**活〆**の動画があると**売れやすい**。FB用に撮った動画もそのまま使えます。」
- **新：「泳いでる魚・とれたての動画は、興味をひきます。」**
