# FISHLINK レビュー入力（相互レビュー）実装仕様

実装の正本は **`fishlink-review-mockup.html`**（4画面・構造/レイアウト/配置はそこを参照）。
本書はHTMLから読み取れない情報＝**評価モデル・データ・表示先・マスク/モデレーション・匿名性・バリデーション・解禁タイミング**を補う。

**相互レビュー**（レストラン⇄農家）の「入力」を扱う。表示側は `fishlink-producer-spec.md`（生産者ページ）／レストランページ（後日・新規）。連絡先マスクは `fishlink-qa-spec.md` §6 共通。状態・送金は `fishlink-payment-spec.md`、農家承認は `fishlink-farmer-order-spec.md`。

> 既存実装はレビュー画面あり。本書は差分（§15）が中心。

### 実装方針の優先順位（他spec共通）
- 評価モデル・データ・表示先・状態・遷移 → 本書とモックの通り。
- 見た目 → 既存DS準拠。**CTA＝青／良かった＝緑・残念だった＝赤（good/badトグルの例外）／アイコンは絵文字でなく線画スマイリー。**

---

## 1. 概要

- 取引完了後、**レストランと農家が互いを評価**する（双方向）。
- **3軸 × 二択（良かった／残念だった）＋コメント（任意）**。二択は**好評価率%（★N%）**に集計＝低リテラシー向き。
- レビューは**任意**（出さなくてよい）。送信後は編集不可（MVP）。

---

## 2. 画面構成・両側（軸は役割別）

| 項目 | レストラン→農家 | 農家→レストラン |
|------|------------------|------------------|
| 3軸 | 取引の満足度／魚の品質／配送時間 | 総合評価／やり取り／受取対応 |
| 評価対象 | 農家名＋魚種・数量・取引日＋FL番号 | レストラン名＋同上 |
| 公開先 | **生産者ページ** | **レストランページ**（後日・新規） |
| 入口 | レストラン注文状況「完了済み」カードの「レビューを送信」 | 農家 注文確認「完了済み」カードの「レビューを送信」 |

- 軸が役割別なのは正しい（各自が相手の関係ある面を評価）。**仕組み（二択／必須／マスク等）は両側共通**。

---

## 3. 評価モデル

- 各軸 **良かった／残念だった の二択** → **好評価率%** に集計（例 `★92%（40件）`）。3軸それぞれの%＝生産者ページ／レストランページの3軸メトリクスになる。
- **代表値**：レストラン→農家＝「取引の満足度」%／農家→レストラン＝「総合評価」%（カードや一覧の★%）。
- **0件（新規）は%を出さず「新規」表示**（#1の%統一ルール・低評価誤認を防ぐ）。
- コメントは公開テキスト（§9 マスク／モデレーション対象）。

---

## 4. データモデル

```
review {
  id, orderId,
  direction,            // "restaurant_to_farmer" | "farmer_to_restaurant"
  raterId, ratedId,
  axes: {               // 方向で軸が変わる・各 true(良かった)/false(残念だった)
    // R→F: satisfaction, fishQuality, deliveryTime
    // F→R: overall, communication, handover
  },
  comment,              // 任意・マスク後表示
  rawComment,           // マスク前原文（運営閲覧）
  masked,               // bool
  hidden,               // モデレーションで非表示にされたか
  createdAt,
}
```
- 1注文・1方向につき1レビュー。集計%は ratedId 単位で各軸ごとに算出。

---

## 5. 入力UI

- **アイコン＝線画スマイリー**：良かった＝笑顔／残念だった＝困り顔。未選択＝グレー、選択時＝**良かった 緑／残念だった 赤**（枠＋背景＋顔を着色）。**絵文字は使わない**（端末差・DS不一致）。タップ領域は大きめ（低リテラシー）。
- **3軸はすべて必須・コメントは任意**。
- **上部にガイド**「3つすべて選んでください（コメントは任意）」。
- **3軸は1枚のカードに区切り線**でまとめる（高さを圧縮）。見出しは1つ（「レビュー／取引を評価してください」）。
- **「残念だった」を選んだら、コメントを軽く促す**（コメント欄上に「差し支えなければ理由を教えてください（任意）」・**強制しない**）。理由が相手の改善に効く。
- **コメントの placeholder ＝中立「良かった点・気になった点など」**（良かった/残念どちらでも合う。**良かった/残念で文脈出し分けはしない**＝上のamber促しが補う・2026-06-27確定）。
- **送信完了画面**：送信後は「✓ ありがとうございました／レビューを送信しました」＋**〔戻る〕青（CTAは青）**→ 元の一覧へ（買い手＝注文状況／農家＝注文確認）。
- **問題を報告はこの画面に置かない**（§11）。

---

## 6. バリデーション

- **ボタンは無効化しない**（押せるまま）。押下時に検証（出品フォーム spec §4 と同方式）。
- 未選択の軸は **枠を赤＋その場に「選択してください」** をインライン表示。**最初の未選択軸へ自動スクロール**。
- 下部は**総括のみ**（「未選択の項目があります」）。単一の汎用警告だけにしない（どの軸か分かるように）。
- ※現状は「下に単一警告（評価を入力してください）」＝どの軸か不明 → 上記インラインに格上げ。

---

## 7. 解禁タイミング

- **レビューは支払いの合図の後に解禁**：レストラン＝「支払い完了」押下後／農家＝支払い確認後。**支払い前（QR提示中・支払い確認待ち前）は出さない**。
- 完了済みカードに「レビューを送信」（未送信）／「レビュー済み」（送信後）を出す。

---

## 8. 表示先・用途

- **レストラン→農家**：生産者ページに公開（3軸%＋コメント）。農家の評価。
- **農家→レストラン**：**承認時に農家が見る「レストラン信頼スコア（★%）」**（払いの良さ・受取対応・やり取り＝信頼性／未払いリスクの目安／誰に魚を回すか）。タップで**レストランページ**（生産者ページの対・後日新規）。
  - ※農家側 承認待ち詳細には既にレストラン評価%が表示＝これに該当。
- **公開の意味＝相手側に見える**（世界公開ではない）。**場所はテキストのみ・連絡先なし**（生産者ページと同ルール＝正確住所は配達フェーズだけ・連絡はアプリ内だけ＝中抜き/プライバシー）。
- 0件は「新規」。

---

## 9. コメントのマスク／モデレーション

- **連絡先マスク（自動）**：コメント本文の電話番号・SNS・URL → 伏字（`fishlink-qa-spec.md` §6・チャット共通）。**両方向に適用**。
- **モデレーション（運営）**：場所＝**管理画面「トラブル報告」**（Q&Aと共通）。マスク発動コメントが自動で「要確認」に上がる → 運営が **🗑非表示にする**（公開から消すがDB残す）。新画面は作らない。
- MVP＝「マスク発動分がトラブル報告に上がる＋運営が非表示」まで。
- **【後フェーズ：悪口・嘘（連絡先を含まない誹謗中傷／虚偽）対策】**＝マスクに引っかからないものは自動で上がらない → (a)ページ上でその場非表示 (b)ユーザー通報ボタン (c)被評価側の反論／異議。

---

## 10. 匿名性・報復対策

- **非匿名（評者名が出る）**＝B2Bは「誰が言ったか」分かる方が信頼できる。
- **報復懸念**（狭い市場・毎日リピート・選択肢少→本音の低評価が出にくい／タリ合い）に対しては **ダブルブラインド方式**（両者が出すまで／期限まで互いのレビューを非公開＝Airbnb/Uber方式）を**"正解"として目標**に据える。
- **MVPは即時公開・非匿名＋運営が初期は目視監視**（立ち上げ期はレビュー母数が少なく報復ループが起きにくい／コールドスタートは評価を早く見せたい／後からきれいに足せる）。**ダブルブラインド導入は後送り**（報復の兆候が出たら着手・悪口嘘対策と一緒に）。
- 注意：「正直な低評価は損」の空気が固定されると直しにくい → 初期は運営監視前提。

---

## 11. 問題報告との関係

- **問題を報告はレビュー画面に置かない**。理由＝完了済みカードで「レビューを送信」と「問題を報告」が横並び＝**分岐はカードで済んでいる**。レビュー画面で再掲は重複でポジティブ画面に水を差す。気が変われば←でカードに戻れる。
- 配置は **注文カード（注文状況／注文確認）＋支払い画面** に集約（payment spec／farmer-order spec と一致）。

---

## 12. 通知

- 取引完了（レビュー解禁）時 → 「レビューしませんか」push（解禁を絞っても数を確保）。
- レビュー受領 → 相手へ通知（任意・MVP最小）。

---

## 13. 色・表示ルール

- **送信＝青**（オレンジ廃止）。良かった＝緑／残念だった＝赤（good/badトグルの例外・アクション赤とは文脈別）。
- 評価対象＝**実名＋FL番号**。
- 緑はGAqP・成功・良かった専用／赤はdanger・残念だった専用。

---

## 14. 画面遷移

| 要素 | 遷移先 |
|------|--------|
| 完了済みカード「レビューを送信」 | レビュー入力画面 |
| レビュー送信 | **送信完了画面**（✓ ありがとうございました＋〔戻る〕青）→ 元の一覧へ（買い手＝注文状況／農家＝注文確認）。完了済みカードは「レビュー済み」に |
| 承認待ち詳細のレストラン名/スコア | レストランページ（後日） |
| 商品詳細・カードの農家名 | 生産者ページ |
