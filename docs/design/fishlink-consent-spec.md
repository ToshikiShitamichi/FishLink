# FISHLINK 利用規約・プライバシーポリシー 同意フロー 実装仕様

版: v1.0 / 2026-06-29

本書は規約・プラポリの**同意フロー**（登録同意・表示ページ・常時アクセス・管理画面編集・再同意方針）の正本。**本文（規約/プラポリの文章そのもの）は対象外**＝管理画面で運営が編集し、法務レビューで最終化（§9）。関連mock：`fishlink-consent-mockup.html`（ユーザー側：登録同意／マイページ行／規約ページ）・`fishlink-admin-legal-mockup.html`（管理画面：規約・プラポリ編集）。

---

## 1. 概要・スコープ
- 対象＝規約・プラポリへの**同意の取得**と、その**表示・アクセス・管理**。
- 対象外＝本文の文章（管理画面で編集・§5／§9）。
- 現状：登録同意・両ドキュメントのページは**実装済み**（追加＝マイページ行・管理画面編集・言語切替・表記／`fishlink-request-consent.md`）。

## 2. 同意方式（登録時）
- **みなし同意（b）＝採用**：登録フォーム末尾に「**登録すると 利用規約 と プライバシーポリシー に同意したものとみなされます**」＋各ドキュメントへのリンク、その下に〔**登録する**〕（青）。
- チェックボックスは足さない（低リテラシー向け・摩擦を増やさない）。みなし文＋リンクで同意は明確。
- 両ロール共通（農家・買い手とも同じ）。

## 3. 規約・プラポリのページ（アプリ内・読み取り専用）
- **アプリ内の読み取り専用ページ**で表示（外部Webでなく・本文は管理画面編集の内容を動的表示）。
- **言語切替＝ヘッダーの🌐**（アプリのグローバル言語設定に追従）。選択言語の版を表示。**その言語版が無ければ英語にフォールバック**。
- **最終更新日を表示**（例「最終更新：2026年6月」）。
- **「ドラフト（レビュー中）」等の状態バッジはユーザー画面に出さない**（ユーザーは**公開版のみ**閲覧・draft/公開は管理画面だけの概念）。

## 4. 常時アクセス
- **登録画面のリンク**＋**マイページに2行**（「利用規約」「プライバシーポリシー」を別々に・「情報・サポート」まわり）。それぞれタップで§3のページへ直接。
- 理由：登録後もいつでも読める必要（**プラポリの常時アクセスはApp Store/Google Playの要件**）。

## 5. 管理画面での編集
mock：`fishlink-admin-legal-mockup.html`
- **規約・プラポリを編集する画面を管理画面に追加**（FAQ管理と同じ発想）。運営が本文を編集 → アプリに反映。ナビに「規約・ポリシー」＋ドキュメント選択（利用規約／プライバシーポリシー）＋言語タブ＋公開中バッジ＋〔下書き保存〕〔公開する〕。
- **言語タブ（クメール／日／英）**で言語ごとに編集。
- **「ドラフト／公開」状態＋最終更新日**を管理画面で持つ（公開したものだけユーザーに表示）。
- 編集の保存フィードバック等はFAQ管理に準拠。

## 6. 再同意（改定時）
- **MVPは再同意フローを作らない**：規約ページに**最終更新日**を表示し、**継続利用＝同意とみなす**（規約本文にも「本サービスを利用することで…同意したものとみなされます」）。
- **重要な改定（特にプライバシー関連）時の告知＋再同意は将来**（最終更新日表示がその下地）。

## 7. 多言語・フォールバック
- クメール／日本語／英語。ユーザーの言語設定（🌐）に追従。未整備の言語は**英語へフォールバック**（空表示にしない）。
- 各言語版は**運営が管理画面の言語タブで用意**（§5）。作り方＝**運営が外部ツール（Google翻訳/DeepL等）で下訳 → 人（法務／ネイティブ）がレビュー → 管理画面に貼って公開**。**アプリ／管理画面に自動翻訳機能は持たせない**（MVP・その場で自動翻訳して表示する方式もしない＝法的文書の誤訳リスク回避）。将来、管理画面に「下訳を生成」ボタン（翻訳APIで下書き自動入力）は任意で追加可。※UI文言・FAQは自動翻訳＋軽レビューで可だが、規約・プラポリは別格。

## 8. 表記
- 「**FISHLINK**（大文字）」で統一（本文・ヘルプの「FishLink」を修正）。

## 9. 本文（別途・範囲外）
- 本文の文章は管理画面で編集し、**法務レビューで最終化**（カンボジアの個人情報保護・EC/決済規制）。今のドラフトは叩き台として良い。
- 規約に含める要点（FISHLINK固有・中抜き対策）：**アプリ内決済が原則／直接取引・連絡先交換によるアプリ外取引の禁止／違反はアカウント制限**。ほかサービス概要・アカウント・手数料/返金/キャンセル・行動規範・免責・準拠法（カンボジア）。
- プラポリの要点：取得情報（表示名・電話番号・位置情報・取引・チャット・レビュー）／利用目的／第三者提供（決済・地図・配送）／保管・削除／問い合わせ。
