# 【実装依頼】レビュー導線を分かりやすく（注文確認・注文状況に「レビュー待ち」）（2026-07-04）

下道さん

レビューする導線が奥に潜っていて分かりにくい、という課題の改善です。**農家・買い手の両方**が対象（左右対称）。正本＝`fishlink-review-spec.md` §7.1・`fishlink-farmer-order-spec.md` §2B・`fishlink-order-status-spec.md` §2B／mock＝`fishlink-farmer-order-mockup.html` ①・①'（農家）・`fishlink-order-status-mockup.html` ①・①'（買い手）。

## 背景（なぜ）
今、レビューするには深く潜る必要がある（5〜6タップ）：
- 農家：注文確認 → 取引履歴を見る → 売上・取引 → 取引タップ → 送金状況 → 最下部「レビューを送信」
- 買い手：注文状況 → 購入履歴を見る → 購入・取引 → 取引タップ → 購入詳細 → 最下部「レビューを送信」

深すぎて「どこでレビューするの？」になる。やることリスト（✓✓）にも出ているがヘッダーで気づきにくい（かつ✓✓は将来外す予定）。

## 直してほしいこと

**農家「注文確認」・買い手「注文状況」の両方に「レビュー待ち」を追加**（お手本＝各mock ①・①'）：

1. **入口**：**「⭐ レビュー待ち ◯件 ›」** の1行（見出し「レビュー」・**履歴の上**）を、農家＝注文確認／買い手＝注文状況 に追加。「対応が必要な注文はありません」の空き画面にも出す。
2. **タップ先＝レビュー待ち一覧（子画面）**：レビューできる取引を軽いカードで並べ、**各カードに「⭐ レビューを送信」**（→ 既存のレビュー入力画面へ直接）。評価対象＝農家は「レストラン」／買い手は「農家」。
3. **対象＝取引完了かつ自分が未レビュー・直近2週間だけ**。
   - **0件のときは「レビュー」セクションごと非表示**。
   - 送信済みの取引は一覧から消え、件数が減る。
   - 2週間を過ぎた取引は**入口が消えるだけ**で、完了詳細（農家＝送金状況／買い手＝購入詳細）の「レビューを送信」からは引き続きレビュー可（＝期限ではない）。
4. **完了詳細の「レビューを送信」ボタンは残す**（農家＝送金状況／買い手＝購入詳細・確認のついでに評価＝文脈導線）。push通知も従来どおり。

＝ push（能動）→ レビュー待ち（毎日見る所）→ 完了詳細ボタン（文脈）の**3層**にして、両側の埋没を解消します。

## 補足
- レビュー入力画面・集計・生産者ページ表示は既存のままでOK（入口を足すだけ）。
- 売上・取引/購入・取引の修正（`fishlink-request-farmer-order-fixes2.md`）・100リエル丸め（`fishlink-request-amount-rounding.md`）とは別件です。
