## 全体像（前提の確認）

- Homeの「**魚から探す**」で魚種タイル（例：ヒレナマズ）をタップ → **この魚一覧に「魚種＝ヒレナマズ」フィルタが適用済みの状態で遷移**（＝現状の挙動でOK。専用の「農家一覧」画面は作りません）。
- 理由：価格比較は魚種を絞った後が本命で、絞り込んだ商品一覧がその比較画面そのもの。必要なのは価格・サイズ・在庫・距離・送料＝商品レベルの情報。農家発見はHome「おすすめ農家」が担当。
- 今回はこの「絞り込み中の状態」を2点強化します → ①アクティブ絞り込みチップ行（新規）／②0件の空状態（修正）。

## ① アクティブ絞り込みチップ行（新規追加）

絞り込みシートを開かなくても「いま何で絞っているか」が一覧上で分かり、その場で個別解除できるようにします。

- **配置**：ソート/絞り込みバーの**下**・件数表示「N件の魚」の**上**。
- **表示条件**：絞り込みが**1件以上効いている時だけ表示**（0件なら出さない）。**チップ数＝「絞り込み」ボタンの適用数バッジの数と一致**。
- **チップにするフィルタ＝効いているもの全部**：

| フィルタ | チップ表記 | 備考 |
|---|---|---|
| 魚種 | `ヒレナマズ ✕` | 複数選択時は魚種ごとに1チップ |
| 地域 | `タケオ州 ✕` | 同上 |
| 価格帯 | `〜12,000 ✕` | 上限のみ可動の単一ハンドル＝「〜N」表記（単位省略でコンパクト）。最上端＝上限なし＝フィルタ扱いせずチップ無し |
| ♡お気に入り | `♡ お気に入り ✕` | |
| ⏱過去注文 | `過去注文 ✕` | |

- **✕タップ＝そのフィルタだけ即時解除**（**絞り込みシートは開かない**＝この行の肝）。価格帯チップの✕＝上限を「上限なし」に戻す。解除後すぐ一覧に反映。
- **シート内「リセット」との役割分担**：チップ✕＝個別解除／リセット＝全解除。
- **あふれた時は2行に折り返し**（横スクロールにしない）＝低リテラシー層に全チップを見せる優先。
- **起点非依存**：タイル経由で適用された魚種チップも、手動で付けた他チップと同じく✕で外せる（外せば全件に戻る）。

## ② 0件の空状態（修正）

現状＝「条件に合う魚がありません」＋緑「絞り込みをリセット」＋**0件なのに「もっと見る」が出ている**。これを改善します。

| 箇所 | 現状 | 修正後 |
|------|------|--------|
| 見出し | 常に「条件に合う魚がありません」 | **出し分け**：魚種フィルタ**1つだけ**が効いている時（他の絞り込みなし）→「**本日◯◯の出品はありません**」＋サブ「別の魚なら出品があります」／それ以外（複数魚種・地域・価格帯など混在）→「条件に合う魚がありません」（サブなし） |
| ボタン | 緑「絞り込みをリセット」 | **青「すべての魚を見る」**（全フィルタを外して今日出てる魚を表示） |
| もっと見る | 0件でも表示 | **0件では非表示** |

- **出し分けの理由**：この画面の本命の問いは「今日この魚を誰が出してる？」。魚種単独で0件なら「今日は誰も」と魚種名で答えるとエラーでなく"今日は無い"と腑に落ちる。※価格帯等も同時に効いている時に魚種名を出すと不正確（高い価格帯にはあるかも）になるので、**魚種フィルタ単独の時だけ**魚種名を出す＝安全側。
- **「すべての魚を見る」はフィルタだけ外す＝ソートは触らない**（初期値＝距離が近い順を維持。ユーザーが別ソートに変えていればそれを維持）。ボトムナビ「魚一覧」を素で開いた状態（全件・近い順）と同じ画面に着地。
- **入荷通知は付けない**（生鮮＝今日必要。魚種まるごと0件の通知は将来検討）。
