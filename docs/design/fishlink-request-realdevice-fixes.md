# 【実機 不具合報告】FISHLINK（fishlink-t-shitamichi.web.app）（2026-06-26）

下道さん

実機で確認できた不具合を3点共有します。いずれも `fishlink-t-shitamichi.web.app` で発生。**①は出品ブロッカー（影響大）**です。

---

## ① 農家のQR画像登録が失敗（送金先）　影響度：高 🔴

**現象**：農家が送金先のQR画像を登録しようとすると「保存に失敗しました（រក្សាទុកបរាជ័យ）」。
**環境**：Android 実機。

**エラー内容（アプリのダイアログのデバッグ情報）**
```
phase=resize | msg=Image decode failed
(name=878.jpg, type=image/jpeg, size=413218)
file=878.jpg | size=413218 | type=（空）
```

**読み取れること**
- 失敗箇所＝**クライアント側のリサイズ処理（phase=resize）で画像のデコードに失敗**。
- ファイル自体は 878.jpg・約404KB・image/jpeg ＝サイズは小さく、壊れた巨大ファイルではなさそう。
- **末尾の `type=` が空** ＝ File/Blob を再取得した経路で MIME が落ちている可能性（手がかり）。

**推奨対応**
1. **リサイズ失敗時は原画像のままアップロードにフォールバック**（QR画像は404KBと小さいので、リサイズできなくても保存できれば実用上OK）。**保存を丸ごと失敗にしない**のが最優先。
2. デコードの堅牢化：`createImageBitmap()` ⇄ `<img>`+canvas の二段フォールバック。Android実機のJPEG（EXIF orientation／progressive／CMYK／拡張子jpgのHEIC偽装）も考慮。
3. `type` が空になる経路の調査（再読込で MIME が欠落していないか）。
4. ユーザー向け文言：「保存に失敗しました」だけだと農家が詰まる → 「別の画像でお試しください／そのまま保存」等の案内。

**影響**：送金先が登録できないと**出品できない**（投稿ゲート）＝農家がブロックされる。
**当面の回避策**：送金先は「QR画像 **または** QRリンク」のどちらか1つでOK ＝ **QRリンクでの登録**を案内すれば回避可能。

## ② パスワード再設定で「reCAPTCHA に接続できません」（SMSは届く）　影響度：中

**現象**：パスワード再設定（電話番号）で、**確認コードのSMSは問題なく届く**のに「reCAPTCHA に接続することができません。ネットワーク接続をご確認のうえ、もう一度お試しください。」ダイアログが出る。
**環境**：iPhone・カンボジア回線「Smart 5G」。右下に reCAPTCHA のリフレッシュアイコンが見えている。

**読み取れること**
- Firebase Phone Auth の **invisible reCAPTCHA**。
- **SMSが届いている＝reCAPTCHAは実際には通っている**のにエラーが出ている＝**成功と矛盾する紛らわしい誤エラー**の可能性が高い。
- 原因候補：①reCAPTCHAのエンドポイント（`google.com/recaptcha`・`gstatic.com`）への接続が地域回線で遅延・不安定 ②`RecaptchaVerifier` が試行間で reset されず再試行で接続エラー ③invisible reCAPTCHA のロード/期限切れ。

**推奨対応**
1. **SMS送信が成功した場合はこのエラーダイアログを出さない**（成功フローを優先し、後続の reCAPTCHA 系エラーは握りつぶす／リトライ）。
2. `RecaptchaVerifier` を試行ごとに `clear()/reset()` して再生成。
3. reCAPTCHA ロード失敗時のリトライ＋タイムアウトを緩めに（**カンボジア回線で google reCAPTCHA が遅い前提**）。
4. ⚠️ 構造的懸念：カンボジア回線で google reCAPTCHA への接続が不安定だと、**電話認証全般（新規登録・ログイン・再設定）で同種の問題**が出うる → 認証の根幹なので要ウォッチ。

**影響**：SMSは届くので致命ではないが、ユーザーが「失敗した」と勘違いして離脱する恐れ。

## ③ 下部ナビがホームバーに被る（セーフエリア未対応）　影響度：中〜高

**現象**：下部ナビ（ホーム／魚一覧／カート／注文状況）が端末の**ホームバー／ジェスチャーバーと重なり、タップしづらい・一部隠れる**。
**環境**：農家の Android 実機。
**原因（ほぼ確定）**：下部ナビが**セーフエリアを考慮せず画面最下部に固定**されている。

**推奨対応**
1. 下部ナビ（および固定フッターのCTA）に **`padding-bottom: env(safe-area-inset-bottom)`**（iOS旧来の `constant()` フォールバックも）。
2. `<meta name="viewport" content="width=device-width, initial-scale=1, viewport-fit=cover">` を確認（`viewport-fit=cover` が無いと `env()` が効かない）。
3. 実機（iPhoneのホームバー機・Androidのジェスチャーナビ機）で、下部ナビが帯の**上**に収まるか確認。

**影響**：主要ナビが押しにくいのは回遊の根幹に効く。全画面の下部ナビ・固定フッターが対象＝**1か所直せば全画面に効く**。

- ①は出品ブロッカーなので優先度高め。当面はQRリンク登録で回避可能。
