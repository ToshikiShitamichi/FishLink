# 【実装依頼・訂正】確認ダイアログは「中央モーダル」で統一（過去の「ボトムシート」指示を撤回）（2026-06-19）

下道さん

確認ダイアログ（「◯◯しますか？」の自前ダイアログ）の**形を1つに統一**したく、過去の依頼で一部「ボトムシートに」と書いていた箇所を**撤回**します。すみません、ここを正します。

## 結論

- **確認系ダイアログ ＝ 中央モーダル**（画面中央・背景暗転）。
- **ボトムシート（下からスライド）＝ 入力／選択のみ**（魚一覧の絞り込み・出品の配送設定・チャットのカテゴリ選択 など）。
- **ブラウザ標準ダイアログ（`confirm()` ＝「◯◯.web.app の内容」が出るもの）は全廃**（この方針は不変）。今回変えるのは「自前にする」の**形だけ**＝ボトムシートでなく**中央モーダル**で。

理由：①モックは元々すべて確認＝中央モーダル（承認・辞退・支払い完了 等）②iOS/Android のネイティブ確認も中央 ③確認とフィルタ（絞り込み）が同じ「下から」だと“決断”と“作業”の区別が消える。確認は中央＋暗転で一拍止めるのが正。

## ボタンの規則（確認系・共通）

- 左＝**キャンセル/やめる（ニュートラルグレー）**
- 右＝実行ボタン：**破壊的なら赤**（「削除する」「取り消す」）／**非破壊なら青**（「承認する」「辞退する」「送信」「支払い完了」）。

## まとめ

- やることは「`confirm()` → 自前」だけ変わらず。形は**全部 中央モーダル**で。
- ボトムシートは絞り込み・設定・選択のような**入力UI専用**に温存。
- 各 spec も「確認系＝中央モーダル」に直済み（chat §11 / farmer-order §10 / order-status §13 / payment §13 / qa §9）。

## 追記（2026-06-27）：配送完了モーダルの**文言**を案B（前払い）版に

実機で配送管理チャットの「配送完了」確認ダイアログを確認しました。**形＝中央モーダルで正しく実装OK**（左キャンセル グレー／右「配送完了にする」青）＝形の修正は不要です。**ただし文言が旧・受取時払いのまま**でした → 案B（前払い）に直してください。

| 箇所 | 撤回前（誤・受取時払い） | 確定（正・案B 前払い） |
|---|---|---|
| 配送完了 確認モーダルの本文 | 「配達完了にしますか？ **レストランにお支払い（受け取り時のお支払い）のお知らせが届きます。**」 | 「**配送完了にしますか？ レストランに配達完了が通知されます。問題がなければ数時間後に取引完了し、送金が確定します。**」 |
| 配送完了後の遷移 | レストランは**「お支払い」画面へ**（受け取り時にQR支払い） | **お支払い画面へは進まない**（前払い済み）。レストランは**「お届け済み」**で〔問題を報告〕できる（受取確認ボタンは無し＝Grab/Uber流は不変） |

- 理由：案B（注文時前払い）では、レストランは**注文時にすでに支払い済み**。配送完了時に「受け取り時のお支払い」の通知は飛びません。
- 反映済み：`fishlink-chat-spec.md` §10／`fishlink-farmer-order-spec.md` §7／`fishlink-farmer-order-mockup.html`（⑤ 配送完了の確認モーダル）。お金の流れの正本は `fishlink-payment-spec.md` §4.6。
