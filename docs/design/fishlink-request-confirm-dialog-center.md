# 【実装依頼・訂正】確認ダイアログは「中央モーダル」で統一（過去の「ボトムシート」指示を撤回）（2026-06-19）

下道さん

確認ダイアログ（「◯◯しますか？」の自前ダイアログ）の**形を1つに統一**したく、過去の依頼で一部「ボトムシートに」と書いていた箇所を**撤回**します。すみません、ここを正します。

## 結論

- **確認系ダイアログ ＝ 中央モーダル**（画面中央・背景暗転）。
- **ボトムシート（下からスライド）＝ 入力／選択のみ**（魚一覧の絞り込み・出品の配送設定・チャットのカテゴリ選択 など）。
- **ブラウザ標準ダイアログ（`confirm()` ＝「◯◯.web.app の内容」が出るもの）は全廃**（この方針は不変）。今回変えるのは「自前にする」の**形だけ**＝ボトムシートでなく**中央モーダル**で。

理由：①モックは元々すべて確認＝中央モーダル（承認・辞退・支払い完了 等）②iOS/Android のネイティブ確認も中央 ③確認とフィルタ（絞り込み）が同じ「下から」だと“決断”と“作業”の区別が消える。確認は中央＋暗転で一拍止めるのが正。

## ボタンの規則（確認系・共通）

- 左＝**キャンセル/やめる（ニュートラルグレー）**
- 右＝実行ボタン：**破壊的なら赤**（「削除する」「取り消す」）／**非破壊なら青**（「承認する」「辞退する」「送信」「支払い完了」）。
