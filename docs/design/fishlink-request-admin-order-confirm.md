# 【実装依頼・運営管理画面】注文管理詳細「レストラン入金確認」の修正（2026-07-10）

下道さん

運営管理画面の**注文管理詳細（FL-…）の「レストラン入金確認」セクション**に修正点が出ました。前払いの入金照合を手動で回すための穴埋めです。

※これは**運営管理画面トラック**の一部です（買い手側の支払い画面修正は別依頼 `fishlink-request-payment-review.md`）。運営画面の完全なUIは別途まとめて詰めますが、今回の2点は入金フローが成立するのに必須なので先に。

正本：仕様書 `fishlink-payment-spec.md` §8.2。簡易モック `fishlink-admin-order-detail-mockup.html`（3ケース＋未確認確認モーダル）。

---

## 1. 運営アクションを「2択」で閉じる（最重要）
今の実機は〔**入金確認済にする**〕しか無く、「入金が確認できない時」に状態を動かす手段がありません。→ 注文が「支払い確認中」で永久に止まり、買い手の「お支払い未確認」カードが**一生発火しない**穴。

- 〔**入金確認済にする**〕（青）＝金額・時刻が一致したら（現状維持）
- 〔**お支払い未確認にする（買い手に通知）**〕（**新規**・アンバー／ニュートラル＝赤にしない）
  - 押すと **確認モーダル**：「お支払い未確認にしますか？」＋「**理由（内部メモ）＊必須**」＋「自動キャンセルはしない・理由は内部メモで買い手には汎用通知のみ」
  - 確定 → 注文が「支払い未確認」に遷移 → **買い手の注文状況に「お支払い未確認」カードが発火**（〔運営に相談する〕〔もう一度支払う〕・別依頼B-1）

## 2. 買い手が添付した支払い控え（スクショ）を表示
- 買い手が任意で添付した控え（`payment-review` A-5）を、この入金確認セクションに**表示**する。運営が照合に使う。
- 控えに載る照合情報＝Trx ID／From account（名義+番号）／金額／**Reference #（最強の照合キー・一意）**／To account／日時（実機のABA控えで確認済み）。
- **控えの添付が無いケースでも**、入金控え・メモ欄（メモ・入金スクショ添付・内部メモ保存）は出す（運営が自分で記録）。

## 3. 照合は運営の目視（OCR・自動照合はしない）
- スクショから名義/Refを**自動読み取り（OCR）や自動照合はMVPでやらない**（銀行ごとにレイアウトが違い精度が出ない）。
- 運営が「買い手の控え（表示）」と「**Joint口座（ABAアプリ）の入金履歴**」を見比べて、金額・時刻・Refが一致するか**目視で確認** → 確認済/未確認。構造化された自動フィールドは持たず、一言ガイド＋自由メモのみ。
- ※確認元は**Merchant App ではなく個人/Joint口座のABAアプリ**（マーチャント口座は未取得）。

## 4. 「お支払い未確認にした後」＝別画面は作らない
- 同じ入金確認セクションに「**お支払い未確認・通知済み**」バッジ＋書いた理由を表示。
- 〔**入金確認済にする**〕は残す＝買い手が〔もう一度支払う〕or 控えを再送 → 運営が確認 → 確認済に戻す（注文が進む）＝**解決導線**。
