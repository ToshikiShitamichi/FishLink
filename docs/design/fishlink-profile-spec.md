# FISHLINK プロフィール（マイページ）実装仕様

実装の正本は各モックHTML：
- `fishlink-mypage-mockup.html`（マイページ top・農家／買い手）
- `fishlink-basic-info-mockup.html`（基本情報）
- `fishlink-payout-refund-mockup.html`（送金先／返金先）
- `fishlink-delivery-setting-mockup.html`（配送設定・独立画面）

本書はHTMLから読み取れない情報＝**データ・状態・必須/任意・ルール・遷移・横断方針**を補う。
関連：紹介＝`fishlink-referral-spec.md`／配送設定の中身（出品側）＝`fishlink-posting-create-spec.md`／返金・送金の精算＝`fishlink-payment-spec.md`。

### 実装方針の優先順位（他spec共通）
- 構成・データ・状態・遷移 → 本書とモックHTMLの通り。
- 見た目 → DS準拠。**CTA＝青／緑はGAqP・送料無料専用／赤＝破壊的（削除など）専用／amber＝注意・キャンペーン**。本番デフォルト言語＝クメール語。

---

## 1. 概要

マイページ＝ユーザー（農家／買い手）の自分用の管理ハブ。top（ハブ）＋個別画面（基本情報／公開ページ／送金先・返金先／配送設定／紹介／運営メッセージ）。ヘッダー右上の 👤 から遷移。

**対象（本書の範囲＝束A＋紹介エントリ）**：マイページ top／基本情報／送金先（農家）・返金先（買い手）／配送設定（農家）／紹介エントリ（1行）。
**範囲外（別途）**：紹介専用ページの中身（→ referral spec ＋ 束B）／運営からのメッセージ（→ support-chat）／公開ページの中身（→ producer / restaurant spec）。

## 2. マイページ top（`fishlink-mypage-mockup.html`）

### 2-1 プロフィールヘッダー
- **アバター（円形）＋役割ラベル＋表示名**。**@ハンドル（test_xxx）は出さない**（識別は表示名＋電話番号）。
- **アバターは表示のみ**。設定は公開ページ側（農家＝生産者ページ「顔写真」／買い手＝レストランページ「アイコン写真」）。未設定＝デフォルトアイコン。

### 2-2 並び順（確定）
| 農家 | 買い手 |
|---|---|
| 基本情報 → **生産者ページ** → 送金先情報 → 配送設定 → 売上・取引 → 紹介コード・ボーナス → 運営からのメッセージ → ログアウト | 基本情報 → **レストラン情報** → 返金先 → 購入・取引 → 紹介コード・クーポン → 運営からのメッセージ → ログアウト |

- 考え方：基本情報の**すぐ下に公開ページ**（プロフィール2枚を隣に＝非公開の登録情報／公開ページ の対比で迷わない）→ お金 → 配送設定（農家のみ）→ 売上/購入 → 紹介 → サポート → ログアウト。
- **公開ページ行のサブ**＝「**買い手に公開されるページ**（農家）／**農家に公開されるページ**（買い手）」（公開/非公開の混同防止）。
- **ログアウト＝中立グレー**（破壊的でない＝赤にしない）。

### 2-3 紹介エントリ（1行）
- §6 参照。巨大カードは置かず**1行メニュー**。サブ文言は状態で動的（金額は運営設定を表示）。

## 3. 基本情報（`fishlink-basic-info-mockup.html`）

3項目：**電話番号／表示名／位置情報**（農家・買い手とも同構造）。

### 3-1 各項目
| 項目 | ルール |
|---|---|
| 電話番号 | 旧「ログインID（ユーザー名）」を**電話番号表示**に（＝ログインに使う番号）。**編集可：新番号に SMS で確認コード（OTP）→ 確認できたら変更**（番号の到達性ゲートも兼ねる）。ユーザー名（@ハンドル）は画面に出さない（内部IDはadminに残置可）。 |
| 表示名 | 編集可。**公開ヘルプ「この名前は買い手に公開されます。」**を表示。 |
| 位置情報 | 地図＋ピン操作（**地図/航空写真トグル**）。「現在の位置情報」＝**ピン地点の住所の読み取り専用表示**（手入力不可・ピンが真実の源）。「現在地を使う」可。 |

### 3-2 編集モデル・保存
- **1項目ずつ編集**：ある項目の〔編集〕を開くと他は表示のまま（**保存ボタンは常に画面に1つ**）。
- **保存ボタン＝青**（農家側が緑だったら青へ）。キャンセル＝中立。
- 保存成功＝**「✓ 保存しました」トースト**（§7-1）。

### 3-3 位置情報のプライバシー（既存方針・維持）
- **編集画面（私的）＝詳細住所までOK**。**公開側（生産者ページ・商品詳細・レストランページ）＝州・地区のみ**。買い手の位置情報も保持（農家の配送距離計算に使用）。

## 4. 送金先（農家）／返金先（買い手）（`fishlink-payout-refund-mockup.html`）

同じ画面構造（口座の登録名＋受取用QR）。**画面まるごと編集（1つの〔編集〕→全項目→保存1つ）**＝名義とQRが一致しないと送金/返金が失敗する1つの塊のため（基本情報の項目ごと編集とは別）。

### 4-1 必須/任意（バッジで明示）
- **口座の登録名＝[必須]**。
- **受取用QR＝[いずれか1つ必須]**（**QRリンク または QR画像**）。2欄を1グループにまとめ、間に「**─ または ─**」の仕切り（両方でなく片方でいいを視覚化）。
- 口座の登録名 placeholder は**ABA限定にしない**（例「口座名義（ABA / Wing など）」＝KHQRは銀行横断）。
- 編集時 amber 警告：送金先「**送金はこの名義・QR宛に行われます。お間違いのないようご確認ください。**」／返金先は同文の**返金版**。
- QR画像＝選択／**削除（赤）**。保存＝青／キャンセル＝中立。

### 4-2 送金先（農家）＝必須
- **必須**（出品ゲート＝未登録だと売上を受け取れない）。auth mock ⑧⑨⑩ と整合。
- 送金は**常に満額**（紹介クーポンの原資はFishLink負担・農家は損しない／referral spec §4）。

### 4-3 返金先（買い手）＝任意＋促す
- **任意**（普段KHQR払いで使わず・返金は稀。全員に最初から登録させない）。
- **未登録の空状態＝専用画面**：バラの「−」を出さず、「まだ登録されていません／返金が発生したときの受取先です／登録しておくと万一の返金がスムーズ（必要になってからでもOK）」＋〔返金先を登録する〕。
- 編集時に**なぜ要るかの説明**（ℹ️「返金が発生したときの受取先です…」）。各バッジに「**＊登録する場合**」。
- **登録の促し（必須化しない）**：①空状態で促す（メリット＋ボタン）＋③' マイページの返金先行に**中立グレーの「未登録」**表示（amber/赤にしない）＋④ **JIT：返金が必要になった時に登録を促す**（未登録返金は運営チャット案内＝MVP）。**②初回注文後のポップアップは入れない**。

### 4-4 未定義（要決定・軽微）
- QRリンクとQR画像が**両方ある時**、運営が送金/返金に使う優先（推し＝QR画像が主・リンクは予備）。

## 5. 配送設定（農家・独立画面）（`fishlink-delivery-setting-mockup.html`）

- マイページの入口から開く**独立画面**。中身は出品フォームの配送設定と同じ＝**`farmerDelivery` を共有・正本はマイページ**（出品フォームの〔変更〕ボトムシートは近道）。どちらで変えても全出品に反映。
- 3欄：**配送可能距離＝[必須]／10kmあたりの送料＝[必須]／送料無料距離＝[任意]**。
- **全出品共通の警告**：出品がある時「配送条件は全出品に共通です。変更すると現在の **N件** すべてに反映されます。」／**0件のとき「まだ出品がないため、今後の出品に適用されます。」**に出し分け。
- **初回・未設定＝空＋プレースホルダ**（例：30／例：5,000／例：5）。初回プリフィルしない（出品 spec ⑦と同じ）。一度設定すれば出品フォームは配送サマリー行で済む。
- **数値欄はスピナー非表示**（`inputmode="numeric"`）。
- **送料無料距離のヘルプ＝ニュートラルグレー**（文全体を緑にしない・"送料無料"の語だけ緑は可）＋文言「この距離内は送料無料になります。**近場は送料無料にするのがおすすめです（例：5km）**。空欄なら全配達で送料がかかります。」
- ボタン＝「保存して全出品に反映」（青）／キャンセル（中立）。保存成功＝トースト。
- ※出品フォーム側の同等修正は `fishlink-request-posting-create-fixes.md` 参照（独立画面と同じ内容を揃える）。

## 6. 紹介エントリ（マイページの1行）

巨大カードを置かず**1行メニュー**（中身＝コード/コピー/シェア/ボーナス・クーポン状態は**紹介専用ページ**＝referral spec／束B）。

### 6-1 サブ文言（状態で動的・金額は運営設定を表示）
| ロール | 保留/所持あり | 無し |
|---|---|---|
| 農家「紹介コード・ボーナス」 | 「あと◯回・取引完了ごとに **+{農家ボーナス額} KHR**」 | 「コードを共有して **+{農家ボーナス額}** のボーナスをもらう」 |
| 買い手「紹介コード・クーポン」 | 「使えるクーポン ◯枚」 | 「コードを共有して **{クーポン額} KHR** クーポンをもらう」 |

- `{…額}`＝管理画面の現在値を**動的に引く**（数字を直書きしない）。

### 6-2 付与タイミング（referral spec §3 と整合・確定）
- **被紹介者（招待された新規）＝登録後**（レストランのクーポンは登録時にウォレット先渡し→初回注文で使う／農家のボーナスは登録時に保留付与・表示→自分の初回売上で +5,000 上乗せ）。
- **紹介元（招待した自分）＝被紹介者の初回取引完了後**（現金ボーナスのファーミング防止の要ゲート・緩めない）。
- ※ referral spec の更新が必要：§3「被紹介者農家ボーナス＝登録時に保留付与／初回取引で上乗せ」追記・§7「カード→1行メニュー＋専用ページ」・「順次提供予定／Phase 2」は稼働中に更新（特典は稼働済み）。

## 7. 横断ルール

### 7-1 保存フィードバック（確定）
- **設定の保存（基本情報・送金先/返金先・配送設定・公開ページ 等＝軽い・頻繁）＝トースト「✓ 保存しました」**（画面下中央のダークピル・✓は緑 `#4ADE80`・2〜3秒で自動消滅・暗転なし）。保存後は編集項目が表示モードに戻り新値が出る。
- **投稿する（重要・節目）＝完了画面**（中央・大きな✓・タップで進む）。＝投稿だけ別格。
- **保存エラー**（例：送金先で QRリンクも QR画像も空）＝**インラインのエラー表示**（赤メッセージ）。成功トーストは出さない。

### 7-2 色・ボタン
- CTA/操作＝青／削除など破壊的＝赤／GAqP・送料無料＝緑／注意・キャンペーン＝amber。確認ダイアログは**中央モーダル**（`fishlink-request-confirm-dialog-center.md`）。

### 7-3 必須/任意の明示
- 必須＝[必須]、いずれか片方＝[いずれか1つ必須]、任意＝[任意]（出品フォームのバッジ流儀と統一）。

## 8. データモデル（補足）

```
user {
  id,                       // 内部ID（adminのみ・画面に出さない）
  phone,                    // ログインに使う番号（基本情報で表示・OTPで変更）
  displayName,              // 表示名（公開）
  avatarUrl,                // 公開ページで設定・未設定＝デフォルト
  location { lat, lng, province, district, addressText },
                            // addressText＝ピンの逆ジオコーディング表示（読み取り専用）
                            // 公開は province/district のみ
  role,                     // farmer | restaurant
}
payoutAccount (農家・必須) { name, qrLink, qrImage }   // いずれかのQRは必須
refundAccount (買い手・任意) { name, qrLink, qrImage } // 未登録可
farmerDelivery (農家) { maxKm(必須), feePer10km(必須), freeKm(任意) }
                            // 全出品に共通・出品フォームと共有
referral { code, farmerBonusPendingCount, coupons[] }  // 金額は運営設定（admin）
```

## 9. 遷移

- マイページ top の各行 → 各画面（基本情報／公開ページ／送金先・返金先／配送設定／紹介専用ページ／運営メッセージ）。
- 各編集 → 保存 → **トースト**＋表示モードに戻る（新値表示）。
- 返金先（未登録）→〔返金先を登録する〕→ 編集。
- 電話番号 編集 → 確認コード送信 → OTP確認 → 番号変更。
- ログアウト → 確認（中央モーダル）→ ログイン画面。
