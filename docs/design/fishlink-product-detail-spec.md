# FISHLINK 商品詳細画面 実装仕様

実装の正本は **`fishlink-product-detail-mockup.html`**（構造・レイアウト・セクション順・コンポーネント配置はそこを参照）。
本書はHTMLから読み取れない情報＝**データモデル・状態分岐・表示ルール・遷移・ドメイン制約**を補う。

> ※この画面は既存実装が**レビュー前（旧版）の状態**。本書＋新モックの通りに更新すること（差分は §10 にまとめた）。

### 実装方針の優先順位（Home/魚一覧 specと同じ）
- **構成・データ・状態・表示ルール・遷移** → 本書とモックHTMLの通りに実装する。今回設計し直した部分。
- **見た目のスタイル（色・タイポ・余白・アイコンの絵柄）** → 既存アプリのデザインシステムに準拠。モックの色値・アイコン（ラインアイコン）は**役割を示す参考/プレースホルダー**。取引条件の各行アイコンもDSのアイコンに置換してよい（絵文字にはしない）。

---

## 1. 概要

**商品カード（Home・魚一覧）のタップ先**。レストランが1つの出品（魚種×サイズ×農家）の詳細を見て、**カートに入れる／今すぐ注文**を判断する発注前の確認画面。

- ドメイン制約は Home spec §1 と共通（養殖のみ / KHR / head/kg / GAqP=農家単位 / 供給不安定で複数農家から仕入れ）。
- 1出品 = **魚種×サイズ×農家**の単位。同じ農家が同魚種を別サイズで複数出品しうる。
- 価格は **内臓処理込みの単一値**で表示（§7）。

---

## 2. 画面構成（上から順・#12で確定）

ヘッダー（戻る＋魚名）→ スクロール本体 → 固定CTAバー（下端）。

| 順 | 要素 | 補足 |
|---|------|------|
| — | ヘッダー | 戻る `←` ＋ **魚名**（#3。FISHLINKロゴ/ボトムナビは出さない＝集中画面） |
| ① | 写真カルーセル | 複数枚・ドットインジケータ。**GAqPバッジは写真に重ねない**（#2） |
| ② | 魚名（サイズ） | 例「ヒレナマズ」＋「6-8 head/kg」 |
| ③ | 価格 | **内臓処理込みの単一値**＋直下に小さくグレーで「（内臓処理込み）」（#11/#14） |
| ④ | 農家行 | アバター＋農家名＋**★好評価率%(件数)**（#1）。タップで**生産者ページ**へ |
| ⑤ | GAqP認証ボックス | **認証農家のみ**表示する緑の説明ボックス（#2）。未認証は要素ごと非表示 |
| ⑥ | 取引条件ブロック | 1ブロックに統合（#10/#12）。行＝**産地+距離 / 配達の性質 / 最低注文量(任意) / 在庫** |
| ⑦ | 商品の説明 | 農家の記入文（一次情報）。**未記入ならセクションごと非表示**（#15）。**質問・返信より上**（#12改定） |
| ⑧ | 質問・返信（Q&A teaser） | 最新1スレッド＋「質問をすべて見る（◯件）→」。0件は質問導線（#6/#8） |
| ⑨ | この農家の他の出品 | 横スクロール。**現在の出品は除外・サイズ表示**（#5） |
| ⑩ | 固定CTAバー | **カートに入れる（主・青塗り）／今すぐ注文（副・青枠）**（#9） |

## 3. データモデル（JSオブジェクト形状）

```js
// 商品詳細（1出品）= 魚一覧の product を拡張
productDetail = {
  id,
  fishName,            // "ヒレナマズ"
  sizeHeadPerKg,       // "6-8 head/kg"
  photos: [..],        // 複数枚（カルーセル）
  priceKhrPerKg,       // 6300（内臓処理込みの表示価格＝単一の正。§7）
  stockKg,             // 80（在庫の実数。そのまま表示＝#13）
  minOrderKg,          // 5（最低注文量。null/0 なら行ごと非表示＝任意）
  distanceKm,          // 27.2
  description,         // 商品の説明（空文字/null なら ⑧ セクションごと非表示＝#15）
  farmer: {
    id, name,
    province,          // "プノンペン"（首都は州なし）/ "タケオ州"
    district,          // "チャンカルモン地区"
    ratingPositivePct, // 96（★96%。ratingCount=0 は「新規」＝#1）
    ratingCount,       // 32
    gaqp,              // true/false（true のとき ⑤ GAqPボックスを出す）
  },
  delivery: {          // 農家単位の配送設定（farmerDelivery）。詳細では"性質"のみ見せる（#10）
    deliverableRadiusKm, deliveryFeePer10km, freeDeliveryKm,
  },
  qa: { count, threads: [..] },   // ⑦。count=0 なら質問導線（#6/#8）
  otherListings: [ {id, fishName, sizeHeadPerKg, priceKhrPerKg, photo} ],  // ⑨。現在の id は除外（#5）
}

// Q&A スレッド（⑦・公開Q&A=A案）
qaThread = {
  id,
  question: { text, askerName, createdAt },
  replies: [ { text, authorName, authorRole /* "asker" | "farmer" */, createdAt } ],
}
```

## 4. 状態分岐

| 対象 | 条件 | 表示 |
|------|------|------|
| ⑤ GAqPボックス | `farmer.gaqp === true` | 緑ボックス＋説明文を表示。false なら**非表示** |
| ④ 評価 | `ratingCount === 0` | **「新規」**（`★0% (0)` は出さない＝#1） |
| ⑥ 最低注文量の行 | `minOrderKg` 未設定 | **その行ごと非表示**（任意フィールド・#10） |
| ⑥ 在庫 | 常時 | **実数で表示**「在庫 80kg」（#13。段階表示や"残りわずか"煽りはしない） |
| ⑦ Q&A | `qa.count === 0` | 「まだ質問はありません」＋**「この魚について質問する →」**（"他のコメントを見る"は出さない＝#6/#8） |
| ⑦ Q&A | `qa.count > 表示数` | teaser＋「質問をすべて見る（◯件）→」 |
| ⑧ 商品の説明 | `description` 空/未記入 | **セクションごと非表示**（#15。空状態文は出さない） |
| ⑨ 他の出品 | 0件 | セクション非表示（任意） |
| 在庫 < 最低注文量 | `stockKg < minOrderKg` | 実質購入不可。CTA/数量バリデーションで制御（カート(ii)と連動・§8） |

## 5. 表示ルール（既存DSに色は置換・役割は厳守）

1. **評価（#1）**: `★{好評価率}% ({件数})`。例「★96% (32)」。0件は「新規」。詳細の小数表記（"76.0"等）は**禁止**＝全画面％統一。
2. **GAqP（#2）**:
   - **カード**（Home/魚一覧）= 画像左上の省略バッジ「**✓ GAqP**」。
   - **詳細**（この画面）= 写真には重ねず、④農家行の下に**緑の説明ボックス**（見出し＋一文）。**役割分担**＝カードは識別の印、詳細は説明。
3. **価格（#11/#14）**: **内臓処理込みの単一値**を主役表示（青・大きめ）。直下に小さくグレーで「（内臓処理込み）」。内臓処理を外す操作（−525相当）は**カート**で扱う（詳細には出さない）。
4. **配送＝性質のみ（#10）**: 詳細では「**配達可・送料は配達1回ごと（量で変わらない）。金額はカートで確認**」と**性質だけ**書く。**1品基準の送料実額は出さない**（送料は農家ごと・配達1回に1回だけ＝まとめ買いで過大表示・誤解を招くため）。正確な総額は**カート**（農家ごとに1回計上）で出す。
5. **最低注文量（#10）**: `minOrderKg` があれば「最低注文量 ◯kg〜」と明示。無ければ非表示。
6. **CTA（#9・色は青一本化）**: **主＝「カートに入れる」（青塗り）／副＝「今すぐ注文」（白地青枠）**。**赤は購入に使わない**（danger=削除/出品停止 専用）／緑はCTAに使わない（GAqP/成功 専用）。理由＝核動線が複数農家カート集約（供給不安定）／今すぐ注文も結局カート画面に着地。

## 6. 画面遷移

| 要素 | 遷移先 |
|------|--------|
| 戻る `←` | 前の画面（Home/魚一覧）。**状態は呼び出し元で保持** |
| ④ 農家名・アバター | **生産者ページ**（農家詳細） |
| ⑦「質問をすべて見る（◯件）→」／「この魚について質問する →」 | **公開Q&Aページ**（質問一覧）。0件時は質問入力へ |
| ⑨ この農家の他の出品カード | その出品の**商品詳細**（同画面・別データ） |
| ⑩ カートに入れる | **カート**に追加（複数農家集約） |
| ⑩ 今すぐ注文 | **カート**に着地（1タップ即購入ではない＝#9） |

---

## 7. 画面間の統一ルール（厳守）

- **価格は単一の正を全画面で共有**（カード／詳細／生産者／他の出品／質問ヘッダー／カートが全部同値）。手数料込み・内臓処理込み。画面ごとに別フィールドを参照して食い違わせない（過去に 6,000/6,300/6,825/6,500 と画面で揺れた経緯あり）。
- **内臓処理は「込み」を基準**にし、外す操作（−525相当）はカートでのみ扱う（#11）。
- **評価・GAqP・通貨・サイズ**表記は Home/魚一覧 spec と統一。

## 8. 未確定・次フェーズ（実装範囲外メモ）

- **カート(ii) 最低注文量の下限バリデーション**：ステッパー下限を `minOrderKg` に固定＋初期値も `minOrderKg`（推し）。在庫 < 最低注文量 のときの購入不可制御もここで。
- **生産者ページ**の本レビュー（3軸メトリクス・出品中の魚グリッド・レビューコメント・GAqPをこのページにも出すか）。
- 公開Q&Aページの細部（権限ゲート＝自分のスレッド＋農家のみ「返信する」表示、他は閲覧のみ）。
- 多言語（クメール語）・低速回線対応。

## 9. 補足（モック参照／見た目はDSに置換）

- 取引条件ブロックの各行アイコンは**単色ラインアイコン**を想定（モックは参考。絵文字にはしない・DSのアイコンに置換）。
- 写真カルーセルの寸法・ドットUI・カード幅などの寸法はモックHTMLが参考値。

## 10. 既存実装との差分（旧版→更新・#1〜#15）

| 決定 | 旧実装（要修正） | 更新後 |
|------|------------------|--------|
| #3 ヘッダー | 「注文内容を確認」 | **魚名** |
| #1 評価 | 「★ 62.0 (37件)」小数表記 | **「★62% (37)」**％表記 |
| #9 CTA | 今すぐ注文＝赤塗り(主)／カートに入れる＝白枠 | **カートに入れる＝主(青塗り)／今すぐ注文＝副(青枠)**。赤は使わない |
| #6/#8 Q&A 0件 | 0件でも「他のコメントも見る →」 | **「この魚について質問する →」**（他コメント導線は件数>0のみ） |
| #5 他の出品 | 「この農家の他の魚」＋サイズ無し | **「他の出品」＋サイズ表示＋現出品を除外** |
| #10/#12 取引条件 | 在庫バッジ＋距離のみ | **取引条件ブロックに統合**（産地+距離／配達の性質／最低注文量／在庫） |
| #2 GAqP（軽微） | ボックス見出し「✓ GAqP認証済み」 | 「**GAqP 認証農家**」に統一（任意） |

> 注：価格の単一正（§7）・内臓処理込み注記（#14）・GAqP説明ボックス（#2）・在庫実数（#13）は旧実装でも整合が取れている。
