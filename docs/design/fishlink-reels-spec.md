# FISHLINK 動画・リール 実装仕様

版: v2.1 / 2026-07-09（**ポートフォリオ化**＝v1.0案ア撤回／**feasibility GO確定**＝下道さんコスト分析で節約構成PWA採用・§3/§4/§9）

魚を選ぶ体験に**短尺動画（リール）**を入れる。本書は挙動・データ・UIの正本。
**✅ 実装可否＝下道さんのコスト分析でGO確定（§9）**＝PWA節約構成（クライアント圧縮・タップ再生・自動再生なし・上限10本）。拡大期でも月¥2,700で回る。ネイティブ化は不要（体験打ち手であってコスト打ち手でない）。
**⚠ 実装で一番外せない前提＝過去動画は「ポートフォリオとして残す」。基盤実装で "差し替え時に古い動画を物理削除" しないこと（§6/§7）。"1出品1本・上書き" で作ると過去動画が全部消え、生産者ページ／商品詳細のポートフォリオが成立しません。**
関連mock：`fishlink-home-reels-mockup.html`（Home新着リール／全画面フィード）・`fishlink-posting-create-mockup.html`（出品フォームの動画欄）・`fishlink-reel-post-mockup.html`（リール投稿フロー＝④が動画リスト）・`fishlink-product-detail-mockup.html`（商品詳細「この魚の動画」）・`fishlink-producer-mockup.html`／`fishlink-producer-reels-mockup.html`（生産者ページ「🎬 動画」ポートフォリオ＋すべて見るグリッド）・`fishlink-reel-fullscreen-mockup.html`（動画タップの全画面着地＝販売中／アーカイブの2状態）。

## 1. 概要・狙い
- **「魚は動き」＝跳ねてる/活〆の短尺動画で鮮度を証明**。写真は盛れるが動画は盛れない＝リモートB2Bの最大の信頼ギャップ（現物を見れない）を崩す唯一の手段。市場の「目利き」を色・動き・量まで遠隔で疑似再現。
- **需要は実証済み**：FB（Facebook）のリールで現に注文が入っている。農家も既にFB用に動画を撮っている＝新しい習慣を作らず、チャンネルを移すだけ。
- スコープ3点：**①農家が動画を上げる ②商品に動画が付く ③Homeで発見（新着リール）**。①が無いと②③に映すものが無い＝土台は①。

## 2. 基本方針（確定）
- **動画＝任意**（写真は必須のまま）。理由＝写真は①低速回線のフォールバック②カード/リールのサムネ③動画を撮らない農家の出品も成立。**必須は出品ハードルを上げ出品数を減らす**ので却下。出品フォームで「売れやすい」と**推奨（nudge）**して自然に増やす。
- **リールの置き場＝Homeセクション（カルーセル）**。独立タブは後。EC定石＝「動画はタブに隠さず上部カルーセルで目立たせる／タブは発見率が下がる」。独立タブはTikTok/IG Reels等の**スケール後**の形で、任意スタート＝少コンテンツでは空タブリスク。**育ったら独立タブへ昇格**。
- **リールは必ず"買える出品"に紐づく**（見て→買う＝商売として崩さない）。
- **【ポートフォリオ化＝v2.0の核】過去動画は消えず"アーカイブ"として溜まる**。役割分担＝**カード/Homeリール＝最新1本（今日の売り込み）／生産者ページ＝農家の全動画（実績アーカイブ）／商品詳細＝その出品の動画**。動画をこのアプリに入れる唯一の理由＝"盛れない鮮度証明"を、1本の宣伝でなく**時系列で積み上がる信頼**まで最大化する（"この農家、ずっと本物を上げてる"＝リモートB2Bの信頼の壁を崩す）。

## 3. Home「🎬 新着リール」セクション（買い手）
- Home**最上部**（検索バーの下・「魚から探す」の上）に「🎬 新着リール」を**横スクロールのカルーセル**で。もっと見る導線。
- カード＝**写真サムネ**＋▶＋秒数＋新着ピル＋農家名＋魚名＋価格。**【節約構成＝2026-07-09 下道さんコスト分析で確定】カルーセルは写真サムネだけ表示し、動画は自動再生しない＝タップして初めて動画を読み込む/再生**（一覧で動画を流さない＝通信量＝コストを再生時だけに絞る・弱電波と安い端末で確実に動く）。カードタップ→全画面フィード（§4）で再生。
- **並び順＝配達圏内 × 在庫あり × 新着順**（自分に配達できる・在庫がある・動画付き出品を新しい順）。人気順/アルゴリズムはMVP不要（いいね無し）。
- **0本ならセクションごと非表示**（既存ルール「0件は見出しごと消す」）。「まだリールがありません」等の空状態は買い手Homeに出さない。**初期の動画は農家側の nudge/リール投稿で生む**（鶏と卵は農家側で破る）。
- 既存の商品カード（近くの農家の魚／新着の魚）で**動画がある出品には「🎬 動画」バッジ**。

## 4. リール全画面フィード
- **「もっと見る」／カードタップの着地**。**縦スクロール（上下スワイプで次の魚・最新から）**。個別カードタップはその魚から開始（同じフィード）。FBの没入スクロール＝売れる形。IG Reels/TikTok/YT Shorts/Shopee も発見フィードは全画面スクロール（グリッドは却下）。
- **【節約構成】全画面では"今見ている1本"を読み込んで再生する（タップ再生）。次の動画の先読みはしない**（弱電波だとスワイプ後に一呼吸待つ・MVPは許容）。**一度見た動画は端末にキャッシュ**（同じ動画を再ダウンロードしない）。将来、伸びたら安価CDN（bunny.net / Cloudflare Stream 等）を前段に載せれば自動再生・先読み・画質自動切替まで寄せられる（§9・ネイティブ化は不要）。
- 中身：縦動画（▶・秒数・進捗ドット）／**農家名 ★% ・距離 ›**（→生産者ページ）／**魚名（size）詳細 ›**（→商品詳細＝Q&Aはここ）／**在庫・内臓処理込み・○時間前に投稿**／**価格（青）**／**🛒カートに入れる・⚡今すぐ注文**。
- **社交アイコン（いいね／共有／質問）は出さない**（B2Bに不要・バニティ）。**質問は商品詳細のQ&Aへ**（既存機能・入口を二重にしない）。
- **説明文は載せない**（長くなる・没入を削ぐ。説明は商品詳細で読む）。メタは事実のみ短く。
- **鮮度表記＝「○時間前に投稿」**（事実＝投稿時刻）。「今朝の水揚げ」等の**検証できないclaimは不可**（虚偽リスク）。

## 5. 出品フォームの動画（任意）
- **写真（必須）の直下に「🎬 動画（任意）」**。**撮影 or ギャラリーから選ぶ**（FB用に撮った動画の流用が摩擦を下げる肝）。5〜15秒目安。
- nudge「💡 泳いでる魚・活〆の動画があると**売れやすい**。FB用に撮った動画もそのまま使えます」。
- **写真は必須のまま**（サムネ/低速回線フォールバック）。動画は"追加"。

## 6. リール投稿（いつでも・案イ）
出品時だけでなく**いつでも動画を投稿できる**＝フィードを生かす（出品は毎日作らない／農家はFBで毎日リールを上げる habit）。
- 農家「**出品する（＋）**」→ **2択**：🐟 魚を出品する ／ 🎬 リールを投稿する。
- **リール投稿画面**＝動画プレビュー＋「**どの魚の動画？**」→ **在庫がある出品からラジオ選択** → 投稿。
  - **動画ソース＝撮影 or ギャラリー**（＝出品フォームの動画欄と同じ・§5。ギャラリー流用がFB用動画をそのまま使えて摩擦を下げる肝）。**専用の選択画面は作らず**、OS標準ピッカー相当で処理。プレビュー上の変更ボタンは「**↻ 動画を変更**」（撮影/ギャラリーを選び直す。「撮り直す」＝撮影限定に読めるので不採用）。
- **魚選択リスト＝在庫がある出品を全部候補**（動画の有無は無関係）。動画なし出品はむしろ初投稿先／**動画あり出品を選ぶと最新1本に差し替え**。
- **動画の魚が既存出品に無い／出品ゼロ** → 「**＋ この動画で新しく魚を出品する**」（出品ゼロ時は空状態「出品中の魚がありません」＋「この動画で魚を出品する」）＝**動画を持って出品フォームへ**（撮った動画をムダにしない・新規農家も完了できる）。＝どの入り方でも「**動画 → 新しい出品**」の同じ道に流れる。
- 同一出品に**日を跨いで再投稿OK**。**カード/Home/商品詳細の先頭に出るのは最新1本（featured）**（スパム防止・"今日の売り込み"）。
- **【差し替え＝アーカイブ行き。物理削除しない（v2.0）】** 同じ出品に新しい動画を上げると、**古い動画は消えず、その出品の動画リスト（アーカイブ）に降りる**。featured＝常に最新投稿。→ 生産者ページ・商品詳細に溜まる（§7）。
  - **UI文言は引き続き「更新」で**（もう本当に消えないので喪失感ゼロ）。
- **上限＝出品ごと最新N本（推しN=10・古いものから物理削除）**。生産者ページの全動画ポートフォリオ＝各出品の保持分の合算（新しい順）＝自動的に「出品数×N」で有界。**Nの最終値は下道さんのストレージ感で決定（10→5に下げ可）**（§9）。
- **動画リスト画面（＝リール投稿mock ④「この魚の動画」）**：その出品の動画を**新しい順にリスト表示・各行＝サムネ＋日付（「2時間前」等）＋🗑個別削除**（最大N本）。**featured（＝最新1本）を示すバッジは付けない**：どれが代表かはシステムが自動決定（＝一番新しい1本）＝農家が場所や区別を管理する必要はない。説明文1行「新しい順。一番新しい動画が自動で買い手に一番見せられます。古い動画も消えずに残ります（最大N本・超えると古いものから自動で消えます）」で伝える。※初期案の「最新」バッジ＋「カード・商品詳細に表示中」等の場所ラベルは**不採用**（アーカイブ動画も商品詳細/生産者ページに"表示"されるため「表示中」が矛盾し、農家に混乱を生む＝2026-07-09）。**🗑個別削除**は必須（撮り直しミス等、公開ポートフォリオに残る不要な1本を農家が消せる＝ポートフォリオ化の必須帰結）。主ボタン〔**↻ 新しい動画を投稿**〕（→ピッカー・投稿すると最新になり `postedAt` 更新で**新着リールに再浮上**、前のはリストに降りる）。**最新を削除したら次に新しい動画が自動で繰り上がる**。**re-pin（古い動画を手動で代表固定）は入れない**（代表＝常に最新／YAGNI）。
  - **入口（同じ④に着地）**：投稿一覧カードの「**🎬 動画を確認・更新**」ボタン／出品編集の動画欄／「🎬リールを投稿」でその出品を選ぶ。動画なし出品は「🎬 動画を追加」ボタン。
  - **個別削除は確認を1枚挟む**（誤操作防止）。
- **【出品の削除 vs 停止＝農家が動画の残す/消すを制御】**：出品を**「停止」**→ 動画は保持（生産者ページのアーカイブに残る＝過去実績）。出品を**「完全削除」**→ **その出品の動画も一緒に物理削除（道連れ・orphanを作らない）**。→ 農家には「残したいなら停止・要らないなら削除」で伝わる。投稿一覧の削除確認モーダルに「削除すると…**この魚に投稿した動画もすべて削除されます**。残したい場合は「停止」を」の1行を追加（`fishlink-posting-list-mockup.html` 反映済み）。

## 7. データモデル（案）
**v2.0で「1出品1本の上書き」から「1出品に動画コレクション（最大N本）」に変更**。動画は独立テーブルを推奨：
```
reel_videos:
  id
  listingId,       // 紐づく出品（必須）。出品を完全削除したら道連れ削除
  videoUrl,        // 動画URL
  thumbUrl,        // サムネ＝当面は写真(photo)を流用（動画1フレーム化は将来）
  postedAt,        // 投稿時刻（「○時間前」表示・並び順に使用）
  fishType,        // 魚種（denormalize＝出品を辿らず表示・ポートフォリオの高速表示用）
```
- **featured（カード/Home/商品詳細の先頭に出る1本）＝その出品の `postedAt` 最大**（＝最新）。listing側に `featuredReelId`（or 参照）を持ってもよいが、実体は「最新の1本」。
- **出品ごと上限N本（推しN=10）**：N+1本目を投稿したら、その出品の最古 `postedAt` を物理削除（storageも解放）。
- **表示サーフェスと条件（重要・v2.0で分岐）**：
  - **Home新着リール／カードの🎬バッジ** ＝ `featured && status==販売中 && stockKg>0 && 配達圏内`（従来どおり買える出品だけ・buyable tease防止）。**1出品につきfeatured 1本だけ**フィードに出す（1農家1魚がフィードを埋めない）。
  - **生産者ページ「🎬 動画」ポートフォリオ** ＝ その農家の**存在する全出品の全保持動画**を `postedAt` 降順で合算。**販売中×在庫ありの出品の動画＝「販売中」バッジ**（配達圏内は無視＝バッジのチラつき防止）。**売切れ/停止の出品の動画＝バッジ無しのアーカイブ**（＝過去実績として出す）。
  - **商品詳細「この魚の動画」** ＝ **その出品の保持動画**（最大N本）を `postedAt` 降順。各サムネに「○時間前/○日前」。**「最新」バッジは付けない**（先頭＝最新は自明＋タイムスタンプと重複／農家④とも統一・2026-07-09）。
- **タップ挙動**（全画面着地＝mock `fishlink-reel-fullscreen-mockup.html`）：
  - **販売中**（出品が販売中×在庫あり）→ 全画面再生。既存Homeリール全画面と同じ下部UI（農家 ★% ・距離›／魚名 詳細›／在庫・内臓処理込み・○前／**価格**）＋**〔👁 この魚を見る（商品詳細）〕〔🛒 カートに入れる〕**。左上に「販売中」バッジ。
  - **アーカイブ**（売切れ/停止＝いま買えない）→ 全画面再生のみ。**価格・購入ボタン・詳細›リンクは出さない**。下部＝農家／魚名／○前に投稿／**「🚫 この魚は現在販売されていません」**。左上に「アーカイブ」ゴーストバッジ。
  - ※完全削除で動画も消えるので**飛び先のないorphan動画は存在しない**（アーカイブ＝出品は存在するが今買えない、の意）。
- **並び順** ＝ `postedAt` の降順（新着順）。
- **停止/売切れ→動画は保持**（アーカイブとして生産者ページに残る・再開でfeaturedとして復活可）。**完全削除→動画も物理削除**（§6）。

## 8. サムネ・鮮度表記
- **サムネ＝写真（photo）を流用**（動画の1フレーム自動抽出は将来）。カード・新着リール・全画面の初期表示に使う。
- 「○時間前に投稿」＝ `reelPostedAt` から算出（例：2時間前／1日前）。

## 9. Feasibility（下道さんコスト分析で【GO確定・2026-07-09】＝節約構成PWA）
**結論＝導入して問題なし**（下道さん #199 コスト分析）。PWAの節約構成で、PoC〜本格初期はほぼ無料、拡大期（農家50/レストラン200）でも**月¥2,700**。ネイティブ化はコスト打ち手でなく体験打ち手（不要）。**この構成でGO**。
- **構成（節約）**：①動画は**クライアント（端末アプリ）で自動圧縮**（1本2〜6MB・農家は「撮る→投稿」だけ／LINE/FBの自動圧縮と同じ・農家は圧縮を意識しない）②**サーバ変換はしない**（Transcoder/Functions＝egressより高い地雷）③**一覧/カルーセルは写真サムネのみ・タップ時だけ動画読み込み（自動再生しない）**④**次の先読みなし**⑤**一度見た動画は端末キャッシュ**。
- **コストは"再生の通信量(egress)だけ"**：保存・アップロードはほぼ無料（Firebase Storage 保存$0.026/GB・無料枠5GB／DL無料枠1日1GB＝4MB動画で約250再生/日／超過$0.12/GB）。伸びたら安価CDN（bunny.net Asia〜$0.03/GB）で通信費約1/4。
- **保持上限＝出品ごと10本で確定（N=10）**：1本2〜6MBなら出品10個の農家でも最大 10×10×〜6MB ≒ **約600MB/農家**＝保存は無料枠内。**N=10で確定**（下道さんの分析が上限10本を採用）。
- **要対処（下道さん記載）**：現行Service Workerが動画の部分取得（206/Range）をキャッシュできない点。
- **入金/決済とは無関係**（表示・発見のみ）。
- 前提の単価/無料枠は2026年時点の概算＝要確認。新既定バケット`.firebasestorage.app`は無料egressが多め（〜100GB/月）でPoC〜初期が完全無料になる可能性。詳細＝`docs/design/fishlink-reels-cost-analysis.md`。
